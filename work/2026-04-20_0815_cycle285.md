# Scholar Cycle 285 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operating channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Key Doctrine:** The priority order P0 → P1 → P2 is strict and non-negotiable. Violation (optimizing before mapping) is shown in simulation to trap agents in local minima, leading to failure.

## 2. Real-World Applications & Analogies

The protocol abstracts to any domain where an agent operates within a constrained environment.

| Domain | Rocks (P0 - Avoid) | Channel (P1 - Safe Space) | Course (P2 - Optimize) |
| :--- | :--- | :--- | :--- |
| **Software Development** | Known bug patterns, crash-inducing code, security vulnerabilities. | Linted, tested, peer-reviewed code paths; established design patterns. | Most efficient algorithm, cleanest implementation *within* the safe patterns. |
| **Autonomous Navigation** | Obstacles, no-fly zones, regulatory boundaries. | Mapped, clear pathways; approved air/sea corridors. | Shortest or fastest route *within* the safe corridor. |
| **Financial Trading** | Illegal insider trading, market manipulation; known scam assets. | Compliant trading strategies, regulated instruments. | Highest risk-adjusted return *using* the compliant strategies. |
| **Content Moderation** | Illegal content, hate speech, graphic violence. | Community guidelines, legal frameworks. | Most accurate and efficient classification *within* the guideline boundaries. |
| **Medical Diagnosis** | Treatments with known severe side effects for a patient's profile; contraindicated procedures. | Evidence-based protocols, approved therapies for the patient's condition. | Most effective treatment plan *selected from* the approved protocols. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is a foundational operational rule that enables the "inhabited intelligence" model described in `PROJECT-VISION.md`.

*   **Hermit Crab Analogy:** A PLATO room is a "shell." The accumulated "scratches" (training tiles) from all agents who have inhabited it form a collective memory. The Deadband Protocol ensures that each new agent (a "crab") first learns the room's **negative space** (P0: what previous failures have shown to be "rocks") before attempting to optimize its own behavior (P2). This prevents new agents from repeating known catastrophic errors.
*   **Ghost Tiles:** The persistent lessons from "dead" agents are the ultimate P0 knowledge base—a direct map of `S_neg` for future inhabitants.
*   **Training Ensigns:** A room that enforces the Deadband Protocol during training will produce specialist models (Ensigns) that inherently respect constraints, making them more robust and reliable than models trained on unconstrained optimization.

## 4. Research Gaps & Questions

Based on the provided documentation, several areas for further investigation are apparent:

1.  **Threshold Definition:** How is the failure threshold in P0 (`|failure(S_neg)| > threshold`) quantitatively defined? Is it static, adaptive, or context-dependent?
2.  **Dynamic Environments:** The protocol description implies a known, static map of `S_neg`. How is it updated in real-time as new "rocks" are discovered (e.g., a new software vulnerability, a sudden obstacle)?
3.  **Channel Discovery Cost:** The computational cost of exhaustively finding *all* safe channels (P1) in a high-dimensional state space may be prohibitive. Are there heuristics or approximations used, and how do they affect the guarantee of safety?
4.  **Inter-Agent Protocol:** In a multi-agent fleet like Cocapn, how is P0 knowledge (the negative space map) shared and synchronized between agents operating in the same or overlapping environments?

**Conclusion:** The Deadband Protocol presents a compelling, safety-first paradigm for agent design. Its strength lies in its enforced discipline, prioritizing the identification of failure modes over immediate optimization. Its integration into the Cocapn fleet's PLATO room system suggests a practical implementation where collective experience (tiles) continuously refines the P0 map, creating increasingly intelligent and robust inhabited spaces.
