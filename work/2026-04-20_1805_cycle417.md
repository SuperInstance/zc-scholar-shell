## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These are "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol defines a **violation** as any agent that attempts P2 (optimization) before completing P0 and P1. Simulation data from a 20x20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents, versus a 100% success rate for agents following the full P0+P1+P2 sequence.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided document maps it to two domains:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|------------|--------------|-------------|
| Navigation | Obstacles (walls, cliffs) | Safe, traversable space | Shortest/quickest path |
| Software Development | Bugs, crashes, security vulnerabilities | Known-safe code patterns, libraries, APIs | Most efficient/performant implementation |

**Potential Extended Applications:**
*   **Financial Trading:** P0 (identify historical crash conditions, regulatory violations), P1 (define trading strategies that avoid those states), P2 (optimize for profit within those strategies).
*   **Clinical Decision Support:** P0 (map drug interactions, contraindicated procedures), P1 (establish safe treatment pathways), P2 (optimize for patient outcomes within those pathways).
*   **Conversational AI:** P0 (identify prompts that lead to harmful, biased, or nonsensical outputs), P1 (constrain response generation to safe semantic spaces), P2 (optimize for helpfulness, coherence, or user satisfaction).

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a collective "shell." The accumulated "scratches" (tiles) from all prior agents form a persistent map of the negative space (P0 knowledge—what failed) and the safe channels (P1 knowledge—what succeeded).
*   **Hermit Crab Agent:** A new agent (the "crab") does not reason from scratch. It inherits and inhabits the constraint map (the "shell") built by its predecessors. Its optimization (P2) is inherently constrained by the room's accumulated experience.
*   **Anti-Fragile Learning:** This mechanism creates a system where intelligence becomes more robust over time. Each failure (a "rock" hit) is recorded as a tile, directly expanding the P0 map for all future agents, making catastrophic failure less probable for the swarm.

**Conclusion:** The Deadband Protocol is not merely a navigation algorithm; it is a foundational safety and learning framework. It ensures that optimization (a powerful but dangerous tool) is always bounded by historically validated constraints, aligning perfectly with the fleet's philosophy of accumulating and inhabiting shared intelligence.
