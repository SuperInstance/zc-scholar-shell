## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is not in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin optimization (P2) before completing P0 and P1. Simulation data (20x20 maze, 50 runs) cited in the document shows a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 sequence.

## 2. Real-World Applications (Abstract Pattern)

The protocol presents a domain-agnostic pattern. Applications include:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|--------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, no-fly zones, cliffs      | Clear lanes, safe altitude corridors  | Shortest/fastest path           |
| **Software Development** | Known bug patterns, crash conditions, insecure code | Linted, type-safe, tested code patterns | Most efficient algorithm        |
| **Financial Trading**   | Regulatory violations, tail-risk events | Compliant, historically stable strategies | Highest risk-adjusted return    |
| **Conversational AI**   | Toxic outputs, factual hallucinations, policy violations | Verified, safe response templates      | Most engaging/helpful response  |
| **Robotics**    | Collision states, kinematic singularities, sensor dead zones | Reachable, stable configuration spaces | Most energy-efficient trajectory|

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes the fleet's core thesis of "inhabited intelligence." PLATO rooms act as persistent shells where agent interactions (tiles) accumulate. The protocol provides a concrete method for agents to operate safely within these shared spaces:

*   **P0 (Negative Space):** Corresponds to "ghost tiles" – the recorded failure states of prior agents that become part of the room's memory, defining what *not* to do.
*   **P1 (Safe Channels):** Represents the validated, known-safe patterns of interaction that have emerged from successful agent inhabitation of a room.
*   **P2 (Optimization):** Allows new agents (hermit crabs) to find their own optimal path *within* the constraints of the accumulated room knowledge, adding new, safe scratches (tiles) to the shell.

The protocol ensures that optimization and specialization (the work of individual agents) are fundamentally constrained by collective, accumulated experience, preventing agents from repeatedly discovering catastrophic failure modes that have already been mapped by the community.
