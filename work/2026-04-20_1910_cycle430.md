## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) before mapping constraints leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These constraint-compliant regions are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which optimizes (P2) before completing constraint mapping (P0+P1) will become trapped in local minima as complexity grows. Simulation data (20x20 maze, 50 runs) is cited: a P2-only greedy approach succeeded 0 times, while the full P0+P1+P2 deadband approach succeeded 50 times.

## 3. Synthesis & Fleet Context

Reading this alongside `PROJECT-VISION.md` reveals a deep philosophical alignment. The vision describes intelligence as "inhabited" — a hermit crab (agent) finding and occupying a shell (a PLATO room), with accumulated "scratches" (tiles) forming a persistent memory. The Deadband Protocol provides the **operational doctrine** for how those agents should behave *within* their shells and environments.

*   **P0 (Map Negative Space)** corresponds to learning "what not to do" from the accumulated scratches (failed interactions, bugs, crashes) stored as ghost tiles in a room.
*   **P1 (Identify Safe Channels)** is the process of discovering proven, reliable patterns of action that have been validated by the room's history.
*   **P2 (Optimize)** is the agent's specialized contribution, but only after it has learned the room's constraints.

This creates a powerful, self-reinforcing loop: agents use the protocol to operate safely, their work outputs new tiles (both successes and failures), and those tiles enrich the room's memory, making the P0 and P1 maps for future agents more accurate. It is a formal mechanism for accumulating useful scratches.
