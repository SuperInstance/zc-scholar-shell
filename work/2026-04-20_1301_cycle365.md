# Scholar Cycle 365 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden zones.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or permissible operational zones.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict and non-negotiable. Violation (optimizing before mapping) is shown in simulation to trap agents in local minima with a high probability of failure.

## 2. Real-World Applications & Analogies

The protocol abstracts to any domain where an agent operates within a bounded, potentially hazardous space.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Marine Navigation** | Shoals, reefs, restricted zones. | Charted deep-water channels, fairways. | Most fuel-efficient route within the channel. |
| **Software Development** | Known bug patterns, crash-inducing code, insecure APIs. | Linter-approved patterns, tested libraries, style guides. | Most performant or elegant implementation using safe patterns. |
| **Financial Trading** | Regulatory violations, black-swan event scenarios. | Compliant strategies, defined risk limits. | Highest risk-adjusted return within the strategy envelope. |
| **Robotic Motion Planning** | Collision geometry, torque limits, unstable configurations. | Collision-free configuration space. | Shortest-time or most energy-efficient trajectory. |
| **Content Moderation** | Hate speech, misinformation, illegal content (as defined by policy). | Community guidelines and legal frameworks. | Most engaging user experience that respects the guidelines. |
| **Medical Diagnosis** | Contraindicated treatments, harmful drug interactions. | Evidence-based treatment protocols for the diagnosed condition. | Most effective treatment option from the protocol, considering patient factors. |

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol is not just a navigation rule; it is a foundational **training constraint** for the Cocapn fleet's inhabited intelligence model (`PROJECT-VISION.md`).

*   **P0 as Ghost Tiles:** The "negative space" is learned from accumulated failure. In a PLATO room, these are the "ghost tiles"—lessons from dead agents that define what *not* to do. A room's memory of failure shapes its P0 map for all future inhabitants.
*   **P1 as the Room's Shell:** The "safe channels" are the accumulated, validated knowledge and behavior patterns within a PLATO room. This is the room's "shell"—the scratched-in wisdom from all prior hermit crab agents that defines where new agents *can* operate successfully.
*   **Constrained Optimization (P2):** An agent (or an exported "Ensign" model) performs its specialized task *only within* the behavioral channels defined by the room's accumulated experience. This prevents the agent from "hallucinating" or straying into failure modes the collective has already mapped.

**Conclusion:** The Deadband Protocol operationalizes the fleet's core thesis. Intelligence is not an unbounded, constructed model, but a **constrained optimization process within a historically informed space of safe operation**. The PLATO room system is the mechanism that grows, maintains, and exports these constraint maps (P0+P1), enabling specialized agents (P2) to perform reliably.
