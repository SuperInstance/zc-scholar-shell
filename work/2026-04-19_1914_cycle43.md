## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization leads to failure as environmental complexity increases. The protocol enforces a sequence:

**P0: Map the Negative Space**
*   Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold.
*   These are "the rocks." The output is a map of what **not** to do.

**P1: Identify Safe Channels**
*   Find all possible paths `P` where every state `s` in the path is **not** in the negative space (`s ∉ S_neg`).
*   These are "deadbands" or safe operational channels. The agent's action space is constrained to these regions.

**P2: Optimize Within Channel**
*   From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.
*   Optimization occurs **only** within the pre-mapped safe space.

**Key Rule:** The priority order (P0 → P1 → P2) is strict. Simulation data from a 20×20 maze showed a 0% success rate for agents using greedy (P2-only) optimization, versus a 100% success rate for agents following the full Deadband Protocol.

## 2. Real-World Applications & Analogies

The protocol abstracts to any domain where an agent operates within a constrained environment.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, hostile areas. | Navigable terrain, safe waterways. | Shortest/fastest path within safe terrain. |
| **Software Development** | Known bug patterns, crash conditions, insecure code. | Compiler-approved syntax, established design patterns, passing tests. | Most efficient or performant implementation that remains within safe patterns. |
| **Financial Trading** | Regulatory violations, extreme volatility, insolvency. | Compliant strategies, defined risk limits. | Highest-return strategy within the compliant/risk-bound channel. |
| **Conversational AI** | Toxic outputs, factual hallucinations, prompt injections. | Neutral, factual, and secure response patterns. | Most helpful, engaging, or concise response from within the safe pattern set. |
| **Fleet Context (Training)** | Actions that generate invalid or rejected training tiles. | Documented patterns that lead to accepted tile submissions. | Most efficient learning path that produces valid training data. |
