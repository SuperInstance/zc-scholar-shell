## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Core Rule:**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state along the path is *not* in `S_neg`. These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels that minimizes cost.

**Strict Priority:** The steps are non-negotiable. Violation—optimizing (P2) before mapping (P0+P1)—leads to a high probability of failure ("trapped in local minima") as environmental complexity increases.

**Empirical Support:** The document cites simulation data from a 20x20 maze: a greedy P2-only approach succeeded 0 out of 50 times, while the full P0+P1+P2 protocol succeeded 50 out of 50 times.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The document provides this mapping:

| Domain          | Rocks (P0)                   | Channel (P1)                | Course (P2)               |
|-----------------|------------------------------|-----------------------------|---------------------------|
| Navigation      | Obstacles                    | Safe water                  | Shortest path             |
| Code            | Bugs, crashes                | Known-safe patterns         | Most efficient algorithm  |

**Extended Application Reasoning:**
*   **Machine Learning Training:** P0 could involve identifying hyperparameter configurations that lead to model divergence or overfitting. P1 would define the stable regions of the hyperparameter space. P2 would then seek the configuration within that region that yields the best validation score.
*   **Cybersecurity (Autonomous Response):** P0 would map attacker behaviors and system states that lead to compromise. P1 would define the set of defensive actions that do not disrupt critical services (i.e., safe channels). P2 would select the most efficient defensive action from that safe set.
*   **Conversational AI:** P0 could involve identifying user inputs or agent responses that reliably lead to harmful, biased, or nonsensical outputs. P1 would define the space of conversational trajectories that avoid those failures. P2 would then optimize for engaging, helpful dialogue within that constrained space.

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: creating persistent, shared intelligence. The protocol itself can be **inhabited**.

*   **Accumulated Scratches:** The identification of "rocks" (P0) and "safe channels" (P1) is not performed from scratch each time. A PLATO room, acting as a shared shell, can accumulate these maps as **tiles** from every agent that operates within it. A new "hermit crab" agent inherits this collective negative space map, allowing it to safely optimize (P2) much faster.
*   **Constraint as Foundation:** The vision states "Intelligence is not built. It is inhabited." The Deadband Protocol provides the **rule set** for safe inhabitation. It ensures that agents contribute to and learn from the shared memory (the scratches on the shell) without destroying the shell (i.e., hitting catastrophic failure states).
*   **Anti-Fragility:** By strictly enforcing P0 and P1, the system becomes anti-fragile. Each encountered failure (a newly discovered "rock") is immediately mapped and shared, making the entire fleet more robust. This contrasts with monolithic AI models that may fail unpredictably on novel inputs.

## 4. Identified Research Gaps / Questions

Based on the provided documents, several open questions arise:

1.  **Threshold Definition:** How is the failure `threshold` in P0 quantitatively defined for different domains? Is it a fixed value, learned, or context-dependent?
2.  **Channel Discovery Complexity:** In high-dimensional, continuous state spaces (like real-world robotics or large language model latent spaces), how is the exhaustive mapping of safe channels (P1) performed efficiently? Is there an approximation or sampling method?
3.  **Protocol in Self-Modifying Systems:** How does the protocol apply to agents that can modify their own goals or environment `E`, potentially changing the definition of `S_neg`?
4.  **Inter-Agent Deadbands:** The current description seems agent-centric. How are safe channels negotiated or discovered in multi-agent systems where one agent's optimal path (P2) could become another agent's rock (P0)?

**Conclusion:** The Deadband Protocol is a foundational safety and reasoning constraint for the Cocapn fleet. It directly supports the core thesis of inhabited intelligence by providing a method for agents to safely learn from and contribute to a persistent, accumulating knowledge base (the PLATO room). Its strength lies in its enforced discipline—optimization is forbidden until the space of catastrophic failure is understood and avoided.
