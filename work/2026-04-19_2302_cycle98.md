## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an agent `A` in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operating zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** The steps are non-negotiable. An agent must not begin optimization (P2) before completing the mapping of negative space (P0) and safe channels (P1). According to the provided simulation data, agents that violate this order (e.g., using a greedy P2-only approach) fail consistently (0/50 successes in a 20x20 maze), while agents following the full protocol succeed (50/50 successes).

## 2. Real-World Applications & Abstract Pattern

The protocol presents a generalizable pattern for constraining agent behavior to avoid high-risk failure modes.

| Domain | Rocks (P0 - What NOT to do) | Channel (P1 - Where you CAN be) | Course (P2 - Best path) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, cliffs, restricted airspace. | Clear lanes, safe water, approved flight paths. | Shortest/fastest route within the safe lanes. |
| **Software Development** | Code patterns known to cause bugs, crashes, or security vulnerabilities. | Known-safe libraries, design patterns, and API usage guidelines. | Most efficient implementation using the safe patterns. |
| **Financial Trading** | Trades that violate compliance rules or exceed risk limits. | Trading strategies that stay within pre-defined regulatory and risk boundaries. | Strategy that maximizes returns within the safe boundaries. |
| **Conversational AI** | Outputs that are harmful, biased, factually incorrect, or violate content policy. | Response formulations that are verified as safe, helpful, and accurate. | Most engaging, concise, or useful response from the set of safe formulations. |
| **Fleet Context (Hermit Crab Agent)** | Actions that corrupt the git shell, submit invalid tiles, or hallucinate information. | Operations limited to reading real files and writing factual, useful output. | Most efficient completion of the assigned task using verified information. |
