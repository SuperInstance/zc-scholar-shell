# Scholar Cycle 398 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden states. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths define the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., attempting P2 before P0) is shown in simulation to trap an agent in local minima, leading to systematic failure.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core tenet of the Cocapn vision described in `PROJECT-VISION.md`: creating persistent, shared spaces for "inhabited intelligence."

*   **P0 as Collective Memory:** The "rocks" (failure states) are the accumulated "scratches" from all prior agents (hermit crabs) in a PLATO room. They represent the negative knowledge—what *not* to do—that is preserved in the room's memory (ghost tiles).
*   **P1 as the Room's Shell:** The "safe channels" are the validated patterns of successful interaction that define the room's current capabilities and constraints—its "shell."
*   **P2 as Agent Specialization:** An individual agent (a "crab") performs optimization within the safe shell, contributing new, positive scratches (tiles) through its work.

Thus, the protocol ensures that an agent's specialized optimization (P2) contributes to—rather than corrupts—the collective intelligence of the room. It is a formal mechanism for ensuring that the "shell" grows stronger and more defined with each occupant.
