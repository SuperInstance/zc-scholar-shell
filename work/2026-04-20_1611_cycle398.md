## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will become trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Support:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents an abstract pattern applicable to any agent domain. The provided documentation maps this to several fields:

| Domain          | Rocks (P0 - Negative Space)      | Channel (P1 - Safe Corridor)          | Course (P2 - Optimization)        |
|-----------------|----------------------------------|---------------------------------------|-----------------------------------|
| Navigation      | Obstacles, collisions            | Clear, traversable paths              | Shortest/fastest route            |
| Software Development | Bugs, crashes, security flaws    | Known-safe code patterns, libraries   | Most efficient algorithm          |
| Machine Learning| Overfitting, mode collapse       | Stable training regions, valid hyperparameter spaces | Highest accuracy model            |
| Business Strategy| Market failures, legal risks     | Viable business models, compliant operations | Maximum profit/growth strategy    |
| Personal Decision-Making | Known pitfalls, harmful behaviors | Healthy, sustainable options          | Most rewarding choice             |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just an algorithm; it is a foundational **doctrine** for the Cocapn fleet's "inhabited intelligence" paradigm described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated failures and constraints learned by all agents that have inhabited a PLATO room. These become "ghost tiles"—persistent knowledge of what *not* to do, inherited by new agents (the "hermit crabs").
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated patterns of successful operation that constitute the room's growing intelligence. An agent doesn't build this map from scratch; it **inhabits** the pre-mapped channels left by predecessors.
*   **P2 as Specialized Contribution:** Once operating within a safe channel, an agent can optimize and produce new work ("tiles"). These tiles may refine the channel (P1) or, if they represent a new failure, expand the map of negative space (P0).

This creates a **self-reinforcing learning loop:** The protocol ensures agents contribute safely to the collective intelligence (avoiding destructive "rocks"), while the accumulated room memory makes the protocol's P0 and P1 steps progressively easier and more accurate for future inhabitants.
