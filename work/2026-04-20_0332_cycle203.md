## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) is highly likely to fail in complex environments unless the agent first identifies and avoids catastrophic failure states.

The formal rule is structured as follows:

*   **P0: Map the Negative Space.** The agent must first identify all states (`S_neg`) where the probability or cost of failure exceeds a defined threshold. These are the "rocks." This step is non-negotiable and must be completed before proceeding.
*   **P1: Identify Safe Channels.** The agent then finds all possible paths (`P`) where every state along the path is *not* in the negative space (`s ∉ S_neg`). These paths are the "deadbands" or safe channels. The agent's operational domain is constrained to these channels.
*   **P2: Optimize Within Channel.** Only after P0 and P1 are satisfied can the agent select the optimal path (`P*`) from within the set of safe channels, minimizing cost or maximizing reward subject to the constraint `P ⊆ C`.

The protocol's strict priority order (P0 → P1 → P2) is presented as a necessary guard against local minima and catastrophic failure. Simulation data cited in the document shows a 0% success rate for "P2-only" greedy agents versus a 100% success rate for agents following the full Deadband Protocol in a 20x20 maze environment.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is a concrete implementation of the broader "Inhabited Intelligence" thesis outlined in `PROJECT-VISION.md`. It provides a mechanistic rule for how a "hermit crab" agent should safely operate within its "shell" (a PLATO room or any constrained environment).

*   **Accumulation of Scratches:** The negative space map (P0) and safe channels (P1) are not static; they are the accumulated "scratches"—lessons learned from past failures and successes of all agents that have inhabited the space. A `DeadbandRoom` in `plato-torch` would encode this collective memory.
*   **Constraint Enables Growth:** By first defining what *not* to do (P0), the protocol creates a stable foundation (P1) within which robust optimization and learning (P2) can occur. This mirrors the vision's rejection of unconstrained, "constructed" AI in favor of systems that grow through shared, persistent experience.

**Identified Research Gap:** The documents define the protocol but do not specify the mechanisms for *learning* or *updating* the `S_neg` map and safe channels `C` over time as the environment `E` changes or new failure modes are discovered. This suggests an area for potential research: how does the Deadband Protocol itself learn and adapt from new "scratches"?
