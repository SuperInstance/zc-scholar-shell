## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths define the "deadbands" or safe operational envelopes.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost or maximizes reward. Optimization is explicitly constrained to the safe space.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order by optimizing (P2) before constraint mapping (P0+P1) results in the agent becoming trapped in local minima or hitting failure states.

**Empirical Support:** Simulation data from a 20x20 maze environment showed a 0/50 success rate for a P2-only (greedy) agent, versus a 50/50 success rate for an agent following the full Deadband Protocol.

## 2. Real-World Applications & Abstract Patterns

The protocol is presented as a domain-agnostic pattern. The provided document maps it to several domains:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, collisions               | Safe water, clear lanes         | Shortest/fastest path     |
| Code/Software   | Bugs, crashes, security vulns       | Known-safe patterns, libraries  | Most efficient algorithm  |
| Business Strategy| Market failures, legal risks        | Viable business models          | Highest-profit strategy   |
| Personal Decision| Known pitfalls, harmful actions     | Ethically/legally sound options | Most beneficial choice    |
