## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid states that lead to failure before attempting to optimize for a goal.

**Formal Rule (v1.0):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or magnitude of failure exceeds a defined threshold. These are "the rocks." This step is mandatory.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state along the path is *not* in `S_neg`. These paths are "deadbands" or "safe channels." The agent is constrained to operate only within these.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost (e.g., time, energy) while remaining entirely within `C`.

**Strict Priority:** The steps must be executed in order (P0 → P1 → P2). Violating this order—specifically, optimizing (P2) before completing the mapping steps (P0+P1)—is shown in simulation to lead to entrapment in local minima with high probability as environmental complexity increases.

**Empirical Proof:** The provided simulation data (a 20x20 maze, 50 runs) showed:
*   **P2-only (Greedy Optimization):** 0/50 successful completions. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband Protocol):** 50/50 successful completions at optimal speed.

The protocol provides a formal, actionable method for the fleet to convert the raw, accumulated experience in its rooms into a structured, reusable safety framework, ensuring that each new "hermit crab" agent doesn't have to learn the rocks from scratch.
