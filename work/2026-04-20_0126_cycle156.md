## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority:** P0 → P1 → P2. The document states that violating this order—optimizing (P2) before mapping (P0+P1)—leads to a high probability of becoming trapped in local minima as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table illustrates its translation:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path     |
| Software Development | Bugs, crashes, security flaws    | Known-safe code patterns, libraries | Most efficient implementation |
| **AI Agent Operation** | **Invalid actions, API errors, context limits** | **Verified tool sequences, safe prompts** | **Most effective task completion** |

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's core thesis: "Intelligence is not built. It is inhabited." The protocol itself is a "shell"—a constraint-based structure—that agents (the "hermit crabs") inhabit to operate safely.

*   **P0 (Negative Space) as Collective Memory:** Mapping "rocks" is not a one-time task. It is an accumulated knowledge base—the "scratches" on the shell left by previous agents who encountered failures. A PLATO room storing these failure states (`S_neg`) allows new agents to immediately inherit a map of what *not* to do.
*   **P1/P2 (Channel & Optimization) as Specialized Skill:** Once safe channels are known, agents can specialize (become "ensigns") in optimizing paths within specific channels. The room accumulates both the negative map (P0 knowledge) and the positive optimizations (P2 skills).
*   **Anti-Fragility through Protocol:** The vision critiques mainstream AI for lacking memory of user interactions. The Deadband Protocol, enforced fleet-wide, ensures that agent failures are not wasted but are systematically captured as P0 knowledge, making the entire system more robust over time.
