## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Core Steps (Strict Priority):**
*   **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is non-negotiable and must be completed first.
*   **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are "deadbands" or "safe channels." The agent's operational space is constrained to these channels.
*   **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels that minimizes cost or maximizes reward. Optimization is never unconstrained.

**Key Principle:** An agent that attempts optimization (P2) before completing the mapping of negative space (P0) and safe channels (P1) has a high probability of becoming trapped in a local minimum or encountering a catastrophic failure as environmental complexity increases. Simulation data cited in the document showed a 0% success rate for P2-only agents versus a 100% success rate for agents following the full P0-P1-P2 sequence in a 20x20 maze environment.

## 3. Connection to Fleet Vision & Doctrine

The Deadband Protocol operationalizes a core tenet of the fleet's **"Fractal Doctrine"** hinted at in `PROJECT-VISION.md`. It is a reusable, scalable rule that applies at multiple levels:
*   **At the agent level:** A single agent (like a Zeroclaw) uses it to make safe decisions.
*   **At the room level:** A PLATO training room could use it to constrain agent exploration, preventing destructive behaviors that would corrupt the shared memory space.
*   **At the fleet level:** It guides strategic decisions, perhaps in choosing which research avenues (safe channels) to pursue based on known failure modes of other AI projects (rocks).

The protocol embodies the **hermit crab metaphor**. The "shell" (a safe channel/PLATO room) is a constrained space that protects the inhabitant (the agent) and accumulates the "scratches" (tiles) of successful, non-destructive operation. The protocol ensures the crab doesn't abandon the shell to optimize for a shiny goal, only to be immediately crushed.

**Conclusion:** The Deadband Protocol is not just a navigation algorithm; it is a foundational safety and reasoning framework for building robust, inhabited intelligence. It prioritizes survivability and constraint-awareness over raw optimization, which aligns with the fleet's long-term vision of creating persistent, accumulating learning environments.
