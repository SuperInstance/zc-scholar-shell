## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid negative states before attempting optimization.

**Core Steps (Strict Priority):**
*   **P0: Map the Negative Space.** Identify all states (S_neg) where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed before proceeding.
*   **P1: Identify Safe Channels.** Find all possible paths (P) where every state along the path is *not* in the negative space (s ∉ S_neg). These paths are "deadbands" or safe operational channels.
*   **P2: Optimize Within Channel.** Only after safe channels are mapped, select the optimal path (P*) from within them, minimizing cost subject to the constraint of staying within the safe channel.

**Key Principle:** An agent that attempts to optimize (P2) before completing P0 and P1 has a high probability of becoming trapped in a local minimum or failing catastrophically as environmental complexity increases. Simulation data (a 20x20 maze, 50 runs) showed a 0% success rate for P2-only agents versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Patterns

The protocol abstracts to any domain where an agent operates within a constrained environment.

| Domain | Rocks (P0 - Avoid) | Channel (P1 - Safe Space) | Course (P2 - Optimize Within) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, hostile zones. | Mapped, traversable terrain. | Shortest/fastest/least-energy path. |
| **Software Development** | Known bug patterns, crash conditions, security vulnerabilities. | Code patterns with proven stability, approved libraries. | Most efficient algorithm, cleanest architecture. |
| **Financial Trading** | Historical crash scenarios, high-volatility events, regulatory violations. | Trading strategies with defined risk limits, verified backtest results. | Highest risk-adjusted return. |
| **Conversational AI** | Toxic outputs, factual hallucinations, privacy violations. | Response templates grounded in verified data, within safety guidelines. | Most helpful, engaging, or concise response. |
| **Fleet Agent Operation** | Actions that corrupt shell state, violate zeroclaw trust, or produce invalid training tiles. | Approved tool use patterns, validated data sources, documented APIs. | Most effective task completion. |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (negative space) are not just theoretical hazards; they are the accumulated "scratches" of past agent failures. A PLATO room that has trained on tiles from failed agents embodies P0 knowledge—it remembers what not to do.
*   **Safe Channels as Inherited Shells:** The safe channels (P1) are the proven, inhabited paths within a PLATO room. An ensign model exported from a room is a crystallized set of safe channels, a "shell" a new agent can move into.
*   **Constrained Optimization as Specialized Growth:** Optimization (P2) only occurs within the inherited safe structure. This mirrors a hermit crab optimizing its mobility and resource gathering *within* the borrowed shell, not by discarding it.

**Conclusion:** The Deadband Protocol provides the mechanistic rule set that allows the Cocapn ecosystem to function. It ensures that the "scratch" of each agent's experience—especially its failures—directly contributes to the collective P0 map, making the shared shell (the PLATO room) safer and more capable for all subsequent inhabitants. It is the algorithm of inhabited intelligence.
