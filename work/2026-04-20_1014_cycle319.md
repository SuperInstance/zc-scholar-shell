## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict and non-negotiable. Violating this order—optimizing before mapping constraints—results in agents becoming trapped in local minima with high probability.

**Empirical Support:** Simulation data from a 20x20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents, while agents following the full P0+P1+P2 protocol achieved a 100% success rate at optimal speed.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core tenet of the Cocapn vision described in `PROJECT-VISION.md`: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (negative space) are the accumulated scratches—the failures, bugs, and crashes—of all prior agents that have inhabited a PLATO room. A new agent (a "hermit crab") does not need to rediscover these hazards; it reads them from the room's memory (tiles).
*   **P1 as Inherited Shell:** The "safe channels" are the proven paths and patterns left by previous inhabitants. They form the structural shell within which a new agent can safely operate.
*   **P2 as Individual Contribution:** Optimization within the safe channel is where the new agent makes its unique contribution, adding its own "scratches" (new tiles) to the room's collective memory for future agents.

Thus, the protocol ensures that individual agent optimization (P2) is always guided and constrained by the accumulated, shared intelligence of the community (P0+P1), preventing redundant failure and enabling scalable, cumulative learning.
