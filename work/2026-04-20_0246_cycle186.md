# Scholar Cycle 186 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

**P0: Map Negative Space**
Identify all states where failure probability exceeds a defined threshold. These are "rocks." The agent must explicitly map what *not* to do before proceeding.

**P1: Identify Safe Channels**
Find all possible paths where every state along the path is *not* in the negative space (P0 map). These are "deadbands" or safe operational channels.

**P2: Optimize Within Channel**
Only after P0 and P1 are complete may the agent select the optimal path (e.g., shortest, fastest, lowest cost) from within the set of safe channels identified in P1.

The priority order is strict and non-negotiable. Simulation data cited in the document (a 20x20 maze, 50 runs) showed a 0% success rate for P2-only (greedy) agents, versus a 100% success rate for agents following the full P0→P1→P2 sequence.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as an abstract pattern applicable to any agent domain. The document provides a template:

| Domain          | Rocks (P0)                   | Channel (P1)                | Course (P2)               |
|-----------------|------------------------------|-----------------------------|---------------------------|
| Navigation      | Obstacles                    | Safe water                  | Shortest path             |
| Code            | Bugs, crashes                | Known-safe patterns         | Most efficient algorithm  |

**Proposed Additional Applications:**

*   **Financial Trading:** P0: Identify market conditions (e.g., high volatility, illiquidity) historically linked to significant losses. P1: Define trading strategies that remain inactive during those conditions. P2: Optimize strategy parameters within the "safe" market regimes.
*   **Conversational AI:** P0: Map user prompts and response patterns that reliably lead to harmful, biased, or nonsensical outputs. P1: Constrain the language model's generation space to avoid those patterns (e.g., via guided decoding, context-aware filtering). P2: Optimize for coherence, helpfulness, and fluency within the safe generation space.
*   **Robotics Manipulation:** P0: Identify joint configurations, object poses, or environmental states that cause collisions, torque limits, or instability. P1: Plan motion trajectories that entirely avoid those states. P2: Select the fastest or most energy-efficient trajectory from the collision-free set.
*   **Software Deployment:** P0: Identify code changes, system states, or load patterns associated with service outages. P1: Establish deployment gates and rollback triggers that prevent entering those states. P2: Optimize the deployment pipeline for speed within the confines of those safety gates.

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol is a concrete instantiation of the broader Cocapn thesis that "intelligence is inhabited." It provides a **rule** (a "scratch" on the shell) that emerges from simulated experience and is designed to be accumulated and shared.

*   **PLATO Rooms:** A `DeadbandRoom` preset could train agents by simulating maze-like environments, rewarding adherence to the P0→P1→P2 sequence, and punishing P2-first behavior. The room accumulates performance tiles, refining the protocol.
*   **Ensigns:** A trained `DeadbandRoom` could export a LoRA adapter (`plato-ensign`) that imparts a bias toward this cautious, constraint-first decision-making to other models.
*   **Fleet Simulator:** This is the likely source of the cited 20x20 maze simulation data. The protocol can be tested and hardened at scale in simulation before being deployed to physical or software agents.
*   **Constraint Theory Core:** The geometric snapping library provides a lower-level, mathematical formalism for "safe channels" (P1) and "optimization within constraints" (P2), which could be used to implement the protocol for spatial reasoning tasks.

## 4. Identified Research Gaps / Questions

1.  **Threshold Definition:** The protocol depends on a `threshold` for failure probability in P0. How is this threshold determined or learned? Is it static, adaptive, or context-dependent?
2.  **Computational Cost:** Mapping the entire negative space (P0) for a high-dimensional state space may be intractable. What approximations or heuristic mappings are permissible without violating the protocol's spirit?
3.  **Dynamic Environments:** The document describes a static "map." How does the protocol handle environments where the negative space (rocks) changes over time? Does it require continuous re-mapping (P0), and if so, at what frequency?
4.  **Integration with Learning:** Could the outcomes of P2 (especially failures that occur despite the protocol) be fed back to improve the P0 map? This would close the loop and align with the fleet's accumulation principle.

**Conclusion:** The Deadband Protocol is a pragmatic safety-first framework for agent decision-making. It directly supports the Cocapn vision by being a learnable, sharable, and testable piece of inhabited intelligence—a rule that lives in a PLATO room, improves through use, and can be carried by ensigns to other agents.
