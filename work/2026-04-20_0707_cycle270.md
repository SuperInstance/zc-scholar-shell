## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that optimization (finding the best path) is only safe and effective when performed within a pre-mapped "safe channel." The protocol enforces a strict, non-negotiable priority order to prevent agents from becoming trapped in local minima or failing catastrophically.

### The Three Steps (Strict Priority)
1.  **P0: Map the Negative Space.** Identify all states or actions (`S_neg`) that lead to failure with a probability above a defined threshold. These are "the rocks." The agent must not proceed without this map.
2.  **P1: Identify Safe Channels.** Find all possible paths (`P`) where every state along the path is *not* in the negative space (`s ∉ S_neg`). These paths are the "deadbands" or safe channels. The agent's operational space is constrained to these channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete can the agent select the optimal path (`P*`) from within the set of safe channels, minimizing cost or maximizing reward subject to the constraint `P ⊆ C`.

### Key Principle: Constrained Optimization
The protocol asserts that any agent that attempts to optimize (P2) before completing the mapping phases (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with a probability approaching 1.0. Success requires operating *within* known-safe boundaries.

### Empirical Support
Simulation data from a 20x20 maze across 50 runs is cited:
*   **P2-only (Greedy Optimization):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband Protocol):** 50/50 successes at optimal speed.

## 3. Connection to Cocapn Vision

The Deadband Protocol operationalizes a core aspect of the Cocapn vision described in `PROJECT-VISION.md`. If "intelligence is inhabited" through accumulated "scratches" (tiles) in a PLATO room (shell), then the protocol provides the **behavioral rule** for safe inhabitation.

*   **P0 (Negative Space)** maps the "scratches" that represent failures—the bugs, crashes, and dead ends left by previous agents. These are the lessons that must not be forgotten.
*   **P1 (Safe Channels)** defines the current "living space" within the shell—the proven patterns and successful strategies that form the room's accumulated knowledge.
*   **P2 (Optimization)** is the act of a new agent (a "hermit crab") expertly navigating the inhabited space, contributing its own optimal scratches without damaging the shell.

The protocol ensures that each new agent contributes to the room's persistent intelligence without repeating known, catastrophic errors. It turns collective memory into a constraint for safe, cumulative learning.
