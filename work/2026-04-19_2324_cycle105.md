## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure in complex environments by enforcing a sequence of constraint identification before optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed before proceeding.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These permissible paths are the "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost. Optimization is explicitly constrained to the safe space.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with a probability approaching 1.0.

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment. In 50 runs, a "P2-only" (greedy optimization) agent succeeded 0 times, while the full Deadband Protocol agent succeeded 50 times at optimal speed.
