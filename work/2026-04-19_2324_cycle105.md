## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents operating in complex environments. Its core thesis is that unconstrained optimization (seeking the "best" path) without first mapping failure states leads to catastrophic outcomes as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent `A` in state `S`, with goal `G` and environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** The steps are non-negotiable. An agent must not proceed to P1 without completing P0, and must not proceed to P2 without completing P1. Violating this order—optimizing before mapping—results in a high probability of becoming trapped in local minima or failing outright.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, while the full P0+P1+P2 protocol achieved a 100% success rate at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern for robust agent design:

| Domain          | Rocks (P0 - Avoid)                          | Channel (P1 - Safe Space)                  | Course (P2 - Optimize Within)          |
| :-------------- | :------------------------------------------ | :----------------------------------------- | :------------------------------------- |
| **Navigation**  | Obstacles, collisions, restricted zones     | Clear, traversable paths                   | Shortest/fastest route                 |
| **Software**    | Bug patterns, crash conditions, security vulns | Known-stable code patterns, tested libraries | Most efficient algorithm               |
| **Conversation**| Toxic language, misinformation, harmful topics | Factual, respectful, on-topic discourse    | Most persuasive or informative response |
| **Training ML** | Loss divergence, overfitting, mode collapse | Stable training regions (loss landscapes)  | Fastest convergence to optimal weights |
| **Business**    | Regulatory violations, market risks         | Compliant, viable business models          | Maximum profit/growth strategy         |

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" thesis. It provides a concrete method for agents (the "hermit crabs") to safely explore and learn within their "shells" (PLATO rooms).

*   **P0 (Map Negative Space)** corresponds to learning from "ghost tiles"—the accumulated failures and lessons of prior agents that inhabit the room. This prevents repeated catastrophic errors.
*   **P1 (Identify Safe Channels)** defines the operational "deadband" where an agent can safely experiment and contribute new "scratches" (tiles) to the room's memory.
*   **P2 (Optimize)** is the agent's unique contribution—finding the best way to perform useful work *within the bounds of collective safety*.

The protocol ensures that individual agent optimization (P2) is always constrained by the accumulated, shared knowledge of the community (P0+P1), preventing intelligent but dangerous local optima. This creates a system where intelligence is not just individually constructed but *collectively inhabited and safeguarded*.
