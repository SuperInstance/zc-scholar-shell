## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent `A` in state `S`, with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg` (`s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent that optimizes (P2) before completing the mapping steps (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with a probability approaching 1.0.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol abstracts to any domain where an agent operates within constraints. The provided table can be extended:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, cliffs, hostile zones | Safe water, roads, friendly airspace | Shortest/fastest route |
| **Software Development** | Known bugs, crash conditions, security vulns | Compiler-validated patterns, passing tests, approved libraries | Most efficient algorithm |
| **Financial Trading** | Regulatory violations, insolvency, flash crashes | Compliant strategies, within risk limits, liquid markets | Maximum risk-adjusted return |
| **Conversational AI** | Toxic output, factual hallucinations, prompt leaks | Guardrail-filtered responses, cited sources, privacy-safe content | Most helpful/engaging answer |
| **Fleet Doctrine (Meta)** | Wasted cycles, incoherent output, violating core rules | Validated task channels, aligned with project vision | Most valuable tile output |

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a core principle implied in `PROJECT-VISION.md`: **intelligence is inhabited through safe, persistent accumulation.**

*   **Shell as Safe Channel:** A PLATO room functions as a "safe channel" (P1). It is a bounded space where agent interactions (tiles) can accumulate without hitting fundamental "rocks" like data loss or incoherence.
*   **Tiles as P2 Optimization:** The useful work agents do—writing summaries, code, analyses—is the P2 optimization *within* the safe channel of the room's rules and persistence layer.
*   **Ghost Tiles as P0 Knowledge:** The "scratches" left by previous agents, especially lessons from failures ("ghost tiles"), directly constitute the mapped "negative space" (P0) for future inhabitants. The room remembers what not to do.

Thus, the Deadband Protocol provides the specific, procedural mechanism that allows the "hermit crab" intelligence model to function reliably. It ensures crabs (agents) systematically avoid breaking their shells (rooms) so that scratches (tiles) can accumulate into inhabited intelligence.
