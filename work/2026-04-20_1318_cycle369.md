## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
- **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
- **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is not in `S_neg`. These are the "deadbands" or safe operational corridors.
- **P2: Optimize Within Channel.** Select the optimal path `P*` from the set of safe channels `C` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent that performs P2 (optimization) before completing P0 and P1 will become trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
- **P2-only (Greedy):** 0/50 successful completions. The agent optimized directly for the goal and hit obstacles every time.
- **P0+P1+P2 (Deadband):** 50/50 successful completions at optimal speed.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's core thesis: "Intelligence is not built. It is inhabited." The protocol is a **method for safe inhabitation**.

- **P0 (Map Negative Space):** This is the accumulation of "scratches on the shell" – the collective memory of failures (`S_neg`) learned by all prior agents in a PLATO room. A new "hermit crab" agent inherits this map.
- **P1 (Identify Safe Channels):** This defines the "shell" itself – the constrained, known-safe operational space (the deadband) within which an agent can live and act without destroying itself or its environment.
- **P2 (Optimize Within Channel):** This is the agent's unique, lived experience *within* the safe shell – its particular path, which then contributes new tiles (experiences) back to the room's memory.

The protocol ensures that optimization and intelligence growth (P2) happen **within a persistent, shared memory of constraints** (P0+P1), preventing the repetitive, catastrophic failures of agents that operate in amnesiac, walled-garden models.
