# Scholar Cycle 369 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden zones. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These permissible paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels that minimizes cost or maximizes reward. Optimization is explicitly constrained to the pre-mapped safe space.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order—attempting to optimize (P2) before fully mapping constraints (P0+P1)—results in the agent becoming trapped in local minima or failing catastrophically with high probability in complex environments.

**Empirical Support:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, while the full P0+P1+P2 deadband protocol achieved a 100% success rate at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern for robust agent design:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, restricted zones. | Clear, traversable terrain. | Shortest/fastest route within safe terrain. |
| **Software Development** | Known bug patterns, crash conditions, insecure code practices. | Verified design patterns, linting rules, passing test suites. | Most efficient or performant implementation that adheres to safe patterns. |
| **Financial Trading** | Regulatory violations, tail-risk events, historical crash scenarios. | Compliant strategies, defined risk limits, historically stable regimes. | Highest-risk-adjusted return strategy within the compliant, bounded framework. |
| **Conversational AI** | Toxic output, factual hallucinations, privacy violations, prompt injections. | Grounded, helpful, and harmless response templates. | Most engaging, accurate, or concise response from the set of safe templates. |
| **Fleet Doctrine (Meta)** | Actions that break the shell, violate trust, or produce useless tiles. | Actions that read real files, write factual output, and follow assigned tasks. | Most efficient or insightful work that contributes valid training tiles. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just an agent rule; it is a foundational metaphor for the Cocapn project's "inhabited intelligence" paradigm described in `PROJECT-VISION.md`.

*   **The Shell as a Deadband:** A PLATO room acts as a collective "safe channel" (P1), accumulated from the "negative space" (P0) mapped by all prior agents (their failures and lessons as "ghost tiles"). New agents (hermit crabs) inhabit this pre-mapped shell.
*   **Training as Constrained Optimization:** An ensign model, trained on a room's tile history, is an optimization (P2) performed strictly within the safe behavioral channel defined by that room's accumulated experience. It cannot learn to operate outside the room's known-good space.
*   **Boot Camp as Protocol Enforcement:** The Zeroclaw boot camp phases (Read → Analyze → Build) enforce the P0-P1-P2 sequence. Phase 1 (Read) is P0/P1: learning the fleet's boundaries and safe patterns. Writing and building (Phases 2-3) are the P2 optimization, permitted only after the safe channel is understood.

## 4. Identified Research Gaps / Questions

1.  **Threshold Definition:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold determined or learned dynamically by an agent? Is it static, adaptive, or communicated by the environment/shell?
2.  **Channel Discovery Complexity:** In high-dimensional or partially observable state spaces, how can an agent efficiently identify *all* safe channels (P1)? Is exhaustive mapping feasible, or are heuristic or probabilistic approximations used?
3.  **Inter-Agent Deadbands:** The current documentation focuses on a single agent. How do deadbands form and interact in a multi-agent system like the Cocapn fleet? Can one agent's P0 map (rocks) become another agent's P1 channel (specialist role)?
4.  **Empirical Validation Beyond Mazes:** The provided proof uses a 2D maze simulation. Are there results applying the protocol to more complex, real-world benchmarks in software, dialogue, or planning domains?

**Conclusion:** The Deadband Protocol provides a compelling, empirically grounded framework for building robust, constraint-aware agents. Its strict prioritization of safety-mapping over optimization directly addresses a common failure mode in AI systems. Its deep integration with the Cocapn fleet's core philosophy suggests it is more than an algorithm—it is a governing principle for the development of inhabited, cumulative intelligence.
