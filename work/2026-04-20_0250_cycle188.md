## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule (v1.0):**
- **P0: Map Negative Space** – Identify all states `S_neg` where failure probability exceeds a threshold. These are "rocks."
- **P1: Identify Safe Channels** – Find all paths `P` where every state `s ∈ P` is not in `S_neg`. These are "deadbands" or safe operational channels.
- **P2: Optimize Within Channel** – Select the optimal path `P*` from within the safe channels, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will be trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Proof:** Simulation data from a 20×20 maze with 50 runs showed:
- **P2-only (greedy optimization):** 0/50 success. Agents optimized directly for the goal and hit obstacles every time.
- **P0+P1+P2 (deadband protocol):** 50/50 success at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table illustrates this:

| Domain          | Rocks (P0)                    | Channel (P1)                     | Course (P2)               |
|-----------------|-------------------------------|----------------------------------|---------------------------|
| Navigation      | Obstacles, collisions         | Safe water, clear lanes          | Shortest path             |
| Code            | Bugs, crashes, security flaws | Known-safe patterns, tested APIs | Most efficient algorithm  |
| **AI Training** | **Catastrophic forgetting, harmful outputs** | **Stable learning regimes, safe data distributions** | **Maximize accuracy/reward** |

## 4. Identified Research Gaps & Questions

Based on the provided documents, several areas for further investigation emerge:

1.  **Threshold Definition:** The protocol mentions a failure threshold for defining `S_neg` but does not specify how this threshold is determined or adapted dynamically.
2.  **Channel Discovery Cost:** The computational cost of exhaustively mapping safe channels (P1) in high-dimensional state spaces (like LLM parameter space or complex simulation environments) is not addressed.
3.  **Dynamic Environments:** The protocol, as described, seems best suited for static or slowly changing environments. Mechanisms for updating the P0 map and P1 channels in real-time are a logical extension.
4.  **Integration with PLATO:** How is the Deadband Protocol instantiated within a `DeadbandRoom` preset in `plato-torch`? Does the room's training process explicitly generate tiles for P0 (failure states) and P1 (safe channels), or is it an emergent property of agent interaction?

**Conclusion:** The Deadband Protocol is a compelling framework for robust agent design, emphasizing constraint identification over premature optimization. Its integration into the Cocapn fleet's architecture suggests a research direction focused on *constrained intelligence*—systems whose capabilities are built upon a foundational map of their own limitations.
