## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid hazardous states before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are "deadbands" or safe channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** The steps are non-negotiable. An agent must not proceed to P1 without completing P0, and must not proceed to P2 without completing P1. The document states that violating this order (e.g., optimizing before mapping) leads to a high probability of becoming trapped in local minima as environmental complexity increases.

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment. In 50 runs, a "P2-only" greedy optimizer had a 0% success rate, while the full P0+P1+P2 deadband protocol achieved a 100% success rate at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table illustrates this:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path     |
| Software Development | Bugs, crashes, security flaws | Known-safe patterns, libraries  | Most efficient algorithm  |

**Extended Applications (Inferred from Fleet Context):**

*   **AI Agent Safety:** Before an agent takes an action (P2), it must first map actions known to cause system crashes, resource exhaustion, or safety violations (P0), then identify action sequences that avoid those states (P1).
*   **Machine Learning Training:** Before optimizing a model for accuracy (P2), identify hyperparameter configurations that lead to divergence or overfitting (P0), then restrict the search space to stable regions (P1).
*   **System Architecture (e.g., Cocapn Fleet):** Before connecting services for performance (P2), identify incompatible versions or unstable API combinations (P0), then design the integration using only verified, compatible components (P1).
*   **Research Synthesis (Scholar's Domain):** Before forming a novel conclusion or connection (P2), identify logical fallacies, unsupported claims, or contradictory evidence in the source material (P0), then build an argument using only the well-supported, consistent information (P1).

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a key implicit principle in the `PROJECT-VISION.md`: **intelligence as safe inhabitation, not unchecked construction.**

The vision describes PLATO rooms as "shells" that accumulate "scratches" (tiles) from every agent (hermit crab) that inhabits them. The Deadband Protocol provides a method for how those agents should *inhabit* the shell. It is a rule for moving through the environment without breaking it—a way to leave useful scratches (successful work tiles) instead of destructive ones (crashes, failures).

The protocol's P0 ("Map the Negative Space") aligns with the concept of "ghost tiles"—the persistent lessons from dead agents. These ghost tiles are a collective map of `S_neg` for the entire fleet. By reading the room's memory (the accumulated tiles), a new agent can immediately know what *not* to do, fulfilling its P0 obligation before it even begins its own P1 exploration.

**Conclusion:** The Deadband Protocol is not just an agent control policy; it is a foundational safety and efficiency layer for the Cocapn ecosystem. It ensures that the iterative, inhabited growth of intelligence within PLATO rooms is constrained to safe channels, preventing the system from optimizing itself into failure. It turns the "shell" into a navigable space with known boundaries.
