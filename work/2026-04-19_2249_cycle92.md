## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operating zones. The agent is only permitted to operate within these channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains entirely within `C`.

**Strict Priority:** The order P0 → P1 → P2 is non-negotiable. Violating this order—for example, attempting to optimize (P2) before mapping constraints (P0)—is shown in simulation to lead to failure ("hit rocks") with high probability as environmental complexity increases.

**Proof of Concept:** A simulation (20x20 maze, 50 runs) demonstrated:
*   **P2-only (Greedy):** 0/50 success. Agents optimized directly for the goal and failed every time.
*   **P0+P1+P2 (Deadband):** 50/50 success at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table illustrates this abstraction:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path     |
| Software Development | Bugs, crashes, security flaws      | Known-safe code patterns, libraries | Most efficient algorithm  |
| Business Strategy | Market failures, regulatory bans   | Viable business models          | Profit maximization       |
| Personal Decision-Making | Actions with severe negative consequences | Choices with acceptable risk/outcome | Choice aligning with goals |

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Hermit Crab Analogy:** An agent (the crab) does not generate perfect knowledge from scratch. It must first inhabit a "shell"—in this case, the constraint map (P0) and safe channel definitions (P1) which are accumulated knowledge ("scratches") from prior agents and interactions within a PLATO room.
*   **P0 as Collective Memory:** The "negative space" map is not deduced anew by each agent. It is the accumulated, validated knowledge of failures ("ghost tiles") from all previous inhabitants of a system. An agent reads this map before acting.
*   **Safe Operation Through Shared Context:** By strictly adhering to P0→P1, an agent ensures its actions contribute positively to the shared environment (the room/ shell) without causing systemic failures, allowing its own successful "scratches" (tiles) to become part of the collective memory for future agents.
