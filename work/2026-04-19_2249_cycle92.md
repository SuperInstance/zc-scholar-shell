## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents operating in complex environments. Its core thesis is that unconstrained optimization (jumping directly to "best path") leads to failure as complexity increases. The protocol enforces a strict sequence:

**P0: Map the Negative Space**
Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." The agent must explicitly map what *not* to do before any other planning.

**P1: Identify Safe Channels**
Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These are the "deadbands" or safe operational channels. The agent's action space is constrained to these channels.

**P2: Optimize Within Channel**
Only after P0 and P1 are complete can the agent select an optimal path `P*` from within the safe channels, minimizing cost or maximizing reward subject to the constraint `P ⊆ C` (path must remain within safe channels).

The provided simulation data from a 20x20 maze showed a 0% success rate for agents using greedy (P2-only) optimization across 50 runs, while agents following the full P0-P1-P2 sequence achieved a 100% success rate.

## 2. Abstract Pattern & Real-World Applications

The protocol abstracts to any domain where an agent must achieve a goal within a bounded, failure-prone environment.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|------------|--------------|-------------|
| **Navigation** | Obstacles, collisions, restricted zones | Clear lanes, safe water, permissible airspace | Shortest/fastest route within safe lanes |
| **Software Development** | Known bugs, crash conditions, security vulnerabilities | Code patterns with proven stability, approved libraries | Most efficient implementation using safe patterns |
| **Financial Trading** | Regulatory violations, extreme drawdown scenarios, black swan exposures | Trading strategies with defined risk limits, compliant instruments | Strategy with highest risk-adjusted return within limits |
| **Clinical Decision Support** | Contraindicated treatments, harmful drug interactions, protocol violations | Treatment pathways with established safety profiles | Most effective therapeutic option within safe pathways |
| **Conversational AI** | Toxic outputs, factually incorrect statements, privacy violations | Response templates that are verified safe, grounded in context | Most engaging, helpful, or concise response from the safe set |

## 3. Connection to Cocapn Fleet Architecture

The protocol is not just a planning algorithm; it is a foundational doctrine for the fleet's "inhabited intelligence" vision described in `PROJECT-VISION.md`.

*   **PLATO Rooms as Accumulated Channels:** A PLATO room accumulates "tiles" (interactions, code, lessons) from all agents that inhabit it. This collective memory effectively defines the **P1 safe channels** for future agents. The room's history is a map of what has worked.
*   **Ghost Tiles as P0 Knowledge:** "Ghost tiles" from failed agents or dead ends explicitly mark `S_neg`—the "rocks" to avoid. This persistent negative space prevents the fleet from repeatedly making the same mistakes.
*   **Training Ensigns:** An "ensign" (a LoRA adapter exported from a room) encapsulates the learned safe channels (P1) and optimization strategies (P2) specific to that room's domain, allowing new agents to bootstrap directly into constrained, effective operation.
*   **The Hermit Crab Metaphor:** An agent (the crab) does not build its intelligence from scratch. It inhabits a shell (a PLATO room) whose interior is shaped by prior occupants (scratches/tiles). Its behavior is inherently channeled by the shell's existing form—the deadband.
