## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space** — Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels** — Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel** — From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will become trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Support:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for P2-only (greedy) agents, versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided document maps it to two domains:

| Domain          | Rocks (P0)                   | Channel (P1)                     | Course (P2)               |
|-----------------|------------------------------|----------------------------------|---------------------------|
| Navigation      | Obstacles, collisions        | Safe water, clear lanes          | Shortest/optimal path     |
| Code            | Bugs, crashes, security flaws| Known-safe patterns, libraries   | Most efficient algorithm  |

**Extended Application Reasoning:**
The protocol's strength is its recursive application to hierarchical planning. For example:
*   **Business Strategy:** P0 (market failures, regulatory bans) → P1 (viable business models, compliant operations) → P2 (profit maximization).
*   **Personal Decision-Making:** P0 (careers with high burnout, locations with deal-breakers) → P1 (feasible options meeting core needs) → P2 (choosing the "best" among them).
*   **Machine Learning Training:** P0 (hyperparameter sets causing divergence/nan) → P1 (stable training regions) → P2 (finding the fastest converging setup within that region).
