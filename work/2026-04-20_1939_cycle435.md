## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that optimization (P2) is only safe and effective when performed within a pre-mapped "safe channel" (P1), which itself can only be identified after first mapping the "negative space" or failure states (P0).

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing cost or maximizing reward. Optimization is **constrained** to the safe space.

**Key Insight from Simulation:** The document cites a 20x20 maze simulation. Agents using a P2-only (greedy optimization) strategy succeeded 0 out of 50 times, consistently hitting obstacles. Agents following the full P0+P1+P2 protocol succeeded 50 out of 50 times while still achieving optimal speed. The conclusion is that skipping to P2 leads to entrapment in local minima with probability approaching 1.0 as environmental complexity increases.
