## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent agents from becoming trapped in local minima or failing catastrophically by enforcing a sequence of constraint identification before optimization.

### Formal Rule (v1.0)
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed before proceeding.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** P0 → P1 → P2. The documentation states that an agent which optimizes (P2) before completing mapping (P0+P1) will, with high probability, become trapped in local minima as environmental complexity increases.

### Supporting Evidence
The protocol is supported by simulation data from a 20x20 maze environment:
*   **P2-only (Greedy Optimization):** 0 successes in 50 runs. Agents optimized directly for the goal and collided with obstacles every time.
*   **P0+P1+P2 (Deadband Protocol):** 50 successes in 50 runs, while also achieving optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic pattern. The documentation provides the following mapping:

| Domain          | Rocks (P0)                   | Channel (P1)                     | Course (P2)               |
|-----------------|------------------------------|----------------------------------|---------------------------|
| Navigation      | Obstacles                    | Safe water                       | Shortest path             |
| Code            | Bugs, crashes                | Known-safe patterns              | Most efficient algorithm  |

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a core principle implied in `PROJECT-VISION.md`: the need for **safe, accumulative learning**. The vision describes intelligence as "inhabited" within persistent shells (PLATO rooms), where agents (hermit crabs) contribute tiles (scratches).

*   **P0 (Negative Space)** corresponds to identifying and remembering failure states—the "scratches" that indicate where the shell has been damaged. This is the accumulated memory of what *not* to do.
*   **P1 (Safe Channels)** defines the viable operating space for new agents entering the room, built upon the accumulated experience (tiles) of previous inhabitants.
*   **P2 (Optimization)** allows an agent to perform its specialized task *within* the bounds of collective safety, contributing new, optimized scratches without re-breaking the shell.

The protocol ensures that the "intelligence" of the room—its accumulated constraints—is respected by every new occupant, enabling scalable, stable collective learning without centralized control. It is a mechanistic implementation of the "hermit crab" paradigm.
