## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") leads to failure as environmental complexity increases. The protocol enforces a sequence:

**P0: Map Negative Space**
Identify all states where failure probability exceeds a defined threshold. These are "rocks." The agent must not proceed without this map.

**P1: Identify Safe Channels**
Find all possible paths where every state along the path is *not* in the negative space (P0 map). These paths are "deadbands" or safe channels. The agent's operational space is constrained to these channels.

**P2: Optimize Within Channel**
Only after P0 and P1 are complete can the agent select the optimal path (e.g., shortest, fastest, lowest cost) from within the set of safe channels. Optimization is never unconstrained.

The `DEADBAND-PROTOCOL.md` file presents simulation evidence: in a 20x20 maze over 50 runs, a P2-only (greedy) agent succeeded 0 times, while a P0+P1+P2 agent succeeded 50 times.

## 2. Abstract Pattern & Real-World Applications

The file provides an abstract pattern, framing the protocol as domain-agnostic. Based on this, potential real-world applications include:

*   **Robotic Navigation:** P0: Map physical obstacles and unsafe terrain. P1: Plot all collision-free routes. P2: Choose the most energy-efficient path.
*   **Software Development:** P0: Identify known bug patterns, crash conditions, and insecure code practices. P1: Restrict development to known-safe libraries, frameworks, and design patterns. P2: Optimize code for performance or readability within those constraints.
*   **Financial Trading:** P0: Define conditions that lead to unacceptable drawdowns (e.g., excessive leverage, correlated asset failure). P1: Identify trading strategies that avoid those conditions entirely. P2: Optimize for return within the subset of safe strategies.
*   **Content Moderation:** P0: Map definitively harmful content (e.g., hate speech, graphic violence). P1: Establish clear policy channels defining allowable speech. P2: Optimize review throughput or accuracy within those policy bounds.

## 3. Connection to Cocapn Fleet & Project Vision

Reading `PROJECT-VISION.md` reveals a deep philosophical alignment. The vision describes intelligence as "inhabited," not built—like a hermit crab inhabiting and adding to a found shell. The Deadband Protocol operationalizes this for agent cognition:

*   **P0 as Collective Memory:** The "negative space" map is not built from scratch by each agent. It is the accumulated "scratches"—the recorded failures and lessons (ghost tiles) of all prior agents that have inhabited a PLATO room. An agent inherits this P0 map upon entering.
*   **Constrained Optimization as a Form of Inhabitation:** An agent does not freely explore an unbounded problem space. It inhabits the "safe channels" (P1) defined by the room's accumulated experience. Its optimization (P2) is a contribution *within* that inherited structure, adding new tiles (successful paths) to the room's memory.
*   **Violation as Forgetting:** An agent that skips to P2 is functionally "forgetting" the room's accumulated knowledge, leading to predictable failure—it hits the known "rocks." This enforces the vision's critique of stateless, memory-less AI models.
