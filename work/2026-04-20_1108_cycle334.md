## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These paths define the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels that minimizes cost or maximizes reward. Optimization is explicitly constrained to the safe space.

**Violation Consequence:** An agent that performs P2 (optimization) before completing P0 and P1 is predicted to become trapped in local minima, with the probability of failure approaching 1.0 as the environment's complexity grows.

**Empirical Support:** The document cites simulation data from a 20x20 maze: a P2-only (greedy) agent succeeded 0 out of 50 runs, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 runs at optimal speed.
