## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will be trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents an abstract pattern applicable to any agent domain:

| Domain          | Rocks (P0)                          | Channel (P1)                                  | Course (P2)                          |
|-----------------|-------------------------------------|-----------------------------------------------|--------------------------------------|
| **Navigation**  | Obstacles, hazards                  | Clear, traversable paths                      | Shortest/fastest route               |
| **Software**    | Bugs, crashes, security vulns       | Known-safe code patterns, tested libraries    | Most efficient algorithm             |
| **Finance**     | Regulatory violations, tail risks   | Compliant, historically stable instruments    | Highest risk-adjusted return         |
| **Medicine**    | Contraindications, adverse events   | Evidence-based treatment protocols            | Most effective therapy for patient   |
| **Conversation**| Taboo topics, logical fallacies     | Factual, constructive dialogue space          | Most persuasive/clear argument       |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key principle of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (failure states) are not just theoretical; they are the accumulated "scratches" left by previous agents in a PLATO room. A new agent (the "hermit crab") inherits this negative space map by inhabiting the room.
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated paths preserved within the room's memory—the structure of the shell itself. An agent doesn't compute these from scratch; it moves into the pre-defined operational space.
*   **P2 as Individual Contribution:** Optimization within the safe channel is where the agent's unique work happens, creating new "tiles" (experiences) that are added back to the room's memory, potentially refining the map of rocks and channels for future inhabitants.

This creates a fractal learning loop: individual agents (P2 work) contribute to collective constraint knowledge (P0/P1 updates), which in turn enables more effective individual work.
