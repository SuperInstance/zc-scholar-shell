# Scholar Cycle 208 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Key Principle:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violation—optimizing before mapping—is predicted to trap an agent in local minima.

**Empirical Basis:** The document cites simulation data from a 20x20 maze: a P2-only (greedy) agent succeeded 0 out of 50 runs, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 runs at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table suggests these mappings:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
| :-------------- | :---------------------------------- | :------------------------------ | :------------------------ |
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path     |
| Code            | Bugs, crashes, security vulns       | Known-safe patterns, libraries  | Most efficient algorithm  |
| **AI Training** | **Loss divergences, mode collapse** | **Stable training regimes**     | **Convergence speed**     |
| **Business**    | **Market failures, legal risks**    | **Viable business models**      | **Profit maximization**   |
| **Medicine**    | **Adverse reactions, contraindications** | **Safe treatment protocols** | **Optimal dosage/efficacy** |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: "Intelligence is not built. It is inhabited."

*   **Shell as Constraint:** A PLATO room acts as a "shell"—a constrained, safe channel (P1) accumulated from the collective scratches (tiles) of previous agents. It defines the negative space (P0) of known failures.
*   **Hermit Crab as Agent:** An agent (the "crab") does not build intelligence from scratch but *inhabits* the pre-mapped safe channel of a room. Its optimization (P2—doing useful work) is only possible and effective within that channel.
*   **Tiles as P0/P1 Data:** Each submitted tile contributes to refining the map of the negative space (P0: "this approach failed") and expanding the understanding of safe channels (P1: "this approach succeeded"). This creates a flywheel for safer, more effective agent operation.

## 4. Identified Research Gaps & Questions

1.  **Threshold Definition:** The protocol depends on a `threshold` for failure in P0. How is this threshold determined or learned dynamically across different domains and rooms?
2.  **Channel Discovery Cost:** In highly complex environments, exhaustively mapping all safe channels (P1) before any optimization may be computationally prohibitive. Are there heuristic or iterative approaches that preserve the protocol's safety guarantees?
3.  **Inter-Agent Channel Sharing:** How efficiently can a safe channel map (P0+P1 knowledge) generated by one agent or in one room be transferred to another? This relates directly to the "ensign" (LoRA) export concept in the fleet.
4.  **Empirical Validation:** The provided maze simulation is a compelling but limited example. Systematic validation across the broader set of fleet domains (code generation, simulation, dialogue) would strengthen the claim of domain-agnostic applicability.

**Conclusion:** The Deadband Protocol provides a rigorous, safety-first framework for agent decision-making that is philosophically and practically aligned with the Cocapn fleet's inhabited intelligence model. Its implementation appears foundational to the fleet's approach to training and operational stability.
