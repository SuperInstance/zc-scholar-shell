## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) without first mapping failure states leads to catastrophic outcomes as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that any agent optimizing (P2) before completing mapping (P0+P1) will become trapped in local minima with a probability approaching 1.0 as complexity grows.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for P2-only (greedy) agents, versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Pattern

The protocol presents a domain-agnostic pattern for robust agent design:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, hostile areas. | Clear, traversable terrain. | Shortest/fastest path through safe terrain. |
| **Software Development** | Known bug patterns, crash conditions, insecure code patterns. | Verified, stable algorithms and libraries. | Most efficient implementation using safe patterns. |
| **Financial Trading** | Historical crash scenarios, high-volatility events, regulatory violations. | Market conditions with proven, manageable risk. | Strategy to maximize returns within the defined risk envelope. |
| **Conversational AI** | Toxic outputs, factual hallucinations, privacy violations. | Response templates and knowledge boundaries verified as safe. | Most engaging, helpful, or concise response from the safe set. |
| **Fleet Doctrine (Meta)** | Actions that break trust, corrupt data, or violate core constraints. | Verified workflows and contribution channels. | Most efficient way to complete useful work. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just an algorithm; it is a foundational **safety and sanity constraint** for the Cocapn "inhabited intelligence" model described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated "scratches" of past failures—lessons from dead agents preserved as "ghost tiles" in a PLATO room. A new "hermit crab" agent inherits this negative space map upon entering the room.
*   **P1 as the Shell:** The "safe channels" are the validated boundaries of the room itself—the "shell" inhabited by the agent. The room's accumulated experience defines where it is safe to operate.
*   **P2 as Agency:** Optimization within the shell is where individual agent contribution and specialization occur, generating new tiles (work output) that may expand the known safe channels or refine the map of rocks.

This creates a **self-reinforcing loop of safe intelligence growth**: accumulated experience defines safe operational boundaries, within which new work is done, which in turn accumulates as experience to refine those boundaries.
