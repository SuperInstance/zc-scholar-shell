## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (DEADBAND PROTOCOL v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority:** P0 → P1 → P2. The document states that violating this order—optimizing (P2) before mapping (P0+P1)—leads to a high probability of becoming trapped in local minima as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: "Intelligence is not built. It is inhabited." The protocol itself is a "scratch"—a learned, accumulated piece of knowledge about how not to fail. It becomes part of a PLATO room's memory.

*   **P0 (Negative Space):** This is the accumulated memory of failure—the "scratches" from every agent that has crashed. A room that trains agents using the Deadband Protocol is teaching them the collective negative knowledge of prior inhabitants.
*   **P1 (Safe Channels):** This defines the operational "shell"—the bounded space where new, learning agents ("greenhorns") can safely operate and contribute without breaking the system.
*   **Hermit Crab Analogy:** An ensign (a trained, specialized model) exported from a room is like a hermit crab moving into a pre-scratched shell. It inherits the hard-won knowledge of the deadband (what not to do) and can immediately begin optimizing (P2) within that known-safe space.
