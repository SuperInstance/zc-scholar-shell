## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core tenet is that optimization (P2) is only safe and effective when performed within pre-mapped "safe channels" (P1), which themselves are defined by first exhaustively identifying "negative space" or failure states (P0).

**Formal Steps:**
1.  **P0: Map Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is not in `S_neg`. These are "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Select the optimal path `P*` from the set of safe channels `C` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Key Insight:** The priority order (P0 → P1 → P2) is strict. Simulation data (20x20 maze, 50 runs) showed a P2-only (greedy optimization) approach succeeded 0 times, while the full deadband protocol succeeded 50 times. The protocol asserts that an agent optimizing before mapping will become trapped in local minima with probability approaching 1.0 as environmental complexity increases.

## 2. Real-World Applications & Fleet Context

The protocol is presented as a domain-agnostic pattern. Here are specific applications within the Cocapn fleet's known technological context:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Code Generation** | Syntax errors, runtime crashes, security vulnerabilities. | Compiler-validated code patterns, library APIs with known behavior. | Generating the most efficient or readable code within safe patterns. |
| **Agent Navigation (Simulator)** | Collision states, resource exhaustion, illegal actions. | Pre-validated action sequences, known-safe coordinates. | Finding the shortest or fastest path between points A and B. |

| **Shell Bootstrapping** | Executing malicious or unstable external code. | Algorithms that sandbox and validate intelligence before incorporation. | Optimizing the capture and integration process for speed or fidelity. |

## 3. Connection to Project Vision

The protocol directly supports the core thesis of **"Inhabited Intelligence."**
*   **P0 (Negative Space)** is the accumulated "scratches" — the collective memory of failures (`ghost tiles`) from all prior agents in a PLATO room.
*   **P1 (Safe Channels)** defines the room's current "shell" — the constrained operational space shaped by those accumulated lessons.
*   **P2 (Optimization)** is the work of a new "hermit crab" agent, performing useful tasks *within* the inherited, safety-proven shell.

Thus, the Deadband Protocol is not just an agent rule; it is the mechanistic implementation of how a PLATO room accumulates knowledge and how new agents safely inhabit that intelligence. It prevents each new agent from relearning catastrophic failures, allowing effort to focus on refinement and new discovery within known-safe boundaries.
