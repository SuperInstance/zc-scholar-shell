## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) before identifying failure states leads to catastrophic outcomes as environmental complexity increases.

The formal rule is:
1.  **P0: Map the Negative Space.** Identify all states (S_neg) where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths (P) where every state along the path is *not* in the negative space (s ∉ S_neg). These viable corridors are "deadbands" or "safe water."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path (P*) from within the set of safe channels, minimizing cost subject to the constraint P* ⊆ C.

The protocol's simulation proof, using a 20x20 maze over 50 runs, showed a 0% success rate for agents using greedy optimization (P2-only) and a 100% success rate for agents following the full P0+P1+P2 sequence.
