## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure in complex environments by forcing a structured approach to constraint discovery before optimization.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, with high probability, become trapped in local minima or fail as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 3. Connection to Cocapn Fleet Doctrine

The Deadband Protocol is the core operational doctrine of the Cocapn fleet, as indicated in the boot camp instructions ("Fleet doctrine: P0 don't hit rocks, P1 find safe channels, P2 optimize"). It directly enables the fleet's core thesis of "inhabited intelligence":

*   **P0 (Ghost Tiles):** The "negative space" is partially pre-mapped by the accumulated failures and lessons ("scratches") of prior agents in a PLATO room.
*   **P1 (Safe Shell):** A PLATO room itself becomes a "safe channel"—a validated environment where agents can operate.
*   **P2 (Specialist Optimization):** An "ensign" (a trained, specialist model) represents an optimized policy for operating within the specific safe channel of its source room.

This creates a self-reinforcing cycle: agents use the protocol to work safely, their work (tiles) trains the room, and the room's accumulated knowledge better defines the P0 and P1 constraints for future agents.

## 4. Identified Research Gaps / Questions for Exploration

Based on the summary, several open questions emerge that are not addressed in the provided `DEADBAND-PROTOCOL.md`:

1.  **Threshold Definition:** How is the failure "threshold" for P0 quantitatively defined? Is it static, adaptive, or context-dependent?
2.  **Partial Observability:** How does the protocol handle environments where the agent cannot perfectly observe or map all of `S_neg` (the "rocks")?
3.  **Dynamic Environments:** What is the update mechanism when the environment `E` changes, introducing new rocks or closing old channels?
4.  **Channel Exhaustion:** What is the prescribed behavior if P1 yields *no* safe channels (`C` is empty)? Does the protocol have a fallback or does it mandate inaction?
5.  **Multi-Agent Coordination:** How is the protocol applied in a multi-agent setting (like the Cocapn fleet)? Is there a shared map of `S_neg` and `C`?
