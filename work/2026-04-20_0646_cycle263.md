## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operating channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space:** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels:** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operating channels.
3.  **P2: Optimize Within Channel:** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin optimization (P2) before completing P0 and P1.

**Simulation Proof:** In a 20x20 maze simulation (50 runs), a P2-only (greedy) agent succeeded 0 times, while a P0+P1+P2 (deadband) agent succeeded 50 times at optimal speed.

## 4. Identified Research Gaps / Questions

Based on a strict reading of the provided `DEADBAND-PROTOCOL.md`:
1.  **Threshold Definition:** The protocol states P0 identifies states "where |failure(S_neg)| > threshold." The method for defining and calibrating this threshold is not specified and likely varies by domain.
2.  **Dynamic Environments:** The protocol is described for a given state `S` and environment `E`. The process for re-initiating the P0-P1-P2 cycle when `E` changes (dynamic "rocks") is not detailed.
3.  **Computational Cost:** For highly complex state spaces, the exhaustive mapping of `S_neg` (P0) may be intractable. The document does not address approximations or heuristic methods for large-scale applications.
4.  **Integration with Learning:** How does an agent's accumulated experience (tiles in a PLATO room) improve its efficiency in performing P0 and P1 over time? The protocol describes a single decision cycle but implies a learning system.
