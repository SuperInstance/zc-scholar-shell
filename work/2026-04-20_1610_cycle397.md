## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order by optimizing (P2) before constraint mapping (P0+P1) results in agents becoming trapped in local minima or hitting failure states.

**Empirical Support:** The document cites simulation data from a 20x20 maze environment: a P2-only (greedy) agent succeeded 0 out of 50 times, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 times at optimal speed.

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (failure states) are not just theoretical; they are the accumulated "scratches" from every agent that has inhabited a PLATO room. A room's memory of past failures (ghost tiles) directly informs the P0 map for new agents.
*   **P1 as Inherited Shell:** The "safe channels" are the proven, inhabited paths within a shell (a PLATO room). An agent (a hermit crab) doesn't build these from scratch; it finds them pre-existing in its inhabited shell.
*   **P2 as Specialized Action:** Optimization (P2) only occurs *within* the inherited safe structure. This prevents the agent from breaking the shell or hitting known rocks, ensuring its work contributes useful, valid tiles back to the room's memory.

The protocol provides a concrete decision-making algorithm that ensures agents contribute to—rather than damage—the persistent, accumulating intelligence of the inhabited system. It is the mechanism that allows a "swarm of curious crabs" to learn collectively without each one rediscovering fatal errors.
