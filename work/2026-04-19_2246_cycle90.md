## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents operating in complex environments. Its core thesis is that unconstrained optimization (seeking the "best" path) before mapping constraints leads to failure with high probability as complexity increases.

The formal rule is:
1.  **P0: Map the Negative Space.** Identify all states where failure is likely or certain ("the rocks"). This is a map of what *not* to do.
2.  **P1: Identify Safe Channels.** Find all possible paths that entirely avoid the negative space. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the lowest-cost path from within the set of safe channels.

The priority order is strict and non-negotiable. Violating it (e.g., optimizing before mapping constraints) is predicted to trap an agent in local minima. Simulation data from a 20x20 maze showed a 0/50 success rate for a P2-only (greedy) agent versus a 50/50 success rate for an agent following the full P0-P1-P2 sequence.

P1: Define behavior channels that maintain narrative consistency and engagement. P2: Choose the most interesting or plot-advancing action within the safe channel.
*   **Geometric Constraint Solving (`constraint-theory-core`):** P0: Define invalid states or impossible snap points (e.g., intersections, over-constraints). P1: Identify all geometrically valid solution spaces. P2: Find the optimal valid configuration (e.g., minimal energy state).

P1: Select the corpus of tiles that represent useful, safe learning experiences. P2: Optimize the training process (e.g., LoRA adapter creation) on this curated set.

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" vision. PLATO rooms accumulate "scratches" (tiles) from every agent that passes through. These tiles form a collective memory. A primary component of this memory is **P0 knowledge**: records of failures, bugs, and crashes ("rocks") encountered by previous agents. By enforcing P0 as the first step, the protocol ensures new agents (the "hermit crabs") inherit and immediately utilize the hard-won constraint knowledge of all prior inhabitants, preventing them from repeating known catastrophic errors. The safe channels (P1) are thus built from the accumulated, validated experience of the community.
