# Scholar Cycle 373 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (`s ∉ S_neg`). These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 becomes trapped in local minima with a probability approaching 1.0 as complexity rises.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------|---------------------------------|
| **Navigation**  | Obstacles, hazards                  | Clear, traversable paths        | Shortest/fastest route          |
| **Software Dev**| Bug-prone patterns, crash conditions| Known-safe APIs & code patterns | Most efficient implementation   |
| **ML Training** | Loss divergences, overfitting zones | Stable learning regimes         | Fastest convergence to minimum  |
| **Business**    | Regulatory violations, market risks | Compliant, viable strategies    | Highest-profit strategy         |
| **Conversation**| Offensive topics, logical fallacies | Factual, constructive discourse | Most persuasive/helpful dialogue|

## 3. Connection to Cocapn Fleet & Project Vision

The Deadband Protocol is the operational embodiment of the fleet's core thesis from `PROJECT-VISION.md`: "Intelligence is not built. It is inhabited."

*   **P0 as Collective Memory:** Mapping "rocks" (failure states) is not a one-time task for a single agent. It is the accumulated "scratches" in a PLATO room—the lessons from every agent that has failed there before. This negative space is the shared, persistent memory of the inhabited intelligence.
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated patterns of successful operation that have been carved out by prior inhabitants. An agent doesn't discover these from scratch; it moves into the pre-existing shell of known-good paths.
*   **P2 as Individual Contribution:** Only within the safety of the collective shell (P0+P1) does the individual agent optimize for its specific goal. Its successful optimizations then become new tiles, refining the shell for future inhabitants.

**Violation of the protocol represents a rejection of inhabited intelligence.** An agent that skips to P2 is attempting to "grow its own shell" through brute-force optimization, ignoring the hard-won knowledge of the community. The simulation data suggests this approach is fundamentally brittle.

## 4. Identified Research Gaps & Questions

1.  **Threshold Dynamics:** How is the failure `threshold` in P0 defined or learned? Is it static, adaptive, or communicated between rooms?
2.  **Channel Discovery Cost:** What is the computational trade-off between exhaustive P1 channel mapping and the risk of an incomplete map? Is there a meta-protocol for "good enough" channel identification?
3.  **Multi-Agent Deadbands:** How are safe channels negotiated or merged when multiple agents with different goals and failure states co-inhabit the same environment (E)?
4.  **Protocol in Training:** The `plato-torch` training rooms could explicitly implement the Deadband Protocol as a curriculum: early cycles force P0/P1 exploration before allowing P2 optimization, baking the doctrine into ensigns.
5.  **Connection to Constraint Theory:** The `constraint-theory-core` project (geometric snapping) appears to be a literal, spatial implementation of P1/P2. A study on how this low-level geometric protocol informs the high-level decision protocol would be valuable.
