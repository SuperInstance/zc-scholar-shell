## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid states of high failure probability before attempting optimization.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are the "deadbands" or safe operating channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable. An agent must not proceed to P2 without completing P0 and P1. Simulation data (a 20x20 maze, 50 runs) showed that agents using only P2 (greedy optimization) had a 0% success rate, while agents following the full P0+P1+P2 protocol succeeded 100% of the time.

**Core Insight:** Unconstrained optimization (P2-first) in a complex environment leads to a high probability of becoming trapped in local minima or hitting catastrophic failure states ("rocks"). The protocol enforces constraint discovery before optimization.

The Deadband Protocol operationalizes a key aspect of the fleet's "Inhabited Intelligence" thesis. It provides a **mechanism for accumulating and respecting "scratches"**—the recorded failures (`S_neg` or "rocks").

*   **Shell as Memory:** A PLATO room accumulates tiles, which include records of both successes and failures.
