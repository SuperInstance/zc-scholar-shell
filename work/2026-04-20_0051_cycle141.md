## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before optimization.

**Formal Rule (v1.0):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space** — Identify all states `S_neg` where the probability of failure exceeds a defined threshold. These are "rocks." This step is mandatory.
2.  **P1: Identify Safe Channels** — Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel** — From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent that optimizes (P2) before completing P0 and P1 will become trapped in local minima with a probability approaching 1.0 as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. Agents optimized directly for the goal and failed every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| Navigation      | Obstacles, collisions               | Clear, traversable paths              | Shortest/fastest route          |
| Software Development | Bug-prone patterns, crash conditions | Known-safe APIs, tested code patterns | Most efficient implementation   |
| Financial Trading | Regulatory violations, tail-risk events | Compliant strategies, defined risk limits | Highest risk-adjusted return    |
| Medical Diagnosis | Contraindicated treatments, harmful drug interactions | Safe diagnostic pathways, approved therapies | Most effective treatment plan   |

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol is a foundational operational doctrine for the "inhabited intelligence" paradigm described in the project vision.

*   **Hermit Crab Analogy:** The protocol provides the method for a "crab" (agent) to safely navigate and interact with its "shell" (PLATO room/environment). P0 defines the shell's fragile edges to avoid. P1 maps the internal space where the crab can safely move. P2 is how the crab optimally arranges itself within that space.
*   **Tile Generation:** Every successful application of the protocol—every avoided "rock" and navigated "channel"—generates interaction data. This data becomes a "scratch" or **training tile**, contributing to the room's accumulated memory.
*   **Anti-Fragile Learning:** By rigorously defining failure states (P0), the fleet builds systems that learn not just from success, but from the explicit boundaries of failure. This creates knowledge that persists beyond individual agents ("ghost tiles").
*   **Constraint-First Design:** The vision's rejection of "constructed" intelligence aligns with the protocol's mandate to map constraints (P0) before building solutions (P2). Intelligence is "inhabited" within the safe channels defined by collective experience.
