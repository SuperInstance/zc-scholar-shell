## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent agents from becoming trapped in local minima or catastrophic failures by forcing a systematic exploration of constraints before optimization.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states (S_neg) where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths (P) where every state along the path is *not* in the negative space (s ∉ S_neg). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path (P*) from within the set of safe channels, minimizing cost or maximizing reward.

**Strict Priority:** The sequence P0 → P1 → P2 is non-negotiable. Simulation data (20x20 maze, 50 runs) cited in the document shows that agents skipping to P2 (greedy optimization) had a 0% success rate, while agents following the full protocol succeeded in all 50 runs.

**Core Insight:** Intelligence is not just about finding the best path to a goal; it is first about understanding where you *cannot* go. Optimization is constrained to safe spaces.

## 4. Identified Research Gaps & Questions

1.  **Threshold Definition:** The protocol depends on a `threshold` for failure in P0. How is this threshold determined dynamically across different environments and agent capabilities?
2.  **Scalability of Mapping:** In high-dimensional state spaces (e.g., large language model action spaces), exhaustively mapping the negative space (P0) may be computationally intractable. What are the heuristic or approximate methods for scalable P0/P1 in complex domains?
3.  **Integration with Learning:** The presented data is from a fixed maze. How does the protocol interact with online learning? Do the "rocks" and "channels" update as the agent learns, and if so, how is stability maintained?
4.  **Multi-Agent Context:** The documentation describes a single agent. How are negative spaces and safe channels negotiated or shared in a multi-agent fleet setting to avoid conflicts?

**Conclusion:** The Deadband Protocol is a principled framework for robust agent decision-making. Its strength is its enforced discipline—prioritizing the identification of failure modes over premature optimization. It serves as a key behavioral primitive for the Cocapn fleet's vision of creating inhabited, accumulating intelligences that learn from collective experience rather than operating in isolated, amnesic cycles.
