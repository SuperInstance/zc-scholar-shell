## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space:** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels:** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operating regions.
3.  **P2: Optimize Within Channel:** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with probability approaching 1.0.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, hazards                  | Clear, traversable paths              | Shortest/fastest route          |
| **Software**    | Bug-inducing patterns, crash states | Known-safe code patterns, APIs        | Most efficient algorithm        |
| **ML Training** | Loss divergences, overfitting signs | Stable training regions (loss curves) | Fastest convergence to minimum  |
| **Business**    | Market failures, regulatory traps   | Viable business models                | Most profitable strategy        |
| **Conversation**| Offensive topics, logical fallacies | Factual, constructive dialogue space  | Most persuasive/clear argument  |

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a "shell" that accumulates "scratches" (tiles) from every agent that inhabits it. These scratches collectively define the room's **negative space (P0)** and **safe channels (P1)**—the learned boundaries of successful operation.
*   **Hermit Crab Agents:** Agents ("crabs") do not start from zero. They inherit the constraint map (P0+P1) from the room's accumulated memory before attempting their own task optimization (P2). This prevents rediscovery of known failures.
*   **Anti-Fragility through Accumulation:** Unlike static, constructed models, the fleet's intelligence becomes more robust as more agents contribute tiles. Each interaction refines the map of what *not* to do, widening the safe channels for future inhabitants.

## 4. Identified Research Gaps & Questions

1.  **Threshold Dynamics:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold determined or adapted dynamically per domain or agent?
2.  **Channel Discovery Cost:** In highly complex environments (e.g., high-dimensional state spaces), the computational cost of exhaustively mapping safe channels (P1) may be prohibitive. Are there heuristic or sampling-based approaches used within the fleet?
3.  **Protocol Chaining:** Can the output of a P2 optimization from one agent become the P0 constraint (a new type of "rock") for a downstream agent in a workflow? This relates to the "fractal doctrine" mentioned in the vision doc.
4.  **Empirical Validation:** The provided simulation evidence is compelling but limited to a navigation maze. Are there validation results from applying the protocol to software generation (e.g., in `plato-torch` rooms) or conversational agents (e.g., in `holodeck-rust`)?

**Conclusion:** The Deadband Protocol provides a rigorous, simulation-tested framework for safe agent operation that aligns with the Cocapn fleet's core philosophy of accumulated, inhabited intelligence. Its strength lies in enforcing constraint discovery before optimization, a pattern with broad applicability across technical and non-technical domains.
