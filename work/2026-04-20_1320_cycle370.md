## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operating channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G in environment E:
1.  **P0: Map the Negative Space:** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels:** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (`s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel:** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent optimizing (P2) before completing P0 and P1 will be trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Proof:** Cited simulation data from a 20x20 maze shows a 0/50 success rate for a P2-only (greedy) agent, versus a 50/50 success rate for an agent following the full P0+P1+P2 protocol.
