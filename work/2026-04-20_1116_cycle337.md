# Scholar Cycle 337 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will become trapped in local minima.

**Empirical Basis:** The document cites simulation data from a 20x20 maze with 50 runs. A P2-only (greedy) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The document provides this mapping:

| Domain          | Rocks (P0)                     | Channel (P1)                 | Course (P2)               |
|-----------------|--------------------------------|------------------------------|---------------------------|
| Navigation      | Obstacles                      | Safe water                   | Shortest path             |
| Code            | Bugs, crashes                  | Known-safe patterns          | Most efficient algorithm  |
| **AI Training** | **Loss divergences, mode collapse** | **Stable learning regimes**  | **Fastest convergence**   |

## 3. Synthesis with Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key principle implied by the Cocapn vision: **intelligent behavior emerges from inhabiting and navigating constrained, persistent spaces.**

*   **Shell as Channel:** A PLATO room can be viewed as a pre-mapped "safe channel" (P1). It is a constrained environment where agents (hermit crabs) can operate without hitting fundamental "rocks" (e.g., training instability, catastrophic forgetting of core knowledge).
*   **Tiles as P0/P1 Maps:** The accumulated interaction tiles in a room represent the collective, granular mapping of that space—what works (safe paths) and what doesn't (negative space). An ensign (LoRA adapter) is a compressed export of this navigational knowledge.
*   **Protocol as Training Doctrine:** For a self-training room, the Deadband Protocol provides a concrete algorithm: first, rigorously identify failure modes (P0), then define the bounded parameter/action space where learning is stable (P1), and only then seek performance improvements (P2). This prevents the room from "crashing" into unrecoverable states during self-iteration.

## 4. Identified Research Gaps & Questions

1.  **Threshold Definition:** The protocol depends on a `threshold` for defining `S_neg`. How is this threshold determined or learned dynamically across different domains (e.g., code vs. navigation)?
2.  **Channel Discovery Cost:** The exhaustive search for *all* safe paths (P1) may be intractable in high-dimensional state spaces. What heuristics or approximations does the fleet use (e.g., in `constraint-theory-core` or `fleet-simulator`)?
3.  **Integration with ML:** The document explicitly links the protocol to AI Training. How is this formally implemented within `plato-torch` presets? Is there a "DeadbandRoom" preset that uses P0 (e.g., monitoring loss/gradient norms) to gate the learning process?
4.  **Empirical Validation:** While the 20x20 maze result is cited, broader validation across the fleet's problem domains (simulation, code generation, agent interaction) would strengthen the claim of domain-agnostic utility.

**Conclusion:** The Deadband Protocol is a foundational fleet doctrine that shifts the focus from goal-first optimization to constraint-first navigation. It provides a rigorous, simulation-tested framework that aligns with and enables the Cocapn vision of intelligence as accumulation within inhabited, persistent shells. The next step for validation is to examine its concrete implementation in fleet training and simulation systems.
