## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
- **P0 (Map Negative Space):** Identify all states `S_neg` where the probability or cost of failure exceeds a threshold. These are "rocks" to avoid.
- **P1 (Identify Safe Channels):** Find all paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
- **P2 (Optimize Within Channel):** Select the optimal path `P*` from within the safe channels, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will be trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Proof:** Simulation data from a 20×20 maze over 50 runs showed:
- **P2-only (greedy optimization):** 0/50 successful completions.
- **P0+P1+P2 (deadband protocol):** 50/50 successful completions at optimal speed.

**Potential Applications Beyond the Document:**

1.  **Machine Learning Training:**
    *   **P0:** Identify hyperparameter combinations or data regions that cause model divergence, NaN losses, or memorization.
    *   **P1:** Define the subspace of architectures, learning rates, and data batches that produce stable, convergent training.
    *   **P2:** Optimize for accuracy or speed within that stable subspace.

2.  **Autonomous System Security:**
    *   **P0:** Map network ports, user inputs, or system states known to be vulnerable to exploits or instability.
    *   **P1:** Establish a set of verified-safe actions and communication pathways.
    *   **P2:** Optimize system performance (latency, throughput) using only those safe pathways.

3.  **Content Generation & Safety:**
    *   **P0:** Identify prompts, topics, or stylistic choices that reliably generate harmful, biased, or off-topic content.
    *   **P1:** Define a "safe band" of tones, factual frameworks, and content boundaries.
    *   **P2:** Optimize for engagement, clarity, or creativity within those boundaries.

4.  **Multi-Agent Fleet Coordination (Cocapn Context):**
    *   **P0:** For a Zeroclaw agent, identify actions that corrupt the git-shell, produce hallucinated output, or violate the tile validation protocol.
    *   **P1:** Determine the safe operational channel: reading real files from `fleet-knowledge/`, producing factual summaries, writing useful output.
    *   **P2:** Optimize the quality, speed, or insightfulness of research synthesis within that channel.
