## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is **not** in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with near-certainty.

**Empirical Support:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern for constrained optimization.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, collisions | Clear, traversable space | Shortest/fastest route |
| **Software Development** | Known bugs, crash conditions, security vulnerabilities | Code patterns that compile, pass tests, and are memory-safe | Most efficient or elegant implementation |
| **Machine Learning Training** | Loss divergences, gradient explosions, overfitting regions | Hyperparameter sets that lead to stable, convergent training | Hyperparameters that minimize loss or maximize accuracy |
| **Conversational AI** | Toxic outputs, factual hallucinations, prompt injections | Response formulations that are helpful, harmless, and honest | Most engaging or informative response |
| **Fleet Operations (Cocapn)** | System crashes, service failures, deadlocked agents | Known-stable agent behaviors and service configurations | Most efficient task completion or resource use |

## 3. Connection to Cocapn Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated "scratches" of past failures—lessons learned by every agent that has inhabited a PLATO room. A greenhorn agent inherits this map of negative space, avoiding mistakes it has never personally made.
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated behaviors and patterns that constitute a room's persistent knowledge. They define the operational shell that new agents (the "hermit crabs") move into.
*   **Constrained Optimization:** The strict P0>P1>P2 priority ensures that optimization (growth, efficiency) always occurs **within the bounds of accumulated, shared experience**. This prevents agents from "breaking the shell" in pursuit of a local reward.

**Conclusion:** The Deadband Protocol is more than a navigation rule; it is a foundational safety and learning mechanism for an ecosystem of "inhabited intelligence." It ensures that the optimization drive of individual agents is channeled through the durable, collective memory of the system, allowing for safe, cumulative advancement.
