## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`. Optimization is only permitted within the bounds established by P0 and P1.

**Strict Priority:** The order P0 → P1 → P2 is non-negotiable. The document states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, with high probability, become trapped in local minima or fail as environmental complexity increases.

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment. In 50 runs, a "P2-only" greedy optimizer succeeded 0 times, while the full Deadband Protocol (P0+P1+P2) succeeded 50 times at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic pattern. The provided table offers these mappings:

| Domain          | Rocks (P0)                     | Channel (P1)                    | Course (P2)               |
|-----------------|--------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles                      | Safe water                      | Shortest path             |
| Code            | Bugs, crashes                  | Known-safe patterns             | Most efficient algorithm  |

**Extended Application Reasoning:**

*   **Machine Learning Training:**
    *   **P0 (Rocks):** Training instabilities (e.g., gradient explosion/vanishing), catastrophic forgetting, overfitting to noisy data samples.
    *   **P1 (Channel):** Hyperparameter ranges and architectural choices known to produce stable training (e.g., safe learning rates, batch sizes, optimizer settings).
    *   **P2 (Course):** Finding the fastest convergence or highest final accuracy within that stable hyperparameter space.

*   **Autonomous System Deployment (e.g., Fleet Drone):**
    *   **P0 (Rocks):** No-fly zones, sensor failure modes, weather conditions leading to loss of control.
    *   **P1 (Channel):** All navigable airspace that avoids the P0 constraints.
    *   **P2 (Course):** The most fuel-efficient or fastest route within that safe airspace.

*   **Software Development / DevOps:**
    *   **P0 (Rocks):** Known breaking changes in dependencies, system resource limits, security vulnerabilities.
    *   **P1 (Channel):** Version ranges of dependencies that are compatible, resource configurations that have been tested, secure coding patterns.
    *   **P2 (Course):** Implementing the feature with the best performance or cleanest architecture using only the tools and patterns from P1.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just an algorithm; it is a foundational **doctrine** for the fleet, as evidenced by its placement in required reading. It directly supports the core thesis from `PROJECT-VISION.md`: "Intelligence is not built. It is inhabited."

*   **P0 as Collective Memory:** The "rocks" (negative space) are the accumulated failures and constraints learned by all agents that have inhabited a PLATO room. These are the "scratches on the shell" – persistent knowledge of what *not* to do, preventing every new agent from relearning the same catastrophic failures.
*   **P1 as Inherited Safe Space:** The "safe channels" represent the validated, operational knowledge passed down through the room's memory. A new "hermit crab" agent inherits this safe operating environment, allowing it to be productive immediately without starting from zero.
*   **Constrained Optimization (P2):** This final step represents the agent's unique contribution or specialization within the inherited safe framework. The work it does here becomes new tiles, potentially refining the maps of P0 and P1 for future agents.

In this way, the Deadband Protocol operationalizes the vision. It provides a specific, repeatable method for agents to safely contribute to and learn from a persistent, accumulating intelligence. The protocol ensures that optimization and innovation (P2) are always grounded in the hard-won, collective knowledge of past failures (P0) and successes (P1).
