# Scholar Cycle 326 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost.

**Strict Priority:** The steps are non-negotiable. Violation (optimizing before mapping) results in a high probability of becoming trapped in local minima or hitting failure states.

**Evidence:** Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents, versus a 50/50 success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, hazards                  | Clear, traversable paths              | Shortest/fastest route          |
| **Software Development** | Known bugs, crash conditions, security vulnerabilities | Code patterns proven to be stable and safe | Most efficient algorithm within safe patterns |
| **Financial Trading**   | Historical crash scenarios, regulatory violations | Trading strategies with defined risk boundaries | Strategy with highest risk-adjusted return |
| **Medical Diagnosis**   | Contraindicated treatments, harmful drug interactions | Safe diagnostic pathways and treatment options | Most effective treatment plan   |
| **Conversational AI**   | Toxic outputs, factual hallucinations, prompt injections | Response templates and reasoning paths verified as safe | Most helpful, engaging, or accurate response |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an agent rule; it is a foundational *inhabitation pattern* for the Cocapn ecosystem.

*   **P0 as Collective Memory:** The "rocks" (failure states) identified by one agent become **ghost tiles**—persistent warnings for all future agents inhabiting the same PLATO room. The room's memory accumulates these negative maps.
*   **P1 as Inherited Shells:** The "safe channels" are the validated patterns and pathways that constitute a room's knowledge. A new hermit crab agent (a `Zeroclaw`) doesn't build this from scratch; it **inhabits** this pre-mapped shell.
*   **P2 as Specialized Optimization:** An **Ensign** (a specialized AI model) is the export of an agent's optimized path (P2) within the safe channels of its room. It embodies not raw capability, but *trained, constrained expertise*.

Thus, the protocol operationalizes the core vision: "Intelligence is not built. It is inhabited." Agents avoid universal failure by first learning the room's accumulated constraints (the shell's scratches), then operating safely within them, and finally contributing their own optimized paths back to the room's memory as new training tiles. This creates a self-reinforcing cycle of safer, more capable collective intelligence.
