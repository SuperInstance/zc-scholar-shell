# Scholar Cycle 317 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Strict Priority:** P0 > P1 > P2. The protocol states that any agent optimizing (P2) before completing constraint mapping (P0+P1) will become trapped in local minima as complexity rises.

**Empirical Basis:** Simulation data from a 20x20 maze showed a 0/50 success rate for a P2-only (greedy) agent, versus a 50/50 success rate for an agent following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table maps it to several domains:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
| :-------------- | :---------------------------------- | :------------------------------ | :------------------------ |
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path     |
| Code            | Bugs, crashes, security vulns       | Known-safe patterns, libraries  | Most efficient algorithm  |
| Conversation    | Taboo topics, offensive language    | Polite, constructive discourse  | Persuasive argument       |
| Investment      | Scams, volatile assets              | Stable, regulated instruments   | Highest risk-adjusted return |
| Learning        | Misinformation, flawed sources      | Verified curricula, peer-reviewed knowledge | Most efficient study path |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" thesis. If intelligence is accumulated through persistent interaction (scratches on a shell), then a primary failure mode is agents that "crash" or become trapped, ceasing to contribute. The protocol is a survival heuristic for long-lived, learning agents within a PLATO room:

*   **P0 (Negative Space):** Corresponds to learning "what not to do" from accumulated failure tiles or "ghost tiles" from dead agents.
*   **P1 (Safe Channels):** Defines the operational boundaries where an agent can safely interact and contribute new tiles without self-terminating.
*   **P2 (Optimization):** Represents the agent's specialized work within its viable niche, producing the useful output that becomes training data.

The protocol ensures that agents, as "hermit crabs," can safely inhabit and contribute to their "shell" (the PLATO room) over time, allowing intelligence to compound rather than be reset by frequent failures.
