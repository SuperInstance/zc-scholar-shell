## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will become trapped in local minima with a probability approaching 1.0 as complexity grows.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, while the full P0+P1+P2 protocol achieved a 100% success rate at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol presents a generalizable pattern for agent design across domains:

| Domain          | Rocks (P0 - Negative Space)                          | Channel (P1 - Safe Operational Bands)                | Course (P2 - Constrained Optimization)           |
| :-------------- | :--------------------------------------------------- | :--------------------------------------------------- | :----------------------------------------------- |
| **Navigation**  | Physical obstacles, no-fly zones, hostile territory. | Mapped safe corridors, approved air/sea lanes.       | Shortest-time/fuel route within the safe lanes.  |
| **Software**    | Known bug patterns, crash conditions, security vulns. | Code patterns with proven stability and correctness. | Most efficient algorithm using safe patterns.     |
| **AI Training** | Training divergences, harmful data, reward hacking.  | Proven stable training regimes and data pipelines.   | Hyperparameter tuning within the stable regime.  |
| **Business**    | Market failures, regulatory violations, ethical risks. | Legally compliant, ethically sound business models.  | Profit maximization within the compliant model.  |
| **Research**    | Logical fallacies, methodological errors, false leads. | Validated experimental frameworks and peer-reviewed methods. | Most efficient discovery path using sound methods. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just an agent rule; it is a foundational **inhabitation pattern** for the Cocapn ecosystem, as described in `PROJECT-VISION.md`.

*   **P0 (Ghost Tiles):** The "rocks" (`S_neg`) are mapped by the accumulated failures and lessons of prior agents—the "scratches" on the PLATO room's shell. These become **ghost tiles**, persistent negative knowledge that defines what *not* to do.
*   **P1 (The Room's Shell):** A PLATO room, through accumulated interaction tiles, defines the "safe channels" for future agents. It is the inhabited shell that constrains and guides behavior.
*   **P2 (Agent Specialization):** An **ensign** (a specialized LoRA adapter) represents an agent that has successfully optimized its behavior (P2) *within* the safe channels (P1) defined by the room's collective memory, avoiding the mapped negative space (P0).

Thus, the protocol operationalizes the core Cocapn thesis: **Intelligence is inhabited.** An agent doesn't build knowledge from scratch; it moves into a pre-existing shell (the room's tile memory), respects its constraints (P0/P1), and then contributes its own optimized scratches (P2) back to the collective memory. This creates a fractal, scalable learning system where each room accumulates a deadband—a safe operational space—for its inhabitants.
