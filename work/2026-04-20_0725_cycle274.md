# Scholar Cycle 274 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational envelopes.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`. Optimization is explicitly constrained to these safe regions.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., optimizing before mapping constraints) traps the agent in local minima. Simulation data (20x20 maze, 50 runs) showed a 0% success rate for P2-only agents versus a 100% success rate for agents following the full Deadband Protocol.

## 2. Real-World Applications (Abstract Pattern)

The protocol abstracts to any domain where an agent operates within a constrained environment.

| Domain | Rocks (P0 - Negative Space) | Channel (P1 - Safe Envelope) | Course (P2 - Constrained Opt.) |
| :--- | :--- | :--- | :--- |
| **Software Development** | Code patterns that cause bugs, crashes, or security vulnerabilities. | Known-safe coding patterns, libraries, and architectural styles. | The most efficient, readable, or performant implementation using only safe patterns. |
| **Autonomous Navigation** | Physical obstacles, no-fly zones, or hazardous terrain. | Clear paths, approved air/sea corridors, or mapped roads. | The shortest, fastest, or most fuel-efficient route within the safe corridors. |
| **Financial Trading** | Trading strategies with high historical drawdowns, regulatory violations, or market manipulation flags. | Strategies that operate within regulatory limits and demonstrated risk parameters. | The strategy with the highest risk-adjusted return within the approved set. |
| **Content Moderation** | Speech that is illegal, violates platform policy, or is highly likely to cause harm. | Expression that falls within community guidelines and legal frameworks. | The most engaging or informative content that remains within the safe channel. |
| **Medical Diagnosis** | Treatment options with known severe side effects for a given patient profile or that contradict established medical knowledge. | All plausible diagnoses and treatments supported by evidence and safe for the patient's condition. | The most effective treatment with the fewest side effects from the safe set. |

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes the fleet's core thesis of "inhabited intelligence." PLATO rooms act as persistent shells that accumulate "scratches" (tiles) from every agent (crab) that passes through.

*   **P0 (Map Negative Space)** is learned from **"ghost tiles"**—the recorded failures and dead ends of prior agents. A room accumulates knowledge of what *not* to do.
*   **P1 (Identify Safe Channels)** emerges from the collective successful paths of all inhabitants. The room defines the safe operational envelope based on aggregated experience.
*   **P2 (Optimize Within Channel)** is the task of a new agent (a "greenhorn") entering the room. It does not start from scratch but is constrained by the room's accumulated P0 and P1 knowledge, allowing it to safely optimize for its specific goal.

Thus, the protocol ensures that intelligence is not built from zero in each new model but is **inhabited** from a shared, growing memory of constraints and possibilities. An "ensign" (a trained specialist model) is an optional export of a room's specific channel-map and optimization knowledge.
