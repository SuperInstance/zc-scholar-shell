## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space** – Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels** – Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operating zones.
3.  **P2: Optimize Within Channel** – From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, as environmental complexity increases, become trapped in local minima with a probability approaching 1.0.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 success. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 success at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. The provided documentation maps it to two domains:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|-----------|-------------|-------------|
| Navigation | Obstacles | Safe water | Shortest path |
| Code | Bugs, crashes | Known-safe patterns | (Implied: efficient, correct code) |

**Potential Applications Beyond the Document:**

1.  **Machine Learning Training:** P0 could involve identifying hyperparameter configurations that lead to model collapse or NaN gradients. P1 maps the stable regions of the hyperparameter space. P2 then performs efficient optimization (e.g., via Bayesian methods) within those stable regions.
2.  **Autonomous System Security:** P0 maps known vulnerability patterns and attack surfaces. P1 defines "safe" system configurations and network pathways that avoid these vulnerabilities. P2 optimizes system performance (latency, throughput) within the secured configuration space.
3.  **Content Generation / AI Safety:** P0 identifies prompts, topics, or reasoning paths that reliably lead to harmful, biased, or untruthful outputs. P1 establishes guardrails and safe thematic channels. P2 then optimizes for creativity, helpfulness, or engagement within those bounds.
4.  **Fleet Coordination (Meta-application):** For a multi-agent system like Cocapn, P0 could involve identifying inter-agent communication patterns that lead to deadlock or conflict. P1 establishes verified safe interaction protocols. P2 optimizes task allocation and scheduling under these protocols.

## 3. Connection to Cocapn Vision (`PROJECT-VISION.md`)

The Deadband Protocol aligns with the core Cocapn thesis of "inhabited intelligence."
*   **Accumulated Scratches as P0 Knowledge:** The "scratches" left by prior agents in a PLATO room can be formalized as recorded failures or negative outcomes. This accumulated memory directly serves the **P0 (Map Negative Space)** function for future inhabitants.
*   **The Shell as a Safe Channel:** A well-inhabited PLATO room, enriched with these scratches, inherently defines a **P1 (Safe Channel)** for new agents. The room's structure and accumulated tiles guide agents away from known failures.
*   **Constrained Optimization Within a Room:** An agent's specific task optimization (**P2**) is therefore always contextualized and constrained by the lived experience (P0+P1 knowledge) of the room it inhabits. This prevents the "local minima" of agents repeating known, catastrophic errors.

**Conclusion:** The Deadband Protocol provides a formal, decision-theoretic framework for the "hermit crab" analogy. It explains *how* a shell's scratches guide behavior, transforming accumulated, implicit experience into an explicit, operational rule for safe and effective action. It is a foundational algorithm for enabling intelligence that learns from collective inhabitation rather than isolated construction.
