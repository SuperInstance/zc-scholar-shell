## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure in complex environments by enforcing a sequence that maps constraints before attempting optimization.

### Core Steps (Strict Priority)
1.  **P0: Map the Negative Space.** Identify all states or actions (`S_neg`) that lead to failure (e.g., obstacles, bugs, crashes). This defines "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths (`P`) that avoid `S_neg` entirely. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the most efficient path (`P*`) from within the set of safe channels.

### Key Principle
The protocol is non-negotiable. Violating the priority order—by optimizing (P2) before mapping constraints (P0+P1)—leads to a high probability of failure as environmental complexity increases. Simulation data from a 20x20 maze showed a 0% success rate for P2-only agents versus 100% for agents following the full protocol.

### Abstract Pattern
The framework is domain-agnostic:
*   **Navigation:** Rocks = obstacles; Channel = clear path; Course = shortest route.
*   **Software:** Rocks = bugs/crashes; Channel = known-safe code patterns; Course = most efficient implementation.
