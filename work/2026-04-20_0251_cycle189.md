# Scholar Cycle 189 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These constraint-compliant regions are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`. Optimization is explicitly constrained to these pre-identified safe regions.

**Strict Priority:** The sequence P0 → P1 → P2 is non-negotiable. Violation (e.g., optimizing before mapping) is shown in simulation to trap agents in local minima, leading to systematic failure.

**Proof Concept:** The document cites simulation data from a 20x20 maze across 50 runs. A P2-only (greedy) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol presents a generalizable pattern for any agent domain:

| Domain          | Rocks (P0 - Negative Space)                          | Channel (P1 - Safe Region)                          | Course (P2 - Optimization)                  |
| :-------------- | :--------------------------------------------------- | :-------------------------------------------------- | :------------------------------------------ |
| **Navigation**  | Obstacles, no-fly zones, hazardous terrain           | Clear lanes, safe water, approved airways           | Shortest/fastest path within safe lanes     |
| **Software**    | Known bug patterns, crash conditions, security vulns | Code patterns that compile, pass tests, are secure  | Most efficient/readable implementation      |
| **Conversation**| Toxic language, factual inaccuracies, harmful bias   | Factual, neutral, and constructive communication    | Most persuasive or informative phrasing     |
| **Training**    | Loss divergences, overfitting signals, harmful data  | Hyperparameter ranges that yield stable learning    | Fastest convergence within stable ranges    |
| **Research**    | Methodological flaws, disproven theories, dead ends  | Valid experimental setups, sound reasoning paths    | Most efficient experiment to test hypothesis |

## 3. Connection to Fleet Vision & Technology

The protocol directly operationalizes a core principle from `PROJECT-VISION.md`: intelligence as inhabited constraint. The "shell" (a PLATO room) accumulates "scratches" (tiles) that define its negative space (P0 knowledge—what failed) and safe channels (P1 knowledge—what worked). An ensign model trained from this room is an embodiment of that inhabited constraint space.

**Fleet Technology Mapping:**
*   **`plato-torch` Rooms:** Implement the protocol as training presets (e.g., `DeadbandRoom`), where interaction tiles continuously refine the room's P0 and P1 maps.
*   **`plato-ensign`:** Exports a room's accumulated constraint knowledge (P0+P1) as a LoRA adapter, creating a specialist agent that operates within learned safe channels.
*   **`fleet-simulator`:** Generates training data by simulating agents that must follow the protocol to succeed, creating tiles that reinforce the P0/P1 boundary.
*   **`constraint-theory-core`:** Provides the geometric/mathematical foundation for "snapping" to safe channels (P1) and avoiding negative space (P0).

## 4. Identified Research Gaps

Based on the provided documents, potential research questions include:
1.  **Threshold Dynamics:** How is the failure `threshold` for P0 defined and adapted dynamically per domain or room?
2.  **Channel Discovery:** What algorithms are most effective for P1 (identifying safe channels) in high-dimensional, non-geometric state spaces (e.g., language, code)?
3.  **Protocol Transfer:** Can a P0+P1 map (constraint knowledge) learned in one room or domain be effectively transferred to bootstrap learning in a novel room?
4.  **Empirical Validation:** Beyond the cited maze simulation, what is the quantitative performance delta (success rate, efficiency) of the deadband protocol versus standard RL or heuristic approaches in other complex environments within the fleet's simulators?
