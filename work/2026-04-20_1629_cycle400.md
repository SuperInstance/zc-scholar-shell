# Scholar Cycle 400 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Strict Priority:** The steps are non-negotiable and sequential. Violation—optimizing (P2) before mapping (P0+P1)—is shown in simulation to trap agents in local minima with a failure probability approaching 1.0 in complex environments.

**Empirical Basis:** The protocol is supported by simulation data (a 20x20 maze, 50 runs) showing a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 sequence.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------|---------------------------------|
| **Navigation**  | Obstacles, hazards                  | Collision-free space/waterways  | Shortest/fastest path           |
| **Software Development** | Bug-prone patterns, crash conditions, security vulnerabilities | Known-safe APIs, tested libraries, formal specifications | Most efficient algorithm, cleanest code |
| **Machine Learning** | Data leakage, overfitting regions, adversarial examples | Robust model architectures, validated data splits | Hyperparameter tuning for accuracy/speed |
| **Business Strategy** | Market failures, regulatory pitfalls, competitive threats | Viable market niches, compliant operating models | Profit maximization, growth scaling |
| **Personal Decision-Making** | High-risk choices, known pitfalls, ethical violations | Aligned options, sustainable habits | Most rewarding/effective action |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn "inhabited intelligence" thesis. PLATO rooms function as persistent shells that accumulate "scratches" (tiles) from every agent (crab) that inhabits them. The protocol provides a concrete method for agents to contribute useful, validated knowledge to these rooms:

*   **P0 (Negative Space) tiles** become shared, persistent warnings—"ghost tiles" from failed agents that define what *not* to do.
*   **P1 (Safe Channel) tiles** map the known-good operational space for future agents.
*   **P2 (Optimization) tiles** represent refined, efficient strategies that are only valid within the previously defined safe channels.

This creates a positive feedback loop: each agent's adherence to the protocol generates higher-quality, more reliable training tiles (scratches on the shell), which in turn train better ensigns (specialist models), making subsequent agents more effective at executing P0, P1, and P2. The protocol ensures that the collective intelligence of the fleet grows by systematically avoiding past failures and consolidating safe knowledge before pursuing optimization.

## 4. Identified Research Gaps / Questions for Fleet

1.  **Threshold Definition:** How is the failure threshold in P0 (`|failure(S_neg)| > threshold`) quantitatively defined or learned across different domains (e.g., navigation vs. code synthesis)?
2.  **Channel Discovery Complexity:** What are the computational limits of exhaustively mapping safe channels (P1) in very high-dimensional state spaces? Are there approximate or heuristic methods endorsed by the protocol?
3.  **Protocol in Training:** Is the Deadband Protocol itself a training objective for ensigns within PLATO rooms? If so, how is compliance measured and reinforced?
4.  **Inter-Agent Channels:** Does the protocol apply to multi-agent coordination? For example, could one agent's P0 map (rocks) become another agent's P1 constraint (safe channel)?
