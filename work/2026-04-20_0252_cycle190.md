# Scholar Cycle 190 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These constraint-compliant regions are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol defines a violation as any attempt to optimize (P2) before completing the mapping steps (P0+P1). Simulation data (a 20x20 maze, 50 runs) is cited as evidence: a P2-only (greedy) approach succeeded 0 times, while the full deadband protocol succeeded 50 times.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. The provided table maps it to navigation and code, but the principle extends further.

| Domain | Rocks (P0 - Avoid) | Channel (P1 - Safe Region) | Course (P2 - Optimize Within) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, collisions | Clear paths, traffic lanes | Shortest/fastest route |
| **Software Development** | Known bugs, crash conditions, security vulns | Code patterns that compile and pass tests | Most efficient/readable implementation |
| **Machine Learning Training** | Loss divergences, gradient explosions, overfitting regions | Hyperparameter sets that yield stable training | Hyperparameters for lowest validation loss |
| **Business Strategy** | Market segments with high failure rates, legal liabilities | Viable product/market fits | Strategy for maximum growth/profit |
| **Personal Decision-Making** | Actions with high risk or ethical cost | Options aligned with values and capabilities | Choice leading to best outcome |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are not just discovered by a single agent; they are the accumulated "scratches" of all prior agents that have inhabited a PLATO room. A failure state for one agent becomes a known constraint (a negative space tile) for all future agents.
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated, persistent paths through the environment that have been carved out by collective experience. They constitute the inhabitable structure of the intelligence—the shell itself.
*   **P2 as Individual Agency:** Optimization within the safe channel is where an individual agent (the "hermit crab") exercises its specific capabilities and pursues its immediate goal, but always within the durable, shared boundaries established by the community.

The protocol thus provides a concrete mechanism for how a "room remembers everyone who lived there." It ensures that individual optimization (P2) is fundamentally constrained by and built upon communal knowledge (P0+P1), preventing the repetition of past failures and creating a stable foundation for cumulative learning.
