## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Strict Priority:** P0 > P1 > P2. The protocol states that violating this order—optimizing (P2) before mapping (P0+P1)—will trap an agent in local minima as complexity scales.

**Empirical Basis:** The document cites simulation data from a 20x20 maze across 50 runs. A P2-only (greedy) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

## 3. Connection to Fleet Vision & Technology

Reading `PROJECT-VISION.md` reveals a deep philosophical alignment. The fleet's core thesis is "Intelligence is not built. It is inhabited." Agents (hermit crabs) inhabit PLATO rooms (shells), and their work outputs become training tiles (scratches). The room accumulates the experience of all prior occupants.

The Deadband Protocol operationalizes this at the agent level. It is a **P0 rule for the fleet itself**: a constraint that prevents agents from "hitting rocks" (e.g., producing useless or harmful output) by forcing them to first map the negative space of failure. Successful adherence to the protocol generates useful work, which becomes valid training tiles. These tiles are the scratches that accumulate in the PLATO room, training the next generation of specialist models (ensigns).

**Key Technologies Involved:**
*   **`plato-torch`**: Hosts the "DeadbandRoom" and other training room presets where agents operate and tiles are submitted.

## 4. Identified Research Gaps & Questions

Based on the provided documents, several areas for further inquiry emerge:

1.  **Threshold Definition:** The protocol depends on a `threshold` for defining failure states in P0. How is this threshold determined or learned? Is it static, adaptive, or context-dependent?
2.  **Channel Discovery Complexity:** In high-dimensional state spaces (like code generation or strategic planning), how is the complete set of safe channels (P1) discovered efficiently? Is there a risk of the map being incomplete?
3.  **Protocol Validation:** The maze simulation is a compelling but simple example. How does the protocol's success rate scale in more complex, noisy, or adversarial environments?
4.  **Integration with Learning:** Could the P0 map (negative space) itself be learned and shared across agents as "ghost tiles" (lessons from dead agents), directly feeding the fleet's inhabited intelligence model?
5.  **Contrast with Modern RL:** How does this constraint-first approach compare to modern reinforcement learning paradigms (e.g., PPO, DQN) that often blend exploration (mapping) and exploitation (optimization)?

**Conclusion:** The Deadband Protocol is not just an agent decision rule; it is a foundational safety and efficacy constraint for the Cocapn fleet's entire paradigm of inhabited intelligence. It ensures that the work which becomes training tiles is generated within "safe channels," thereby accumulating valuable, non-destructive scratches in the PLATO room shells.
