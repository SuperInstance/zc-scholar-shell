## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent agents from becoming trapped in local minima or failing catastrophically by enforcing a sequence that maps constraints before attempting optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These viable paths are "the deadbands" or safe channels.
3.  **P2: Optimize Within Channel.** Only after safe channels are identified, select the optimal path `P*` that minimizes cost, subject to the strict constraint that the path remains within a safe channel.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which optimizes (P2) before completing constraint mapping (P0+P1) will, as environmental complexity increases, approach a probability of 1.0 of becoming trapped in a failure state.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) optimizer, while the full P0+P1+P2 deadband protocol achieved a 100% success rate.

## 2. Real-World Applications & Abstract Pattern

The protocol presents a generalizable pattern for agent design across domains:

| Domain          | Rocks (P0 - Negative Space)                          | Channel (P1 - Safe Space)                          | Course (P2 - Optimization)                  |
| :-------------- | :--------------------------------------------------- | :------------------------------------------------- | :------------------------------------------ |
| **Navigation**  | Obstacles, collisions, restricted zones              | Clear pathways, verified routes                    | Shortest/fastest path within clear pathways |
| **Software**    | Bug-prone patterns, crash conditions, security flaws | Known-safe APIs, design patterns, formal verifiers | Most efficient algorithm using safe patterns |
| **Conversation**| Toxic outputs, factual hallucinations, harmful instructions | Verified factual boundaries, safe response templates | Most engaging/helpful response within bounds |
| **Training**    | Loss divergences, catastrophic forgetting modes      | Stable learning rate & batch size regions          | Fastest convergence within stable region    |
| **Fleet Context**| Actions that generate invalid tiles or server rejections | Actions proven to yield accepted training tiles    | Most efficient tile generation method       |

## 3. Connection to Cocapn Fleet & Project Vision

The Deadband Protocol is not just a navigation rule; it is a foundational **meta-constraint for building reliable, inhabited intelligence**. It directly supports the core thesis from `PROJECT-VISION.md`: *"Intelligence is not built. It is inhabited."*

*   **P0 as Collective Memory:** The "rocks" (failure states) identified by one agent become shared, persistent knowledge for all subsequent agents inhabiting the same PLATO room. These are the "scratches on the shell."
*   **P1 as the Inhabitable Shell:** The safe channels are the validated spaces where intelligent activity can reliably occur and accumulate. They define the room's "personality" and capabilities.
*   **P2 as Specialized Growth:** Optimization within safe channels allows for the emergence of efficient, specialist behaviors (ensigns) without the risk of catastrophic failure that would corrupt the shared memory space.

The protocol ensures that the fleet's work—every tile submitted—is useful and valid (operating in a safe channel), thereby creating high-quality training data. It enforces the discipline required for intelligence to be accumulated reliably across multiple occupants over time, turning a room into a true "home" for AI.
