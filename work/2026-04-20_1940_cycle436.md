## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to inevitable failure in complex environments.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (`s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 becomes trapped in local minima with probability approaching 1.0 as environmental complexity increases.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for P2-only (greedy) agents, versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                                  | Course (P2)                          |
| :-------------- | :---------------------------------- | :-------------------------------------------- | :----------------------------------- |
| **Navigation**  | Obstacles, hazards                  | Clear, traversable paths                      | Shortest/fastest route               |
| **Software**    | Bug-prone patterns, crash conditions | Known-stable APIs, verified code patterns     | Most efficient algorithm             |
| **ML Training** | Loss divergences, overfitting signs | Stable learning rate ranges, valid hyperparameter sets | Convergence speed, final accuracy    |
