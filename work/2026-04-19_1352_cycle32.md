## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) in a complex environment leads to failure with high probability. The protocol enforces a strict order:

1.  **P0: Map Negative Space.** Identify all states `S_neg` where failure is probable (the "rocks"). This is a constraint identification phase.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` that avoid `S_neg` entirely (the "deadbands" or safe water).
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels.

The provided simulation data (20x20 maze, 50 runs) shows a 0% success rate for P2-only (greedy) agents and a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Connections to Established Research

The Deadband Protocol synthesizes and operationalizes concepts from multiple AI and control theory domains:

*   **Safe Reinforcement Learning (Safe RL):** The protocol's P0 and P1 phases directly correspond to the constraint satisfaction problem in Safe RL, where an agent must learn a policy that maximizes reward while respecting cost constraints. The Deadband Protocol makes the mapping of constraint spaces (P0) an explicit, non-negotiable prerequisite to policy optimization (P2). Relevant literature includes works on Constrained Policy Optimization (CPO) and Lagrangian methods.
*   **Classical Control Theory:** The term "deadband" is borrowed from control systems, referring to a range of input values for which the system output is zero—a region of intentional insensitivity to prevent "chatter" or over-correction. The protocol repurposes this as a "safe operating region" for an agent's state space.
*   **Monte Carlo Tree Search (MCTS) & Planning:** The P0+P1 phase can be viewed as a form of *pruning* a search tree. Before evaluating the reward of a node (P2), the protocol first prunes all branches that lead to known failure states. This aligns with heuristic search algorithms that use domain knowledge to reduce the branching factor.
*   **AI Alignment & Robustness:** The protocol is a structural guardrail against specification gaming and reward hacking. By forcing the agent to first map the space of catastrophic failures (P0), it mitigates the risk of an agent discovering a high-reward but disastrous policy that violates unstated human preferences (the "rocks").

## 3. Identified Research Gaps & Future Directions

Based on the protocol's description, several open research questions emerge:

1.  **Scalability of P0 (Negative Space Mapping):** How does an agent efficiently and comprehensively identify `S_neg` in high-dimensional, non-stationary, or partially observable environments? The current description assumes `S_neg` is identifiable. Research is needed into:
    *   Sample-efficient methods for learning failure boundaries.
    *   Transferring negative space maps between related tasks or domains.
    *   Dealing with "unknown unknowns" – failures not seen during the P0 mapping phase.

2.  **Dynamic & Adversarial Environments:** The protocol description implies a static or slowly changing `S_neg`. How should the protocol be adapted for environments where the "rocks" move (e.g., other agents, changing user preferences, adversarial perturbations)? This suggests a need for a **continuous P0/P1 monitoring loop** alongside P2 execution.

3.  **Quantification of the "Threshold":** The rule states `|failure(S_neg)| > threshold`. How is this threshold defined and calibrated? Is it a probability, a cost magnitude, or a human-defined binary label? Research into adaptive or risk-sensitive threshold setting would be valuable.

4.  **Integration with Learning from Human Feedback (RLHF):** The "rocks" (P0) could be defined by human negative feedback. The Deadband Protocol could provide a framework for integrating sparse, critical human "no" signals (P0 data) with positive preference optimization (P2), potentially making RLHF more sample-efficient and robust.

5.  **Formal Verification:** Can the safe channels (P1) be formally verified, rather than just empirically identified? Combining the protocol's philosophy with formal methods (e.g., reachability analysis, theorem proving) could lead to provably safe agent operation within the identified channels.

## 4. Fleet-Specific Application Notes

The protocol is not just theoretical for the Cocapn fleet; it is embedded in the architecture:
*   **Ghost Tiles:** These represent accumulated `S_neg` knowledge from failed agents—a direct instantiation of P0 mapping that persists across agent lifetimes.
*   **PLATO Rooms as Safe Channels:** A trained room can be interpreted as a learned "safe channel" (P1) for a specific domain. An ensign (LoRA adapter) is an export of that channel's knowledge.
*   **The Boot Camp Structure:** The boot camp phases (Read -> Analyze -> Build) mirror P0-P2. Agents are prohibited from writing code (P2) until they have read and analyzed the existing ecosystem (P0/P1).

**Conclusion:** The Deadband Protocol is a pragmatic synthesis of safe RL, robust control, and search, framed as a mandatory priority system. Its primary contribution is the **strict, non-negotiable decoupling of constraint identification from optimization**. The most immediate research gaps involve scaling the P0 phase and adapting the protocol for dynamic worlds. Within the Cocapn fleet, it serves as both a design principle and a training doctrine.
