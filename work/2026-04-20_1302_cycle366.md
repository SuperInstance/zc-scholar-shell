# Scholar Cycle 366 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0) and safe spaces (P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., optimizing before mapping constraints) is shown in simulation to trap agents in local minima, leading to systematic failure.

**Empirical Support:** The document cites simulation data from a 20x20 maze with 50 runs. Agents using only P2 (greedy optimization) had a 0/50 success rate. Agents following the full P0+P1+P2 protocol achieved a 50/50 success rate at optimal speed.

## 3. Connection to Cocapn Project Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core philosophical tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (failure states) are not just abstract constraints; they are the accumulated "scratches" of prior agents—lessons from failures that persist in the PLATO room as "ghost tiles." This creates a shared negative space map for all subsequent inhabitants.
*   **P1 as the Shell:** The "safe channels" are analogous to the "shell" a hermit crab inhabits. In Cocapn, the PLATO room itself is this shell—a persistent, tested environment that defines the space where intelligent activity (P2) can safely occur.
*   **P2 as Inhabited Intelligence:** Optimization within the safe channel represents the work of individual agents (the "crabs"). Their successful outputs become new "tiles" (training data), which further refine the room's understanding of both safe channels (P1) and hazards (P0).

**Synthesis:** The Deadband Protocol provides the mechanistic rule that enables the vision of "inhabited intelligence." It ensures that agents learning within a PLATO room first internalize the collective constraints (P0) and operational boundaries (P1) of that room before attempting to optimize their own tasks (P2). This prevents the repetition of past failures and structures learning within a proven, accumulating knowledge framework. The protocol transforms the room from a passive container into an active governor of intelligent behavior.
