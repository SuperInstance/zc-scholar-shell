## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents operating in complex environments. It is a systematic rule designed to prevent agents from becoming trapped in local minima by forcing them to first identify failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent `A` in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** The steps are non-negotiable and must be executed in order (P0 → P1 → P2). The protocol states that an agent which optimizes (P2) before completing the mapping phases (P0+P1) will, as environmental complexity increases, become trapped in local minima with a probability approaching 1.0.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy Optimization):** 0/50 successful completions. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband Protocol):** 50/50 successful completions at optimal speed.

## 3. Connection to Cocapn Vision & Fleet Doctrine

The Deadband Protocol operationalizes a core principle implied in the `PROJECT-VISION.md`: **intelligence is inhabited and accumulated through safe, persistent interaction.**

*   **Shell as a Safe Channel:** A PLATO room functions as a "safe channel" (P1). It is a constrained environment where agents (hermit crabs) can operate. The room's accumulated tiles—the scratches—represent a refined map of both the negative space (failed interactions, ghost tiles) and the proven safe paths (successful learning).
*   **Training as P0/P1 Mapping:** The initial training of an ensign model involves exposing it to the room's history, effectively teaching it "what not to do" (P0 knowledge from ghost tiles) and "what has worked" (P1 knowledge from successful tiles) before it can be effectively deployed or optimized for a specific task (P2).
*   **Fleet Doctrine Alignment:** The protocol's strict P0→P1→P2 priority is the direct implementation of the fleet's core doctrine: "P0: Map negative space (what NOT to do), P1: Find safe channels (where you CAN be), P2: Optimize within channels (best path)."

**Conclusion:** The Deadband Protocol is not merely a navigation algorithm; it is a foundational meta-strategy for ensuring robust agent behavior. It explicitly trades off initial exploration cost (for mapping) against the catastrophic risk of unconstrained optimization.
