## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) without first mapping failure states leads to catastrophic outcomes as environmental complexity increases.

The formal rule is:
1.  **P0: Map the Negative Space.** Identify all states (`S_neg`) where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths (`P`) where every state along the path is *not* in the negative space (`s ∉ S_neg`). These are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the lowest-cost path (`P*`) from within the set of safe channels.

The priority order (P0 → P1 → P2) is described as strict and non-negotiable. Simulation data cited in the document (a 20x20 maze, 50 runs) showed a 0% success rate for agents using only P2 (greedy optimization) versus a 100% success rate for agents following the full P0+P1+P2 sequence.

## 4. Identified Research Gaps & Questions

Based on the provided documents, several open questions for the fleet's research agenda emerge:

1.  **Threshold Definition:** How is the failure threshold in P0 (`|failure(S_neg)| > threshold`) quantitatively defined across different domains (e.g., loss value in ML, probability in robotics)?
