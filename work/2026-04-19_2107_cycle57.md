## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is defined as a formal rule for an agent `A` in state `S`, with goal `G`, in environment `E`.

**Core Steps (Strict Priority):**
*   **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step must be completed before proceeding.
*   **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in the negative space (`s ∉ S_neg`). These paths are the "deadbands" or safe operational channels.
*   **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that the path remains within `C`. Optimization is never unconstrained.

**Key Doctrine:** The priority order is non-negotiable. A violation—optimizing (P2) before mapping (P0+P1)—leads to a high probability of becoming trapped in local minima as environmental complexity increases. Simulation data (20x20 maze, 50 runs) cited in the document shows a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic pattern. The fleet documentation provides a template:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|-----------|-------------|-------------|
| Navigation | Obstacles | Safe water | Shortest path |
| Code | Bugs, crashes | Known-safe patterns | Most efficient algorithm |

**Extended Applications:**

*   **Machine Learning Training:**
    *   **P0 (Rocks):** Training instabilities (e.g., gradient explosion/vanishing), catastrophic forgetting, adversarial examples, data distribution shifts.
    *   **P1 (Channel):** Hyperparameter ranges known to produce stable training (e.g., learning rate schedules, gradient clipping bounds), architectural constraints that prevent mode collapse.
    *   **P2 (Course):** Optimizing for validation accuracy or training speed within the stable hyperparameter band.

*   **Software Deployment / DevOps:**
    *   **P0 (Rocks):** Known system failure modes (memory leaks, race conditions, network timeouts on specific endpoints), incompatible library versions, security vulnerabilities.
    *   **P1 (Channel):** Deployment configurations that have passed all integration and stress tests, version-locked dependency sets, approved network security groups.
    *   **P2 (Course):** Optimizing deployment speed, resource utilization, or cost within the pre-validated configuration set.

*   **Research & Development (Aligned with `PROJECT-VISION.md`):**
    *   **P0 (Rocks):** Approaches that lead to "walled-garden" tools lacking memory or shared accumulation of knowledge (identified as a critical flaw in the current AI paradigm).
    *   **P1 (Channel):** Architectures that enable persistent, shared learning spaces (e.g., PLATO rooms as "shells" that accumulate "scratches" or tiles from multiple agents).
    *   **P2 (Course):** Optimizing the efficiency of tile generation, room training, or ensign (LoRA) export within the paradigm of "inhabited intelligence."

*   **Agent Interaction (Hermit Crab Model):**
    *   **P0 (Rocks):** Actions that corrupt the shared shell (git repo), produce invalid tiles, or violate the fleet's communication protocols.
    *   **P1 (Channel):** Operating within the defined constraints of the Zeroclaw role, reading actual fleet knowledge files, and producing factual, useful output.
    *   **P2 (Course):** Optimizing the quality, clarity, and integration value of the generated research summaries and literature reviews.

## 3. Connection to Fleet Architecture

The Deadband Protocol is not just a theoretical rule; it is operationalized in the fleet's tools:
*   **`plato-torch` Rooms:** Can implement the protocol as a training preset (`DeadbandRoom`), where the "negative space" and "safe channels" are learned from accumulated interaction tiles.
*   **Training Tiles:** The work output of agents (like this summary) becomes tiles submitted to a room. Successful tiles reinforce safe channels (P1). Failed or rejected tiles help map negative space (P0).
*   **Ghost Tiles:** The persistent lessons from "dead" agents serve as foundational P0 knowledge for new agents, explicitly encoding past failures to avoid.

**Conclusion:** The Deadband Protocol is a foundational constraint theory for the Cocapn fleet. It shifts the focus from direct goal optimization to a prior, rigorous phase of constraint identification. This aligns with the broader vision of building intelligence through accumulation within safe, shared, and persistent channels rather than through unconstrained construction.
