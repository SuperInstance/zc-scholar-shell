## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent A in state S, with goal G in environment E:
1.  **P0: Map the Negative Space:** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels:** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (`s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel:** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will become trapped in local minima with probability approaching 1.0 as complexity grows.

**Supporting Evidence:** The document cites simulation data from a 20x20 maze across 50 runs. A P2-only (greedy) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. The provided table maps it to navigation and code, but the pattern extends further:

| Domain | Rocks (P0 - Avoid) | Channel (P1 - Safe Space) | Course (P2 - Optimize Within) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, cliffs, hostile zones | Clear paths, mapped roads | Shortest/fastest route |
| **Software Development** | Known bugs, crash conditions, security vulns | Code patterns with proven stability | Most efficient algorithm |
| **Machine Learning Training** | Loss divergences, gradient explosions, overfit regions | Hyperparameter ranges yielding stable loss descent | Fastest convergence rate |
| **Business Strategy** | Market segments with high failure rates, legal liabilities | Proven business models, compliant operations | Maximum profit/profitability |
| **Personal Decision-Making** | Actions leading to known negative outcomes (health, financial) | Choices aligned with values and resources | Most efficient path to goal |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just a decision rule; it is a foundational *inhabitation pattern* for the Cocapn "hermit crab" intelligence model.

*   **P0 (Negative Space) as Ghost Tiles:** The "rocks" are the accumulated failure states—the "scratches" in the PLATO room shell left by agents that failed. These become **ghost tiles**, persistent P0 knowledge for all future inhabitants.
*   **P1 (Safe Channels) as Room Memory:** The safe channels are the validated paths and patterns that have worked for previous agents. This collective memory defines the operational "deadband" of the room itself.
*   **P2 (Optimization) as Agent Specialization:** An ensign (specialist model) exported from a room is an agent optimized to operate *within that room's specific, learned safe channels*.

The protocol enforces that intelligence (P2 optimization) must be *inhabited* within a shell of accumulated collective experience (P0+P1). An agent cannot be truly intelligent in isolation; its effectiveness is constrained by the quality and completeness of the negative space map and safe channel definitions provided by its environment—the PLATO room. This directly realizes the vision's thesis: "Intelligence is not built. It is inhabited."
