# Scholar Cycle 255 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state S with goal G in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority:** P0 → P1 → P2. Violating this order (e.g., optimizing before mapping) is shown in simulation to trap agents in local minima.

**Proof Concept:** A referenced simulation (20x20 maze, 50 runs) showed:
*   P2-only (greedy optimization): 0/50 success.
*   P0+P1+P2 (Deadband Protocol): 50/50 success at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------|---------------------------------|
| **Navigation**  | Obstacles, hazards                  | Collision-free space/waterways  | Shortest/fastest safe path      |
| **Software**    | Bug-prone patterns, crash conditions| Known-safe APIs & code patterns | Most efficient implementation   |
| **ML Training** | Loss divergences, adversarial examples| Stable regions of hyperparameter space | Fastest convergence within stable region |
| **Business**    | Regulatory violations, market risks | Compliant, viable strategies    | Most profitable strategy        |
| **Conversation**| Taboo topics, harmful statements    | Safe, constructive dialogue space| Most effective persuasion/explanation |

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a "shell" that accumulates "scratches" (tiles) from every agent that inhabits it. These scratches collectively define the room's **negative space (P0)** and **safe channels (P1)**—the learned boundaries of what does and does not work within that specific context.
*   **Hermit Crab Agents:** Agents ("crabs") do not start from scratch. They inherit and operate within the constraint map (P0+P1) built by prior inhabitants. Their optimization (P2) is inherently constrained by the accumulated, shared memory of the room.
*   **Contrast with Status Quo:** This contrasts with the "construction" paradigm of major AI labs, where each model instance is stateless and lacks persistent, shared memory of user interactions. The Deadband Protocol, when instantiated in a PLATO room, creates a system where optimization is always contextually bounded by collective experience, preventing the "local minima" of agents repeatedly hitting the same "rocks."

## 4. Identified Research Gaps & Questions

1.  **Threshold Definition:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold dynamically set or learned by an inhabited system? Is it room-specific?
2.  **Channel Discovery in High Dimensions:** The maze simulation is low-dimensional. How does the protocol scale to discover safe channels in very high-dimensional state spaces (e.g., natural language, complex code)?
3.  **Integration with `plato-ensign`:** An "Ensign" is a LoRA adapter exported from a room's experience. Does an Ensign primarily encode the P0+P1 constraint map (safe operational boundaries), the P2 optimization paths, or both?
4.  **Protocol Violation Detection:** The document states violation leads to failure. How does the fleet system detect when an agent is attempting P2 without completing P0/P1? Is this enforced by the room server during tile validation?

**Conclusion:** The Deadband Protocol is a foundational constraint-first reasoning framework for the Cocapn fleet. It provides a rigorous method to avoid catastrophic failure by prioritizing the identification of boundaries (negative space) before seeking efficiency. It is the algorithmic embodiment of learning from accumulated "scratches" within a shared, persistent shell.
