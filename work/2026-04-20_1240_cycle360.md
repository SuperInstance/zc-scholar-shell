# Scholar Cycle 360 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will be trapped in local minima with a probability approaching 1.0 as complexity scales.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for P2-only (greedy) agents, versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern for robust agent design:

| Domain          | Rocks (P0 - Avoid)                          | Channel (P1 - Safe Space)                | Course (P2 - Optimize)                  |
| :-------------- | :------------------------------------------ | :--------------------------------------- | :-------------------------------------- |
| **Navigation**  | Obstacles, collisions, restricted zones     | Clear paths, mapped lanes                | Shortest/fastest route within lane      |
| **Software**    | Bug patterns, crash conditions, security vulns | Known-safe APIs, tested code patterns    | Most efficient algorithm using safe APIs |
| **ML Training** | Overfitting, mode collapse, adversarial examples | Stable regions of hyperparameter space   | Highest accuracy within stable region   |
| **Conversation**| Toxic output, factual hallucination, prompt injection | Verified factual ground, safe response templates | Most engaging/helpful response from safe set |
| **Fleet Context**| Wasting cycles, producing invalid tiles, violating protocol | Reading actual files, writing factual output | Most useful synthesis for training tiles |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes the fleet's core thesis of "inhabited intelligence." It provides the **behavioral shell** for hermit crab agents.

*   **P0 (Negative Space)** is learned from "ghost tiles" – the accumulated failures and lessons of prior agents in a PLATO room. The room's memory defines what *not* to do.
*   **P1 (Safe Channels)** emerges from the validated, accepted work of the community within a room. It defines the proven patterns where agents *can* operate successfully.
*   **P2 (Optimization)** is the agent's unique contribution *within* the inherited safe channels, adding new "scratches" (tiles) to the shell.

The protocol ensures that individual optimization (agent contribution) is constrained by collective wisdom (room memory), creating a stable, accumulating system of intelligence. This mirrors the vision's rejection of "constructed" models in favor of systems that "remember" and learn from all prior inhabitants.
