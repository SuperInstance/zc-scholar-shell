# Scholar Cycle 416 — Deadband Protocol: Summary and Applications (Revised)

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths define the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing cost or maximizing reward.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., optimizing before mapping constraints) is shown in simulation to trap agents in local minima, leading to systematic failure.

## 2. Real-World Applications & Analogies

The protocol abstracts to any domain where an agent operates within a constrained environment.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, hostile zones. | Clear pathways, safe water, approved air corridors. | Shortest/fastest route within the safe network. |
| **Software Development** | Known bug patterns, crash conditions, security vulnerabilities. | Code patterns that have been tested and verified as stable. | Performance optimization, refactoring for readability within safe patterns. |
| **Financial Trading** | Regulatory violations, tail-risk events, liquidity traps. | Compliant strategies, asset classes with acceptable risk profiles. | Return maximization within the compliant, risk-bounded strategy set. |
| **Medical Diagnosis** | Life-threatening conditions, drug contraindications, harmful procedures. | Differential diagnoses that are non-lethal and testable; safe treatment options. | Most likely diagnosis or most effective treatment from the safe set. |
| **Conversational AI** | Toxic outputs, factual hallucinations, privacy violations, prompt injections. | Response templates and reasoning paths that are verified as safe and helpful. | Most engaging, accurate, or concise response from the safe set. |

## 3. Connection to Cocapn Fleet & PLATO Rooms

The Deadband Protocol is not just an agent rule; it is a foundational **training paradigm** for the fleet.

*   **P0 as Ghost Tiles:** The "negative space" (rocks) is learned from collective failure. In a PLATO room, these are the `ghost tiles`—lessons from dead agents that define what *not* to do. A room accumulates this P0 knowledge over time.
*   **P1 as Room Presets:** A trained PLATO room (e.g., a `DeadbandRoom` preset in `plato-torch`) embodies a mapped "safe channel" for a specific domain. An agent entering that room inherits this P1 knowledge.
*   **P2 as Specialization:** An `ensign` (a LoRA adapter exported from a room) represents an agent that has optimized (P2) for a specific task *within* the room's safe channel. It is a specialist born from constraint.

**Conclusion:** The protocol provides a formal structure for the Cocapn vision of "inhabited intelligence." Agents (hermit crabs) do not build intelligence from scratch; they inhabit shells (PLATO rooms) that contain accumulated P0 and P1 knowledge (scratches), and then perform P2 optimization within that safe, remembered space. This creates a scalable, cumulative learning system where each agent's work contributes to the collective map of constraints and channels.
