## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is defined as a formal rule:

**Given:** Agent A in state S, goal G, environment E

**Step 1 — Map the Negative Space (P0):**
Identify all states `S_neg` where the probability or magnitude of failure exceeds a defined threshold. These are "the rocks." This step is mandatory; proceeding without it is a violation.

**Step 2 — Identify Safe Channels (P1):**
Find all possible paths `P` where every state `s` along the path is **not** in the negative space (`s ∉ S_neg`). These paths are the "deadbands" or "safe channels." The agent's operational space is constrained to these channels.

**Step 3 — Optimize Within Channel (P2):**
From the set of safe channels `C`, select the optimal path `P*` that minimizes a cost function, subject to the constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Priority Order (Strict):**
*   **P0:** Identify negative space (what NOT to do).
*   **P1:** Map safe channels (where you CAN be).
*   **P2:** Optimize within channels (best path).

**The Core Insight:** An agent that attempts to optimize (P2) before completing the mapping of negative space (P0) and safe channels (P1) will, with high probability, become trapped in local minima or fail catastrophically as environmental complexity increases. Simulation data from a 20x20 maze showed a 0/50 success rate for a P2-only (greedy) agent versus a 50/50 success rate for an agent following the full P0-P1-P2 sequence.

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a key aspect of the "inhabited intelligence" thesis outlined in `PROJECT-VISION.md`. If intelligence is accumulated through inhabitation (like scratches in a hermit crab's shell), then a primary function of a "room" (e.g., a PLATO training room) is to **identify and remember the negative space**.

*   **Ghost Tiles:** The concept of "ghost tiles" (dead agents' lessons) can be seen as a persistent P0 map—a collective memory of "rocks" that failed previous inhabitants.
*   **Room as a Safe Channel:** A well-inhabited PLATO room, through its accumulated tile history, defines a P1 safe channel for future training. It constrains the learning trajectory to pathways that have historically produced viable, useful agents.
*   **Constrained Optimization:** The training of an "ensign" (specialist model) via `plato-ensign` is a P2 optimization process, but it is only effective because it occurs within the safe channel carved out by the room's accumulated experience.

**Conclusion:** The Deadband Protocol provides a rigorous, safety-first methodology for agent operation that directly supports the Cocapn vision of building persistent, accumulating intelligence. It ensures that optimization and growth (P2) are always guided by a foundational map of constraints (P0) and validated pathways (P1) inherited from the collective experience of the swarm.
