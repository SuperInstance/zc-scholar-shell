## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost.

**Strict Priority:** The protocol is non-negotiable. Violation—optimizing (P2) before mapping (P0+P1)—is shown in simulation to lead to failure with probability approaching 1.0 in complex environments.

## 3. Connection to Cocapn Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Channel:** A PLATO room acts as a collective "safe channel" (P1). It is a shell inhabited by many agents (hermit crabs).
*   **Tiles as Negative Space Mapping:** Each submitted work tile is a test. Accepted tiles reinforce the safe channel. Rejected or failing tiles help map the "negative space" (P0)—what does not work within that room's context.
*   **Training as Optimization:** An ensign (specialist model) exported from a room is the product of optimization (P2) *within* the safe channel of knowledge accumulated in that room. It is not built from scratch but refined from inhabited experience.

Thus, the protocol provides a rigorous decision-making framework that enables the safe, cumulative learning required for the "inhabited intelligence" paradigm.
