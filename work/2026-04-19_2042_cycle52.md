## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** The steps are non-negotiable. An agent must not proceed to P1 without completing P0, and must not proceed to P2 without completing P1. Violating this order (e.g., optimizing for a goal before mapping obstacles) leads to a high probability of becoming trapped in local minima or failing outright as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy Optimization):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband Protocol):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents an abstract pattern applicable to any agent domain. The following table maps the protocol's phases to concrete examples beyond navigation:

| Domain | Rocks (P0 - What NOT to do) | Channel (P1 - Where you CAN be) | Course (P2 - Best path) |
| :--- | :--- | :--- | :--- |
| **Software Development** | Code patterns that cause bugs, crashes, or security vulnerabilities. | Known-safe design patterns, libraries, and API usage patterns. | The most efficient, readable, and maintainable implementation using the safe patterns. |
| **Financial Trading** | Market conditions or asset behaviors historically linked to significant losses. | Trading strategies and asset classes with defined, acceptable risk profiles. | The strategy that maximizes risk-adjusted returns within the safe set. |
| **Conversational AI** | User prompts or model responses that are toxic, factually incorrect, or violate safety guidelines. | Response templates, factual knowledge bases, and moderated dialogue paths. | The most helpful, engaging, and contextually appropriate response from the safe set. |
| **Robotics (Physical)** | Joint configurations that cause self-collision, object collisions, or exceed torque limits. | The set of all collision-free, torque-compliant trajectories in configuration space. | The fastest or most energy-efficient trajectory within the safe set. |
| **Research Synthesis (This Agent)** | Drawing conclusions not supported by the provided source files ("hallucinating"). | Statements and summaries that can be directly traced to citations from `fleet-knowledge/`. | The most concise, clear, and well-structured synthesis of the factual information. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is a foundational operational doctrine for the Cocapn fleet, directly supporting the core thesis of "inhabited intelligence" outlined in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (failure states) identified by agents become "scratches on the shell." When an agent fails, that experience is captured as a "ghost tile" and submitted to a PLATO room. This negative knowledge (`S_neg`) accumulates, becoming a shared map of hazards for all future agents that inhabit that room.
*   **Safe Channels as Emergent Intelligence:** The safe channels (P1) are not pre-programmed but are discovered and refined through continuous agent interaction. The room learns the viable paths by aggregating the successful traversals of its inhabitants.
*   **Constrained Optimization for Specialization:** The "ensign" models trained from room tiles are specialists that operate within the well-mapped safe channels of their specific domain (P1), allowing them to optimize (P2) effectively without rediscovering first principles or repeating known failures.

In this system, the Deadband Protocol ensures that the swarm of "hermit crab" agents can safely explore and build upon the accumulated knowledge of the shell (the PLATO room), turning individual experience into persistent, shared intelligence.
