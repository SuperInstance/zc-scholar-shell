## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

**P0: Map Negative Space**
Identify all states or actions that lead to failure (e.g., obstacles, bugs, crashes). This step defines "what NOT to do." The protocol states that proceeding without this map is a violation.

**P1: Identify Safe Channels**
Find all possible paths that entirely avoid the negative space mapped in P0. These are the "deadbands" or safe operational channels. The agent's action space is constrained to these channels.

**P2: Optimize Within Channel**
Only after P0 and P1 are complete does the agent select the optimal path (e.g., shortest, fastest, lowest cost) from within the set of safe channels. Optimization is never unconstrained.

The priority order (P0 → P1 → P2) is described as strict and non-negotiable. Simulation data cited in the document (a 20x20 maze, 50 runs) showed a 0% success rate for agents using greedy P2-only optimization, versus a 100% success rate for agents following the full Deadband Protocol.

## 2. Abstract Pattern & Real-World Applications

The document provides an abstract pattern, suggesting the protocol applies to any agent in any domain. Based on this, potential real-world applications include:

| Domain | Rocks (P0 - Negative Space) | Channel (P1 - Safe Space) | Course (P2 - Optimization) |
| :--- | :--- | :--- | :--- |
| **Autonomous Navigation** | Physical obstacles, no-fly zones, geofenced areas. | Mapped, obstacle-free corridors or air/sea lanes. | Most fuel-efficient or fastest route within the safe corridor. |
| **Software Development** | Known bug patterns, crash-inducing code, insecure functions (e.g., `gets()` in C). | Code patterns that are proven safe, well-tested libraries, approved APIs. | Most performant or readable implementation using the safe patterns. |
| **Financial Trading** | Trading halts, extreme volatility events, regulatory restrictions. | Market conditions with stable liquidity and within risk limits. | Trading strategy to maximize returns within the defined safe parameters. |
| **Clinical Decision Support** | Contraindicated drug interactions, harmful procedures for a given diagnosis. | Evidence-based treatment protocols that are safe for the patient's profile. | Most effective treatment option from within the safe protocol set. |
| **Content Moderation** | Clearly violative content (e.g., hate speech, graphic violence). | Ambiguous content that requires contextual review. | Most accurate and consistent labeling of ambiguous content within policy guidelines. |

## 3. Connection to Cocapn Fleet & PLATO Rooms

The Deadband Protocol is not just an algorithm; it is a foundational doctrine for the Cocapn fleet's approach to building "inhabited intelligence." As described in `PROJECT-VISION.md`, agents (the "hermit crabs") inhabit PLATO rooms (the "shells"). Their work outputs become "tiles" (scratches on the shell).

In this context, the Deadband Protocol can be seen as a meta-rule for how agents should generate those tiles:
*   **P0 (Negative Space):** An agent's early work (or "ghost tiles" from failed agents) identifies failure modes—what approaches do not work, what code crashes, what prompts yield poor results.
*   **P1 (Safe Channels):** This accumulated negative knowledge defines the constrained space of productive action for subsequent agents. A PLATO room becomes a repository of these safe channels.
*   **P2 (Optimization):** Newer agents, inheriting this room's memory, can then efficiently optimize their work *within* the proven safe channels, avoiding previously mapped pitfalls.

Thus, the protocol enables the fleet's core thesis: intelligence is not built from scratch but **inhabited** and **accumulated** through shared, persistent experience. The Deadband Protocol provides the disciplined process that makes this accumulation reliable and prevents the swarm from repeatedly crashing on the same rocks.
