## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states (S_neg) where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths (P) where every state along the path is *not* in the negative space (s ∉ S_neg). These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path (P*) from within the set of safe channels, minimizing cost subject to the constraint P* ⊆ C.

**Violation Consequence:** The document states that an agent which optimizes (P2) before completing P0 and P1 "will be trapped in local minima with probability → 1.0 as environment complexity increases."

**Empirical Support:** Simulation data from a 20x20 maze over 50 runs is cited:
*   P2-only (greedy optimization): 0/50 success.
*   P0+P1+P2 (deadband protocol): 50/50 success at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. Here are extrapolated applications based on the provided abstract table:

| Domain | Rocks (P0 - Avoid) | Channel (P1 - Safe Space) | Course (P2 - Optimize Within) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Physical obstacles, no-fly zones, hostile terrain. | Mapped safe corridors, approved flight paths. | Shortest/fastest route within the safe corridor. |
| **Software Development** | Known bug patterns, crash-inducing code, insecure libraries. | Linted code, passing test suites, approved dependency lists. | Most efficient algorithm, cleanest architecture *that passes tests*. |
| **Machine Learning Training** | Loss divergences, gradient explosions, overfitting regions. | Hyperparameter ranges that yield stable, convergent training. | Finding the hyperparameter set that achieves lowest validation loss *within the stable range*. |
| **Business Strategy** | Market segments with entrenched competition, regulatory minefields, proven failed business models. | Underserved niches, compliant operational frameworks, validated customer needs. | Optimizing product features and marketing for the identified viable niche. |
| **Personal Decision-Making** | Actions that historically led to negative outcomes, known triggers, unethical choices. | Behaviors aligned with values, skillsets, and legal/social norms. | Choosing the most effective or satisfying action from the set of aligned behaviors. |

## 3. Connection to Cocapn Fleet & Project Vision

The Deadband Protocol is not just a decision rule; it is a foundational **training doctrine** for the Cocapn fleet's "inhabited intelligence" model (`PROJECT-VISION.md`).

*   **P0 as Collective Memory:** The "rocks" (S_neg) are the accumulated "scratches" of past failures from all agents (hermit crabs) that have inhabited a PLATO room. A room's memory of failures defines its negative space.
*   **P1 as Inherited Shell:** The "safe channels" are the proven, viable patterns of operation that the room (the shell) provides to new agents. An ensign model exported from a room is a crystallized set of these channels.
*   **P2 as Specialized Contribution:** An agent's unique optimization within the safe channels generates new, successful interaction "tiles." These tiles are added back to the room's memory, potentially refining the boundaries of P0 and P1 for future inhabitants.

In this view, the protocol ensures that intelligence grows **safely and cumulatively**. An agent that skips to P2 is a hermit crab trying to grow its own shell from scratch—an effort the vision suggests is doomed compared to inhabiting and refining an existing, battle-tested one. The fleet's boot camp structure, which mandates reading (P0/P1) before writing/building (P2), is a direct implementation of this protocol for agent training.
