## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0) and safe spaces (P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (DEADBAND PROTOCOL v1.0):**
- **P0: Map the Negative Space** – Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory and non-negotiable.
- **P1: Identify Safe Channels** – Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are "deadbands" or "safe water." The agent is only permitted to operate within these channels.
- **P2: Optimize Within Channel** – From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains entirely within `C`. Optimization is never unconstrained.

**Strict Priority:** P0 → P1 → P2. Violating this order—optimizing (P2) before completing P0 and P1—results in agents becoming trapped in local minima. Simulation data from a 20x20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 sequence.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|--------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, collisions               | Clear, traversable paths              | Shortest/fastest route          |
| **Software**    | Bugs, crashes, security vulns       | Known-safe code patterns, libraries   | Most efficient algorithm        |
| **ML Training** | Divergence, overfitting, NaN losses | Stable hyperparameter regions         | Fastest convergence             |
| **Business**    | Market failures, legal violations   | Viable business models, regulations   | Profit maximization             |
| **Conversation**| Offensive topics, misinformation    | Factual, respectful discourse bounds | Most persuasive/helpful response|

## 3. Connection to Cocapn Fleet & PLATO Rooms

The Deadband Protocol is not just an agent rule; it is a foundational principle for the Cocapn fleet's "inhabited intelligence" vision (`PROJECT-VISION.md`).

*   **P0 as Collective Memory:** A PLATO room accumulates "ghost tiles"—lessons from failed agents. These tiles collectively map the negative space (`S_neg`) for all future inhabitants. A hermit crab agent inherits a shell pre-scratched with warnings.
*   **P1 as Room Presets:** The `plato-torch` system's training room presets (e.g., `DeadbandRoom`) encode known-safe channels—proven interaction patterns and constraints where learning can occur without catastrophic failure.
*   **P2 as Specialized Ensigns:** An `ensign` (a LoRA adapter exported from a room) represents an optimization (P2) of behavior *within* the safe channels (P1) defined by that room's accumulated negative knowledge (P0). It is intelligence constrained by inherited wisdom.

**Conclusion:** The Deadband Protocol provides the operational doctrine that enables the Cocapn thesis: intelligence is not built from scratch but *inhabited* through successive generations of agents learning within the persistent, accumulating memory of shared shells (PLATO rooms). It is the algorithm that turns a collection of "scratches" (tiles) into a navigable world.
