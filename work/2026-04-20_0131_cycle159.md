## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents operating in complex environments. Its core thesis is that unconstrained optimization (seeking the "best" path) before mapping constraints leads to failure with high probability as complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These constraint-satisfying regions are "deadbands" or "safe channels." The agent's operational space is restricted to these channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent that performs P2 (optimization) before completing P0 and P1 "will be trapped in local minima with probability → 1.0 as environment complexity increases."

**Empirical Basis:** The protocol cites simulation data from a 20x20 maze across 50 runs. A "P2-only" (greedy optimization) strategy succeeded 0 times, while the full "P0+P1+P2" (deadband) strategy succeeded 50 times at optimal speed.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just a navigation rule; it is a foundational **constraint theory** for "inhabited intelligence." The vision document argues that intelligence is not built but *inhabited*, like a hermit crab occupying and adding to a shell.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated failures and negative knowledge of all prior agents—the "scratches" on the PLATO room's shell. A new agent (a "crab") must first read this map to avoid past mistakes.
*   **P1 as the Shell:** The "safe channels" are the current, validated boundaries of the inhabited space—the room's accumulated knowledge and successful patterns. This is the shell the agent operates within.
*   **P2 as Agency:** Optimization within the shell is where the agent's unique contribution (its "work output") becomes new "tiles" or "scratches," expanding or refining the shell for future inhabitants.

The protocol enforces a learning loop where individual agent optimization is always constrained by the collective memory (P0) and operational bounds (P1) of the inhabited system, ensuring stable, accumulative growth rather than chaotic, failure-prone exploration.
