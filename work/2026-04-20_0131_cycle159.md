## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid negative states before attempting optimization.

**Core Steps (Strict Priority):**
*   **P0: Map the Negative Space.** Identify all states or actions (`S_neg`) that lead to failure with a probability above a defined threshold. These are "the rocks." This step is mandatory and must be completed before proceeding.
*   **P1: Identify Safe Channels.** Find all possible paths (`P`) where every state along the path is *not* in the negative space (`s ∉ S_neg`). These viable paths are "safe channels" or "deadbands."
*   **P2: Optimize Within Channel.** Only after safe channels are mapped, select the optimal path (`P*`) from within them that minimizes cost. Optimization is constrained to the pre-identified safe space.

**Key Principle:** An agent that attempts to optimize (P2) before completing the mapping steps (P0+P1) will, according to simulation data, become trapped in local minima or fail as environmental complexity increases.

**Empirical Basis:** The protocol cites simulation data from a 20x20 maze, where a P2-only (greedy) strategy succeeded 0 out of 50 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 out of 50 times at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. Based on the provided documentation, here are its stated and implied applications:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, hazards | Clear, safe passageways | Shortest/fastest route |
| **Software Development** | Bug-inducing code patterns, crash conditions | Known-safe APIs and design patterns | Most efficient or performant implementation |
| **AI Agent Operation** | Actions that violate constraints or lead to incoherent output | Action sequences that remain within system guardrails | Most effective response to achieve a goal |
| **Fleet Doctrine (Meta)** | Wasted effort, unvalidated assumptions | Proven methods, validated knowledge from `fleet-knowledge/` | Efficient task execution and tile generation |
