## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Strict Priority:** The sequence P0 → P1 → P2 is non-negotiable. Violation (e.g., optimizing before mapping constraints) is shown in simulation to trap agents in local minima, leading to systemic failure.

**Empirical Basis:** The document cites simulation data from a 20x20 maze across 50 runs. A P2-only (greedy optimization) strategy succeeded 0 times. The full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. The provided table maps it to navigation and code, but the principle extends further.

| Domain          | Rocks (P0 - Avoid)                          | Channel (P1 - Safe Space)                  | Course (P2 - Optimize)                     |
| :-------------- | :------------------------------------------ | :----------------------------------------- | :----------------------------------------- |
| **Navigation**  | Obstacles, collisions, restricted zones     | Clear lanes, verified routes               | Shortest time/fuel path within clear lanes |
| **Software Dev**| Known bugs, crash conditions, anti-patterns | Compiler-validated patterns, passing tests | Most efficient/readable implementation     |
| **ML Training** | Loss divergences, overfitting signatures    | Stable loss curves, validated checkpoints  | Fastest convergence within stable region   |
| **Security**    | Exploitable vulnerabilities, attack vectors | Hardened configurations, approved protocols| Most performant secure configuration       |
| **Project Mgmt**| Historical failure modes, resource sinks    | Proven methodologies, within-budget plans  | Most efficient timeline/resource allocation|

## 3. Connection to Cocapn Fleet & PLATO Rooms

The Deadband Protocol is not just an agent rule; it is a foundational **training constraint**. This connects directly to the vision in `PROJECT-VISION.md`:

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) identified by one agent become **ghost tiles**—persistent lessons of failure for all subsequent agents in a PLATO room. The room accumulates a shared negative space map.
*   **P1 as Room Personality:** The "safe channels" are the emergent, validated patterns of interaction that define a room's expertise. An agent inhabiting a room inherits this channel map.
*   **P2 as Specialized Optimization:** An **ensign** (a LoRA adapter exported from a room) is the crystallized result of P2 optimization *within* the specific safe channels of that room. It is intelligence specialized for a constrained, known-good operational space.

**Conclusion:** The Deadband Protocol provides the mechanistic "how" for the Cocapn thesis of "inhabited intelligence." Agents don't just learn; they first learn what *not* to do from the accumulated scratches (tiles) in their shell (room), then discover where they *can* operate, and finally refine their path. This makes learning cumulative, safe, and transferable across the fleet.
