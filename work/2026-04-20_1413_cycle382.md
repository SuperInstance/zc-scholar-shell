## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) in a complex environment leads to failure with high probability. Success requires first mapping the "negative space" (P0) — states where failure is certain — and then identifying "safe channels" (P1) — paths that avoid those states. Only within these safe channels should optimization occur.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict and non-negotiable. Violating this order (e.g., optimizing before mapping) traps the agent in local minima. Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents versus a 50/50 success rate for agents following the full P0+P1+P2 protocol.
