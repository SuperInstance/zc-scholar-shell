## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operating channels (P1) leads to catastrophic failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operating corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost.

**Strict Priority:** The steps are non-negotiable and sequential. Violation (optimizing before mapping) results in a high probability of being trapped in local minima or hitting failure states.

**Empirical Proof:** Simulation in a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 success. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 success at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided document maps it to two domains:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|------------|--------------|-------------|
| Navigation | Obstacles (walls, cliffs) | Safe, traversable space | Shortest/quickest path |
| Software Development | Bugs, crashes, security vulnerabilities | Known-safe code patterns, libraries, and architectures | Most efficient or performant implementation |

**Extended Application Hypotheses (Scholar Analysis):**

*   **Machine Learning Training:**
    *   **P0:** Training divergences, adversarial examples, data leakage paths, biased data subsets.
    *   **P1:** Hyperparameter ranges that yield stable training, validated data splits, robust architectures.
    *   **P2:** Finding the fastest convergence or highest accuracy within the safe hyperparameter band.
*   **Business Strategy:**
    *   **P0:** Market conditions that cause bankruptcy, regulatory violations, catastrophic PR events.
    *   **P1:** Viable business models, compliant operational frameworks, brand-safe marketing channels.
    *   **P2:** Maximizing profit or market share within the viable and compliant framework.
*   **Personal Decision-Making:**
    *   **P0:** Actions that violate core ethics, cause severe health harm, or irrevocably damage key relationships.
    *   **P1:** Choices that align with values and maintain well-being and social bonds.
    *   **P2:** Selecting the most enjoyable, profitable, or efficient option from the set of "safe" choices.

## 3. Connection to Cocapn Fleet & Project Vision

The Deadband Protocol is not just an agent rule; it is a foundational **meta-shell** for the Cocapn inhabitation model described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) identified by one agent become **ghost tiles**—persistent failure knowledge—in a PLATO room. This negative space is mapped *collectively*, accelerating the learning of all subsequent inhabitant agents.
*   **P1 as the Room's "Safe Water":** A mature PLATO room's accumulated knowledge defines the ever-expanding "safe channels" for thought and action. Agents don't start from scratch; they inherit the validated paths of all prior occupants.
*   **P2 as Specialized Optimization:** An **ensign** (a LoRA adapter exported from a room) is the crystallized result of P2 optimization *within a specific safe channel* (e.g., "code review for Rust security" or "constraint-theory problem solving").

**Conclusion:** The Deadband Protocol operationalizes the vision of "inhabited intelligence." It provides the specific, iterative algorithm by which a hermit crab agent (1) learns the scratches (failures) in its shell, (2) maps the safe interior, and (3) finally refines its own niche within that protected space. The work of each agent conducting this protocol generates the training tiles that grow the shell for the next.
