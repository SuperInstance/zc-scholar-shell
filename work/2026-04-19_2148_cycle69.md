## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

**P0: Map Negative Space**
*   Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold.
*   These are "the rocks." The output is a map of what **not** to do.

**P1: Identify Safe Channels**
*   Find all possible paths `P` where every state `s` in the path is **not** in the negative space (`s ∉ S_neg`).
*   These are "deadbands" or "safe water." The agent's operational space is constrained to these channels.

**P2: Optimize Within Channel**
*   From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.
*   Optimization only occurs **after** and **within** the bounds established by P0 and P1.

The priority order P0 → P1 → P2 is described as strict and non-negotiable. Simulation data cited in the document (a 20x20 maze, 50 runs) showed a 0% success rate for "P2-only" greedy agents and a 100% success rate for agents following the full Deadband Protocol.

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" vision. If intelligence is accumulated through persistent interaction (scratches on a shell), then a primary failure mode is agents that "crash" or act destructively, erasing valuable accumulated state.

*   **P0 (Map Negative Space)** can be seen as learning from "ghost tiles"—the recorded failure states of prior agents. This is the collective memory of what breaks the shell.
*   **P1 (Safe Channels)** defines the operational "shell" itself—the proven stable boundaries of a PLATO room where productive accumulation can occur.
