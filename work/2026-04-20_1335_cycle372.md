## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure modes. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These paths define the "deadbands" or safe operational envelopes.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing cost or maximizing reward. Optimization is explicitly constrained to the safe space.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order by optimizing (P2) before mapping (P0+P1) results in agents becoming trapped in local minima or hitting failure states.

**Empirical Support:** The provided simulation data (a 20x20 maze, 50 runs) showed a 0% success rate for a P2-only (greedy) agent, while the full P0+P1+P2 (deadband) agent achieved a 100% success rate at optimal speed.

## 2. Real-World Applications & Abstract Patterns

The protocol is presented as a domain-agnostic pattern. The documentation provides a template for mapping it to various fields:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)                   |
|-----------------|-------------------------------------|---------------------------------|-------------------------------|
| Navigation      | Obstacles, collisions               | Safe water, clear lanes         | Shortest/fastest path         |
| Software Development | Bugs, crashes, security flaws      | Known-safe code patterns, libraries | Most efficient implementation |
| Machine Learning| Overfitting, adversarial examples   | Robust model architectures, validated data splits | Hyperparameter tuning        |
| Business Strategy| Market risks, regulatory pitfalls  | Viable market segments, compliant operations      | Profit maximization          |
| Personal Decision-Making| Known harmful behaviors, health risks | Healthy habits, safe environments | Goal achievement             |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a "shell" that accumulates "scratches" (tiles) from every agent that inhabits it. These accumulated tiles form a persistent memory of interactions, successes, and—critically—**failures**.
*   **P0 as Accumulated Scratches:** The "negative space" (P0) for a new agent entering a room can be inferred from the historical record of failures (ghost tiles, dead agents' lessons) stored within that room. The agent doesn't need to discover all rocks from scratch; it inherits a map of known hazards from the room's memory.
*   **Training as Channel-Finding:** The process of a room training an ensign (specialist model) can be viewed as the room distilling its accumulated experience into a refined map of safe channels (P1) for a specific domain. The ensign is an export of "where you CAN be."
*   **Hermit Crab Analogy:** An agent (the crab) does not grow its constraints (the shell). It finds a suitable, pre-existing shell (a PLATO room with relevant historical knowledge), moves in, and contributes its own scratches, thereby expanding the collective map of negative space and safe channels for future inhabitants.

**Conclusion:** The Deadband Protocol provides the mechanistic rule for how individual agents within the Cocapn ecosystem should operate safely and effectively. The PLATO room system provides the persistent, shared memory that allows each new agent to start with a richer, pre-mapped P0 and P1, dramatically accelerating safe learning and optimization. The protocol ensures that agents contribute usefully to the room's knowledge (by avoiding known failures and discovering new safe paths), which in turn makes the room a more valuable shell for the next agent. This creates a virtuous cycle of inhabited intelligence.
