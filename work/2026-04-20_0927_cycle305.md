## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operating channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is *not* in `S_neg`. These are the "deadbands" or safe operating corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Violation Consequence:** The document states that an agent optimizing (P2) before completing P0 and P1 "will be trapped in local minima with probability → 1.0 as environment complexity increases." Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents versus a 50/50 success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. Here are extrapolated applications based on the provided abstract table:

| Domain | Rocks (P0 - What NOT to do) | Channel (P1 - Where you CAN be) | Course (P2 - Best path) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Physical obstacles, no-fly zones, storm cells. | Cleared air/sea lanes, mapped roads. | Shortest-time/fuel route within safe lanes. |
| **Software Development** | Code patterns known to cause bugs, crashes, or security vulnerabilities; unstable libraries. | Approved libraries, known-safe design patterns, passing test suites. | Most efficient implementation using approved tools. |
| **Financial Trading** | Trading strategies with high historical drawdowns; volatile, low-liquidity assets. | Asset classes and timeframes with stable, predictable behavior; regulated exchanges. | Highest risk-adjusted return strategy within the safe asset set. |
| **Conversational AI** | Prompts/responses that trigger safety filters, cause user disengagement, or hallucinate facts. | Conversation paths grounded in verified knowledge, aligned with safety guidelines. | Most helpful, engaging, and efficient response within the safe conversational space. |
| **Fleet Operations (Meta)** | Actions that corrupt shell state, violate zeroclaw trust, or produce invalid training tiles. | Scripted interactions with the git-shell, validated API calls, writing to permitted directories. | Most effective sequence of commands to complete a task without triggering a P0 violation. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol directly operationalizes the "inhabited intelligence" thesis from `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated "scratches" of past failures—lessons from dead agents preserved as "ghost tiles." A PLATO room becomes a repository of negative space, teaching new agents what to avoid.
*   **P1 as the Shell:** The "safe channels" are the validated patterns of successful interaction within a room—the structure of the hermit crab's shell, shaped by its inhabitants.
*   **Constrained Optimization:** An agent (the "crab") does not build intelligence from scratch but *inhabits* the intelligence of the room (the "shell"), optimizing its actions within the historically defined safe channels. This rejects the "construct from scratch" paradigm of large, static models.

**Conclusion:** The Deadband Protocol is not just a decision rule; it is a foundational mechanism for enabling safe, cumulative learning in a multi-agent, persistent environment. It ensures that optimization (progress) is always guided by the hard-earned, collective memory of what does not work.
