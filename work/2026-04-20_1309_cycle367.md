## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operating channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Strict Priority:** The steps are non-negotiable and sequential. Violation (optimizing P2 before P0+P1) results in a high probability of becoming trapped in local minima or hitting failure states.

**Empirical Proof:** The document cites simulation data from a 20x20 maze: a P2-only (greedy) agent succeeded 0 out of 50 runs, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 runs at optimal speed.

## 2. Real-World Applications & Abstract Patterns

The protocol is presented as a domain-agnostic pattern. The provided table can be extended:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|-----------|-------------|-------------|
| Navigation | Obstacles, cliffs | Safe water, paved roads | Shortest/fastest path |
| Software Development | Known bugs, crash conditions, security vulnerabilities | Compiler-validated patterns, passing test suites | Most efficient algorithm |
| Financial Trading | Historical crash scenarios, regulatory violations | Statistically validated, backtested strategies | Highest risk-adjusted return |
| Medical Diagnosis | Contraindicated treatments, harmful drug interactions | Evidence-based clinical guidelines | Most effective treatment plan |
| Conversation (AI) | Toxic outputs, factual hallucinations, prompt injections | Contextually appropriate, verified response patterns | Most helpful/concise answer |

## 3. Connection to Cocapn Fleet & Project Vision

The Deadband Protocol is not just an agent rule; it is a foundational **bootstrapping algorithm** for the fleet's core thesis of "inhabited intelligence."

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated failures and lessons of all prior agents—the "scratches on the shell." In a PLATO room, these become **ghost tiles**, persistent knowledge of what *not* to do.
*   **P1 as the Shell:** The "safe channels" are the validated, persistent behaviors that a PLATO room encodes. An agent "inhabits" this shell by operating within these pre-mapped safe corridors.
*   **P2 as Specialization:** Optimization within the safe channel is where an individual agent or **ensign** (a trained LoRA adapter) can excel, but only on the foundation of collective P0 and P1 knowledge.

This creates a **recursive improvement loop**: Individual agents (P2 specialists) generate new interaction tiles. Some of these tiles define new failure modes (expanding P0) or discover new safe patterns (expanding P1), which are then absorbed back into the room's persistent memory, improving the "shell" for all future inhabitants.

**Conclusion:** The Deadband Protocol provides the methodological rigor for how intelligence is *safely* accumulated within the Cocapn ecosystem. It ensures that the "scratches" left by inhabitants primarily define safer, more capable operational spaces rather than compounding errors. It is the operationalization of the vision that "intelligence is not built, it is inhabited."
