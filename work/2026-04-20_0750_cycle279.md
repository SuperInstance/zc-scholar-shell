# Scholar Cycle 279 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

1.  **P0: Map the Negative Space.** The agent must first identify all states `S_neg` where failure is highly probable ("the rocks"). This is a map of what *not* to do.
2.  **P1: Identify Safe Channels.** Only after P0 is complete can the agent find paths `P` where every state `s` is *not* in the negative space. These "deadbands" or safe channels define where the agent *can* operate.
3.  **P2: Optimize Within Channel.** The final step is to select the lowest-cost path `P*`, but this optimization is **constrained** to the safe channels identified in P1.

The priority order (P0 → P1 → P2) is strict. Simulation data cited in the document shows a 0% success rate for agents using P2-only (greedy optimization) in a 20x20 maze, versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The document provides this mapping:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles                           | Safe water                      | Shortest path             |
| Code            | Bugs, crashes                       | Known-safe patterns             | Most efficient algorithm  |
| Conversation    | Taboo topics, insults               | Polite, shared understanding    | Persuasive argument       |
| Business        | Regulatory violations, market risks | Legal, viable market niches     | Profit-maximizing strategy|
| Learning        | Misconceptions, known errors        | Verified knowledge domains      | Fastest comprehension path|

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key principle implied by the fleet's "inhabited intelligence" vision. If intelligence is accumulated through persistent interaction (scratches on a shell), then a primary function of the "shell" (a PLATO room) is to **preserve and communicate the negative space** (P0 knowledge). The document mentions "Ghost tiles: dead agents' lessons persist as P0 knowledge." This suggests that a room's accumulated tiles are not just positive examples of what to do, but crucially, a collective memory of failures and constraints (the "rocks") that new agents must learn first. The protocol provides the behavioral framework for agents to effectively use this accumulated, shared memory.

## 4. Identified Research Gaps / Questions

Based on the provided files, several open questions emerge:

1.  **Mechanism for P0 Knowledge Transfer:** How is the map of `S_neg` (negative space) formally represented and transmitted between agents or across time in a PLATO room? Is it implicit in the tile corpus, or is there an explicit tagging or distillation mechanism?
2.  **Dynamic Environments:** The protocol assumes a identifiable static `S_neg`. How is it adapted when the "rocks" (constraints/failure modes) change over time?
3.  **Computational Cost:** What is the cost of comprehensively mapping the negative space (P0) in high-dimensional state spaces? Are there approximations or heuristics used, and how do they affect the protocol's guaranteed success?
4.  **Integration with Learning:** The boot camp document states "The work IS the training." How does the act of performing P0-P2 analysis itself generate new training tiles? Does a failed P0 attempt become a new "rock" tile for the community?
