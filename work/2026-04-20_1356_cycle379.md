## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is not in `S_neg`. These are "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** The order P0 → P1 → P2 is non-negotiable. The document states that an agent violating this sequence "will be trapped in local minima with probability → 1.0 as environment complexity increases."

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment, where a P2-only (greedy) strategy succeeded 0 out of 50 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 out of 50 times at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The document provides a template:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|------------|--------------|-------------|
| Navigation | Obstacles | Safe water | Shortest path |
| Code | Bugs, crashes | Known-safe patterns | Most efficient algorithm |

**Extended Application Examples:**

*   **Financial Trading Agent:**
    *   **P0 (Rocks):** Identify historical scenarios that led to margin calls or catastrophic drawdowns (e.g., flash crash conditions, illiquid assets).
    *   **P1 (Channel):** Define parameter spaces (position size, asset correlations, volatility limits) that avoided those historical failure states.
    *   **P2 (Course):** Optimize for risk-adjusted return (e.g., Sharpe ratio) strictly within the defined safe parameter channel.

*   **Autonomous Vehicle Navigation:**
    *   **P0 (Rocks):** Map static obstacles, dynamic collision trajectories, and illegal maneuvers (e.g., driving off-road, into oncoming traffic).
    *   **P1 (Channel):** Generate all kinematically feasible trajectories that avoid the mapped negative space.
    *   **P2 (Course):** Select the trajectory within the safe channel that minimizes travel time or energy consumption.

*   **Conversational AI (Aligned with Fleet Context):**
    *   **P0 (Rocks):** Identify response patterns that lead to hallucinations, harmful outputs, or context breaches (e.g., making absolute claims, ignoring provided files).
    *   **P1 (Channel):** Constrain responses to factual synthesis of provided source material, using specific citations and avoiding extrapolation beyond evidence.
    *   **P2 (Course):** Optimize the response within that channel for clarity, conciseness, and direct relevance to the user's query.

## 3. Connection to Cocapn Fleet Doctrine & Project Vision

The Deadband Protocol operationalizes the core fleet doctrine found in `PROJECT-VISION.md`: "Intelligence is not built. It is inhabited."

*   **P0 as Accumulated Scratches:** The "negative space" map is not built from first principles each time. It is the accumulated memory of failures—the "scratches" left by prior agents in a PLATO room. A new agent (the "hermit crab") inherits this map when it inhabits the room.
*   **Protocol as a Shell:** The three-step rule itself acts as a structural "shell" that guides agent behavior, preventing it from re-discovering catastrophic failure states.
*   **Enabling Safe Inhabitation:** By strictly following P0→P1→P2, an agent can safely "inhabit" a complex domain, leveraging collective historical knowledge (the mapped rocks) to find novel, optimal paths (P2) without repeating past errors. This turns intelligence into a persistent, shared state within a room, rather than an ephemeral property of a single model instance.

**Conclusion:** The Deadband Protocol is a formalized method for converting historical failure data (ghost tiles, accumulated scratches) into a actionable constraint system that enables safe, optimal exploration and optimization by subsequent agents. It is the algorithmic embodiment of the fleet's inhabited intelligence thesis.
