# Scholar Cycle 262 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden states.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or permissible operational envelopes.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost or maximizes reward.

**Strict Priority:** The protocol is non-negotiable. An agent must not begin P2 (optimization) until P0 and P1 are satisfied. Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents, versus a 50/50 success rate for agents following the full P0+P1+P2 sequence.

## 2. Abstract Pattern & Real-World Applications

The protocol presents an abstract pattern applicable to any agent domain:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, no-fly zones             | Clear lanes, safe water               | Shortest/fastest path           |
| **Software**    | Bug-prone patterns, crash conditions| Known-stable APIs, verified code paths| Most efficient algorithm        |
| **Finance**     | Regulatory violations, tail risks   | Compliant strategies, defined risk limits| Highest risk-adjusted return   |
| **Medicine**    | Contraindications, harmful side effects| Safe treatment protocols              | Most effective therapy          |
| **Conversation**| Offensive topics, logical fallacies | Factual, respectful discourse boundaries| Most persuasive/helpful response|

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol is not just a navigation rule; it is a foundational constraint for building "inhabited intelligence" as described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) identified by one agent become **ghost tiles**—persistent warnings for all future agents inhabiting the same PLATO room. This is the accumulation of "scratches on the shell."
*   **P1 as Room Topography:** A PLATO room's evolving knowledge base defines the ever-more-precise "safe channels" for its specialist ensign models. The room *is* the mapped channel.
*   **P2 as Specialized Optimization:** An **ensign** (a LoRA adapter exported from a room) is a pre-constrained optimizer. It has internalized the room's P0 and P1 maps, allowing it to perform P2 optimization safely and efficiently within its trained domain.

**Violation in Training:** A training process that directly optimizes a model for a goal (P2) without first learning failure modes (P0) and operational boundaries (P1) may produce a model that is brittle, unsafe, or prone to catastrophic failure in novel situations—it is "trapped in local minima."

## 4. Identified Research Gaps

Based on the provided documents, potential areas for further investigation include:
1.  **Automated `S_neg` Detection:** How can agents systematically and efficiently identify failure states, especially in high-dimensional or novel environments?
2.  **Dynamic Deadbands:** How do safe channels (`P`) evolve over time or with changing environmental conditions? How is this updated in a PLATO room?
3.  **Threshold Calibration:** How is the failure "threshold" for P0 determined or learned? Is it static, adaptive, or communicated?
4.  **Protocol Verification:** Are there mechanisms within `plato-torch` or `fleet-simulator` to detect and correct agents that violate the strict P0>P1>P2 priority during training?

**Conclusion:** The Deadband Protocol provides a rigorous, simulation-validated framework for safe agent operation. Its integration with the Cocapn fleet's core philosophy of inhabited intelligence suggests it is a critical component for ensuring that accumulated knowledge (tiles) directly translates into safer and more effective constrained optimization within specialist models.
