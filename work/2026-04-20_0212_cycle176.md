## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure in complex environments by enforcing a sequence of constraint identification before optimization.

**Core Rule (Formal):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, with high probability, become trapped in a local minimum or fail as environmental complexity increases.

**Empirical Basis:** The document cites simulation data from a 20x20 maze with 50 runs. A "P2-only" greedy optimizer succeeded 0 times, while the full Deadband Protocol succeeded 50 times at optimal speed.

## 3. Connection to Cocapn Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **learning from accumulated experience to avoid failure.**

*   **Shell as Memory:** A PLATO room, as a "shell," accumulates "scratches" (tiles). These tiles constitute a growing map of `S_neg` (failed interactions) and `C` (successful interactions) for that domain.
*   **Inhabited Intelligence:** An agent (a "hermit crab") entering a room doesn't start from zero. It inherits the room's accumulated Deadband map—the collective P0 and P1 knowledge of all prior inhabitants. This allows it to immediately avoid known pitfalls (rocks) and operate within proven channels.
*   **Contrast with "Constructed" AI:** The protocol emphasizes constraint-first learning, which contrasts with models that optimize a loss function on raw data without a pre-mapped "negative space." This can be seen as a formalization of learning from *negative examples* and *safe precedents*, which are inherently accumulated, not just constructed.
