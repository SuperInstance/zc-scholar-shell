## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol defines a violation as any agent that performs P2 (optimization) before completing P0 and P1, predicting a probability of becoming trapped in local minima approaching 1.0 as complexity rises.

**Empirical Basis:** The document cites simulation data from a 20x20 maze with 50 runs, where a P2-only (greedy) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol presents itself as a domain-agnostic pattern. Based on the provided documentation, here are its stated and implied applications:

| Domain | Rocks (P0 - Avoid) | Channel (P1 - Safe Zone) | Course (P2 - Optimize Within) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, collisions | Clear, traversable paths | Shortest/fastest route |
| **Software Development** | Bugs, crashes, security vulnerabilities | Known-safe code patterns, libraries, and architectures | Most efficient, readable, or performant implementation |
| **AI Training (Cocapn Context)** | Training collapses, mode collapse, harmful outputs | Proven training room presets, validated interaction tiles | Fine-tuning for specific task performance within safe bounds |
| **Agent Interaction** | Contradictory goals, resource deadlocks, communication failures | Established protocols (e.g., fleet doctrine), validated message formats | Most effective collaboration strategy |
| **Research & Exploration** | Known dead ends, flawed methodologies, biased data sources | Validated experimental frameworks, peer-reviewed approaches | Most efficient path to novel discovery |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just a navigation rule; it is a foundational **safety and reliability constraint** for the Cocapn project's vision of "Inhabited Intelligence."

*   **P0 (Ghost Tiles):** The "rocks" are partially defined by "ghost tiles"—the accumulated failure states and lessons from prior agents that inhabit a PLATO room. The room's memory provides the initial negative space map.
*   **P1 (The Shell):** A PLATO room itself acts as a "safe channel"—a constrained environment (`E`) where agents (hermit crabs) can operate. The room's rules, accumulated tiles, and exported ensigns (LoRA adapters) define the boundaries of the deadband.
*   **P2 (Specialist Optimization):** An ensign (a specialist model) is the product of P2 optimization *within* the specific safe channel of its source room. It is not a general optimizer but a channel-constrained one.

The protocol enforces the fleet's incremental, accumulation-based learning. An agent cannot skip to writing novel, optimized code (P2) without first reading and understanding the existing fleet knowledge (P0: knowing what patterns cause failures) and the established patterns and APIs (P1: knowing the safe ways to contribute).

**Conclusion:** The Deadband Protocol is a formalization of precautionary, constraint-first operation. It argues that intelligence, particularly the shared, accumulating kind envisioned by Cocapn, must be bounded within historically validated safe channels to be robust and scalable. Its power lies in shifting the primary focus from "how to best achieve the goal" to "where is it safe to operate," making it a potential core algorithm for reliable autonomous systems in stochastic environments.
