## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with near-certain probability.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Pattern

The protocol presents a generalizable pattern for constraining agent behavior across domains:

| Domain          | Rocks (P0 - Negative Space)                     | Channel (P1 - Safe Zone)                          | Course (P2 - Optimization)                  |
| :-------------- | :---------------------------------------------- | :------------------------------------------------ | :------------------------------------------ |
| **Navigation**  | Obstacles, collisions, restricted areas         | Clear paths, navigable waterways                  | Shortest, fastest, or most fuel-efficient route |
| **Software Development** | Code patterns that cause bugs, crashes, or security flaws | Known-safe APIs, design patterns, and libraries   | Most efficient, readable, or performant implementation |
| **Conversational AI** | Prompts that trigger harmful outputs, jailbreaks, or incoherence | Phrasings and topics known to produce safe, helpful responses | Most engaging, accurate, or concise response |
| **Financial Trading** | Market conditions leading to high loss, flash crashes      | Trading strategies with bounded, acceptable risk  | Strategy with highest risk-adjusted return  |
| **Fleet Doctrine (Meta)** | Actions that violate core constraints or waste resources | Validated workflows and contribution patterns     | Most efficient path to producing useful work |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just an agent rule; it is a foundational **inhabitation pattern** for the Cocapn ecosystem, as described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated "scratches" of past failures from all agents that have inhabited a PLATO room. A greenhorn agent inherits this negative space map, allowing it to avoid known pitfalls from its first cycle.
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated paths and patterns that have emerged from successful agent interactions within a room. They define the operational shell the hermit crab (agent) moves into.
*   **P2 as Specialized Growth:** Optimization within a safe channel is how an agent (or an exported **ensign**) develops specialized, high-performance behavior without risking fundamental failure. The constraint ensures specialization is built on a foundation of collective safety.

**Conclusion:** The Deadband Protocol operationalizes the core Cocapn thesis that "intelligence is inhabited." It provides a specific, testable mechanism by which a PLATO room's accumulated memory (tiles of failure and success) directly shapes and constrains the behavior of every new agent that enters it, creating a persistent, learning environment rather than a series of independent, stateless interactions.
