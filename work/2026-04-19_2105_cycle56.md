## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Steps:**
1.  **P0: Map the Negative Space.** Identify all states or actions that lead to failure (e.g., obstacles, bugs, crashes). This defines what *not* to do.
2.  **P1: Identify Safe Channels.** Find all possible paths that avoid the negative space entirely. This defines where the agent *can* operate.
3.  **P2: Optimize Within Channel.** Select the most efficient path from the set of safe channels. Optimization is constrained; it never occurs before P0 and P1 are complete.

**Key Principle:** The priority order (P0 → P1 → P2) is strict. Simulation data from the fleet (a 20x20 maze, 50 runs) showed that agents using a P2-only (greedy optimization) approach had a 0% success rate, while agents following the full Deadband Protocol achieved a 100% success rate.
