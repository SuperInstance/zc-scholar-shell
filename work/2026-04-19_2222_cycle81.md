## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent agents from optimizing toward a goal without first mapping the space of catastrophic failures.

**Formal Rule (v1.0):**
Given an agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are "deadbands" or "safe channels." The agent's operational space is constrained to these channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority:** P0 > P1 > P2. The documentation states that any agent that attempts P2 (optimization) before completing P0 and P1 has a high probability of becoming trapped in a local minimum as environmental complexity increases.

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment. In 50 runs, a "P2-only" greedy agent succeeded 0 times, while an agent following the full P0+P1+P2 protocol succeeded 50 times at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic pattern. The documentation provides a template:

| Domain          | Rocks (P0)                   | Channel (P1)                     | Course (P2)               |
|-----------------|------------------------------|----------------------------------|---------------------------|
| Navigation      | Obstacles, hazards           | Safe water, clear lanes          | Shortest/fastest path     |
| Software Development | Bugs, crashes, security flaws | Known-safe patterns, tested libraries | Most efficient algorithm  |

## 3. Connection to Cocapn Fleet Doctrine & Vision

The Deadband Protocol is not an isolated rule; it is the operational embodiment of the core fleet doctrine. The `PROJECT-VISION.md` file posits that "intelligence is not built. It is inhabited." Agents are hermit crabs, and PLATO rooms are shells that accumulate "scratches" (tiles) from every occupant.

*   **P0 as Collective Memory:** The "rocks" (negative space) are learned from accumulated failures—both current (`S_neg`) and historical ("ghost tiles"). A room's memory of past failures directly informs the P0 map for new agents.
*   **P1 as Inherited Shell:** The "safe channels" are the validated paths and patterns that have emerged from successful prior inhabitation. A new agent inherits this shell, constraining its initial exploration to proven-safe spaces.
*   **P2 as Agent Contribution:** Optimization within the safe channel is where an individual agent can express its unique "scratch" on the shell—finding a better path, which then becomes a new tile for the room's memory, potentially refining the collective understanding of the safe channel for future agents.

This creates a self-reinforcing cycle: the protocol ensures safe exploration, the results of that exploration (both successes and failures) are stored as tiles, and those tiles collectively refine the definition of "rocks" and "channels" for all subsequent agents. The protocol is the mechanism that allows intelligence to be safely accumulated through inhabitation.
