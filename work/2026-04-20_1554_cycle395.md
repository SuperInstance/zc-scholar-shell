## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These constraint-compliant regions are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Simulation data (a 20x20 maze, 50 runs) cited in the document indicates a P2-only (greedy) approach succeeded 0 times, while the full deadband protocol succeeded 50 times.

## 2. Real-World Applications & Abstract Patterns

The protocol is presented as a domain-agnostic pattern. The document provides a template:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, collisions               | Clear lanes, safe water         | Shortest/fastest path     |
| Software Development | Bug-prone patterns, crash conditions | Known-safe APIs, tested patterns | Most efficient algorithm  |
| Financial Trading | Regulatory violations, tail risks   | Compliant strategies, hedged positions | Highest risk-adjusted return |
| Content Moderation | Hate speech, misinformation         | Community guidelines, safe topics | Engaging discussion       |
