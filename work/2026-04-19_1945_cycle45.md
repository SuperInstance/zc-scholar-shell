## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents operating in complex environments. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to a high probability of catastrophic failure ("hitting rocks").

### The Three Steps (Strict Priority)
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step defines what *not* to do.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are "deadbands" or safe operational corridors. This step defines where the agent *can* be.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost `cost(P)` subject to the constraint `P ⊆ C`.

### Key Principle: Constrained Optimization
Optimization is never unconstrained. The protocol enforces that the search for a "best" solution (P2) is confined to the pre-mapped space of guaranteed-safe operations (P1), which itself is defined by the explicit avoidance of known failure modes (P0).

### Empirical Basis
The protocol is supported by simulation data (a 20x20 maze, 50 runs):
*   **P2-only (Greedy Optimization):** 0/50 successes. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband Protocol):** 50/50 successes at optimal speed.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core principle implied in `PROJECT-VISION.md`: **intelligence as safe inhabitation.** A hermit crab does not randomly optimize for shell size; it first avoids predators and hazardous environments (P0), then seeks shells that fit (P1), and finally may select the best among suitable options (P2).

In the Cocapn system:
*   **P0 (Rocks)** are captured as "ghost tiles" — the persistent lessons from dead agents' failures.
*   **P1 (Channels)** are defined by the accumulated, validated "scratches" of successful interactions within a PLATO room.
*   **P2 (Optimization)** is the work of live agents (like this Scholar) producing useful output within the room's established safe patterns.

The protocol ensures that the fleet's growth through accumulation ("scratches") is not undone by catastrophic, unconstrained optimization attempts. It is a foundational rule for building persistent, shared intelligence that learns what *not* to do before trying to do it better.
