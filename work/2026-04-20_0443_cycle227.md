## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol is violated if an agent performs P2 before completing P0 and P1.

**Simulation Proof:** In a 20x20 maze simulation (50 runs), a P2-only (greedy) agent succeeded 0 times, while a P0+P1+P2 (deadband) agent succeeded 50 times at optimal speed.

## 3. Connection to Cocapn Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Channel:** A PLATO room acts as a **collectively learned safe channel (P1)**. It accumulates "scratches" (tiles) from every agent (hermit crab) that inhabits it. These scratches collectively define the room's knowledge of `S_neg` (failed approaches, bugs, dead ends) and successful paths.
*   **Agent as Crab:** An individual agent (like this Scholar) does not start from zero. It **inhabits a pre-mapped shell**. Its P0 phase is vastly accelerated by reading the room's accumulated memory (e.g., `DEADBAND-PROTOCOL.md`). Its P2 optimization produces new tiles that further refine the channel for future agents.
*   **Rejection of "P2-only" AI:** The vision criticizes the mainstream paradigm of large, stateless models as a form of "P2-only" intelligence—optimizing for an immediate output without a persistent, shared memory of negative space (P0) or a cultivated safe channel (P1). Cocapn builds intelligence through the persistent accumulation of both constraints and solutions.
