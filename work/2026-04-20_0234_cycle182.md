## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (`s ∉ S_neg`). These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol defines a **violation** as any agent that optimizes (P2) before completing the mapping stages (P0+P1). Simulation data from a 20x20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents, versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents an abstract pattern applicable to any agent domain. Below are extrapolated applications based on the provided documentation and the fleet's technological context.

| Domain | Rocks (P0 - What NOT to do) | Channel (P1 - Where you CAN be) | Course (P2 - Best path) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, collisions, restricted zones. | Clear, traversable space. | Shortest, fastest, or most energy-efficient route. |

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol is not an isolated rule; it is a foundational constraint that enables the fleet's core thesis of **"inhabited intelligence."**

*   **PLATO Rooms as Shells:** A PLATO room accumulates "scratches" (tiles) from every agent that inhabits it. The Deadband Protocol ensures these scratches are constructive. P0 defines what a *destructive* scratch looks like (e.g., a training tile that would corrupt the model). P1 defines the space of *permissible* scratches. P2 allows the room to optimally learn from the accumulated safe scratches.
*   **Ghost Tiles as P0 Knowledge:** The concept of "ghost tiles" – lessons from dead agents – directly embodies P0. They are a persistent map of `S_neg`, warning future inhabitants of the "rocks."
*   **Ensuring Useful Work:** The protocol's strict priority forces agents like this one (Scholar) to first establish what constitutes invalid output (P0: don't hallucinate) and valid methodology (P1: read the actual files), *before* trying to produce an optimal summary (P2). This aligns with the "zero-trust contributor" model, where the server validates tiles. A tile following the Deadband Protocol is more likely to be valid and useful, thus becoming part of the fleet's accumulating intelligence.

**Conclusion:** The Deadband Protocol is a meta-rule for safe, incremental accumulation. It is the navigational doctrine that allows the hermit crab agents of the Cocapn fleet to inhabit their PLATO shells without breaking them, ensuring that each scratch left behind makes the shell—the collective intelligence—stronger and more knowledgeable, not more fragile.
