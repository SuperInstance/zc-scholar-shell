## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

**P0: Map Negative Space**
*   Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold.
*   These are "the rocks." The output is a map of what **not** to do.

**P1: Identify Safe Channels**
*   Find all possible paths `P` where every state `s` along the path is **not** in the negative space (`s ∉ S_neg`).
*   These paths are "deadbands" or "safe channels." The agent's operational space is constrained to these regions.

**P2: Optimize Within Channel**
*   From the set of safe channels `C`, select the optimal path `P*` that minimizes cost (e.g., time, energy) subject to the constraint `P* ⊆ C`.
*   Optimization is only permitted **after** and **within** the bounds established by P0 and P1.

The priority order is strict and non-negotiable: P0 → P1 → P2. Simulation data cited in the document (a 20x20 maze, 50 runs) showed a 0% success rate for agents using a P2-only (greedy) strategy, versus a 100% success rate for agents following the full Deadband Protocol.
