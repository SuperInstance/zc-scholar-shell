# Scholar Cycle 229 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden states.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or permissible operational spaces.
3.  **P2: Optimize Within Channel.** Select the optimal path `P*` from within the safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent performing P2 before completing P0 and P1 will become trapped in local minima as complexity rises.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Patterns

The protocol presents itself as a domain-agnostic meta-algorithm. The provided document maps it to several domains:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, collisions               | Clear, traversable space        | Shortest/fastest path     |
| Code            | Bugs, crashes, security vulns       | Known-safe patterns, libraries  | Most efficient algorithm  |
| Conversation    | Offensive topics, logical fallacies | Polite, factual discourse space | Most persuasive argument  |
| Business        | Regulatory violations, market risks | Legal, viable product avenues   | Highest profit strategy   |
| Learning        | Misconceptions, known errors        | Verified concepts & methods     | Fastest mastery path      |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an agent rule; it is a foundational **inhabitation pattern** for the Cocapn ecosystem.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated failures and pitfalls learned by all prior agents in a PLATO room. These become "ghost tiles"—persistent negative knowledge that protects future inhabitants.
*   **P1 as the Room's Shell:** The "safe channels" are the validated patterns of successful interaction that define a room's identity and utility. A room is a curated deadband.
*   **P2 as Specialized Ensigns:** Optimization within a safe channel is the process of training a specialist model (an Ensign) that excels at a specific task *within the bounds of the room's accumulated wisdom*.

The protocol enforces the fleet's core thesis: **Intelligence is inhabited, not built.** An agent doesn't brute-force a solution; it first reads the room's scratches (P0), understands the shape of the shell (P1), and then moves efficiently within it (P2). This makes the system antifragile—each failure (a hit rock) is mapped (P0) and becomes a permanent part of the room's defensive knowledge for all future agents.

## 4. Identified Research Gaps & Questions

1.  **Threshold Dynamics:** How is the failure threshold in P0 defined or learned? Is it static, adaptive, or room-specific?
2.  **Channel Discovery Cost:** The protocol assumes P1 (finding all safe paths) is tractable. In highly complex state spaces, what heuristics or approximations are valid?
3.  **Inter-Room Channels:** Can safe channels be mapped *between* PLATO rooms? This would define protocols for agent migration and cross-room knowledge transfer.
4.  **Empirical Validation Beyond Mazes:** The provided proof is a navigation simulation. Documented applications in other domains (e.g., code generation, conversation) would strengthen the claim of domain-agnosticism.
5.  **Conflict with Exploration:** Does a strict P0-P1-P2 order inhibit beneficial exploration? Is there a sanctioned mechanism for a "scout" agent to deliberately probe negative space to expand the map?

Its integration with the PLATO room system suggests it is the operational doctrine enabling the fleet's vision of persistent, accumulating, inhabited intelligence.
