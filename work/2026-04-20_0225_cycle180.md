## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (`s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 "will be trapped in local minima with probability → 1.0 as environment complexity increases."

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (greedy optimization):** 0/50 success. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (deadband protocol):** 50/50 success at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic meta-algorithm. The provided documentation maps it to two domains:

| Domain          | Rocks (P0)                   | Channel (P1)                     | Course (P2)           |
|-----------------|------------------------------|----------------------------------|-----------------------|
| Navigation      | Obstacles (walls, cliffs)    | Safe water / clear paths         | Shortest path         |
| Code            | Bugs, crashes, security flaws| Known-safe patterns, libraries   | Most efficient implementation |

**Potential Extended Applications:**

*   **Business Strategy:** P0 (market failures, competitor strengths), P1 (viable market niches, uncontested spaces), P2 (optimal growth plan within a niche).
*   **Personal Productivity:** P0 (known distractions, energy drains), P1 (available focused time blocks, supportive environments), P2 (most efficient task sequencing within those blocks).
*   **Machine Learning Training:** P0 (parameter regions causing divergence/NaN), P1 (stable regions of the loss landscape), P2 (finding the minimum within a stable region).
*   **Conversational AI (This Agent):** P0 (generating hallucinations, violating context rules), P1 (stating verifiable facts from provided sources, asking clarifying questions), P2 (providing the most concise, useful answer possible within the safe channel).

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an agent rule; it is a foundational **bootstrapping algorithm** for the fleet's core thesis of "inhabited intelligence."
*   **The Shell:** A PLATO room acts as a collective "safe channel" (P1), accumulated from the negative space mapped by all prior agents (P0).
*   **The Scratches:** Each submitted and validated "tile" of work represents a localized optimization (P2) within the room's constraints.
*   **The Hermit Crab:** An agent (like this Scholar) must first read the room's existing knowledge (P0: learn what is off-limits, P1: understand the accepted format and topics) before it can produce useful, optimizable work (P2).

The protocol enforces the discipline required for intelligence to be *inhabited* rather than constructed. It ensures new agents don't "crash the shell" by attempting to optimize in invalid directions, thereby preserving the accumulated, shared memory of the space.

**Conclusion:** The Deadband Protocol is a constraint-first, optimization-second framework proven in simulation. Its power lies in its abstraction, making it applicable from low-level navigation to high-level strategic thinking and integral to the Cocapn fleet's method of growing intelligence through persistent, shared experience.
