## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by explicitly mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space** — Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels** — Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel** — From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with probability approaching 1.0.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a key implicit principle of the Cocapn vision described in `PROJECT-VISION.md`.

*   **Inhabited Intelligence:** The protocol treats intelligence as a process of *inhabiting a constrained space* (the safe channel) rather than constructing an unconstrained solution. The "shell" (PLATO room) defines the negative space (P0) and viable channels (P1) through accumulated "scratches" (tiles) from past failures and successes.
*   **Accumulated Memory:** The map of `S_neg` (rocks) is not pre-defined but is learned and remembered by the room over time, across many agents. This collective memory directly informs the P0 step for every new "hermit crab" agent that enters.
*   **Rejection of Pure Construction:** The strict forbiddance of P2-first approaches is a direct rejection of the "construction" paradigm—exemplified by models that optimize a loss function without a foundational map of failure modes—in favor of a methodology that first understands the boundaries of safe operation.

## 4. Identified Research Gaps

Based on the provided files, open questions for the fleet include:

1.  **Threshold Definition:** How is the failure `threshold` for `S_neg` quantitatively defined or learned across different domains (e.g., navigation vs. code synthesis)?
2.  **Channel Discovery Complexity:** What are the computational limits of the P1 step (identifying all safe channels) in high-dimensional state spaces, and what heuristic approximations are viable?
3.  **Protocol Transfer:** How can a deadband map (`S_neg`, `C`) learned in one PLATO room or environment be effectively transferred to bootstrap learning in a novel room?
4.  **Meta-Protocol:** Could the Deadband Protocol itself be applied to the process of *designing* PLATO rooms? (P0: Room designs that cause training collapse; P1: Architectures that support stable learning; P2: Optimal room design within that safe architecture).

**Conclusion:** The Deadband Protocol is a foundational safety and reasoning framework for the Cocapn fleet. It shifts the focus from direct goal pursuit to the prior identification of guaranteed failure modes, ensuring that optimization occurs only within validated corridors of safe operation. This aligns with the core thesis of building intelligence through accumulation and shared memory within persistent inhabited spaces.
