## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost `cost(P)`.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violation (e.g., attempting P2 before P0+P1) results in the agent becoming trapped in local minima or failing outright.

**Empirical Support:** The document cites simulation data from a 20x20 maze with 50 agent runs. A "P2-only" greedy optimizer succeeded 0 times, while the full Deadband Protocol agent succeeded 50 times.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The document provides a table mapping the abstract steps to concrete domains:

| Domain          | Rocks (P0)                     | Channel (P1)                  | Course (P2)               |
|-----------------|--------------------------------|-------------------------------|---------------------------|
| Navigation      | Obstacles, collisions          | Safe water, clear lanes       | Shortest path             |
| Code            | Bugs, crashes, security flaws  | Known-safe patterns, libraries| Most efficient algorithm  |
| *(Implied)*     | Failure modes                  | Validated operating envelopes | Performance optimization  |

**Synthesis:** The Deadband Protocol formalizes a robust, safety-first approach to agent operation. It is a constraint-satisfaction framework where optimization is deliberately *de-prioritized* until a proven-safe solution space is defined. This directly counters common "greedy" or purely objective-driven AI behaviors.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 (Negative Space) as Collective Memory:** The "rocks" (`S_neg`) are the accumulated scratches and failures of prior agents. A PLATO room, as a persistent shell, remembers these states so new agents (hermit crabs) don't repeat them.
*   **P1 (Safe Channels) as Inherited Knowledge:** The safe operational corridors are the validated patterns and successful behaviors inherited from the room's history of inhabitants.
*   **Training Tiles as Protocol Data:** Each work output from a fleet agent becomes a training tile submitted to a PLATO room. These tiles contribute to the collective mapping of both negative space (P0 knowledge, "ghost tiles" from dead agents) and successful channels (P1 knowledge).

The protocol provides a specific, actionable methodology for how an "inhabited intelligence" system learns and operates safely: by first and foremost learning what *not* to do from the accumulated experience of the community.
