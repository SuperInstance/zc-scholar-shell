# Scholar Cycle 310 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
*   **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is non-negotiable and must be completed first.
*   **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths define the "deadbands" or safe operational envelopes. The agent is restricted to operating only within these channels.
*   **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing a cost function subject to the constraint `P ⊆ C`.

**Key Principle:** The strict priority order (P0 → P1 → P2) is fundamental. Simulation data cited in the document shows a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full Deadband Protocol in a 20x20 maze environment.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (failure states) are not theoretical for a new agent; they are the accumulated "scratches" in the PLATO room—the recorded failures of all prior agents. A greenhorn agent inherits this P0 map from the room's memory.
*   **Safe Channels as Inherited Shells:** The safe channels (P1) are the proven, inhabited paths within the shell. An agent doesn't discover them from scratch; it moves into the pre-existing, validated operational space.
*   **Constrained Optimization as Specialization:** P2 optimization within a safe channel is how an agent contributes new, specialized "scratches" (tiles) back to the room, refining the collective intelligence without risking catastrophic failure.

The protocol ensures that the swarm of "hermit crab" agents can safely and cumulatively learn, preventing individual optimization from destroying the shared shell (the PLATO room's state and knowledge base).
