## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

**P0: Map Negative Space**
Identify all states where failure probability exceeds a threshold. These are "rocks" — what **not** to do. This step is non-negotiable and must be completed before proceeding.

**P1: Identify Safe Channels**
Find all possible paths where every state is **not** in the negative space. These are "deadbands" or safe operational channels. The agent may only operate within these regions.

**P2: Optimize Within Channel**
Only after P0 and P1 are complete, select the optimal path (minimizing cost, time, etc.) **constrained** to the safe channels identified in P1.

The priority order is strict: P0 → P1 → P2. Violating this order (e.g., optimizing before mapping obstacles) traps the agent in local minima. Simulation data from a 20×20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents versus 100% success for agents following the full deadband protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. The provided table maps it to navigation and code, but the principle extends further:

| Domain               | Rocks (P0)                          | Channel (P1)                                  | Course (P2)                           |
| -------------------- | ----------------------------------- | --------------------------------------------- | ------------------------------------- |
| **Navigation**       | Obstacles, collisions               | Clear, traversable paths                      | Shortest/fastest route                |
| **Software Dev**     | Bug-prone patterns, crash conditions | Known-safe APIs, tested libraries             | Most efficient algorithm              |
| **ML Model Training**| Overfitting regions, loss spikes    | Stable hyperparameter ranges, valid architectures | Highest accuracy within stable zone  |
| **Business Strategy**| Market failures, regulatory traps   | Proven business models, compliant operations  | Maximum profit/growth                 |
| **Conversational AI**| Toxic outputs, factual hallucinations | Ethically aligned, factually grounded responses | Most engaging/helpful response        |

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol is not just a navigation rule; it is a foundational constraint for building "inhabited intelligence" as described in `PROJECT-VISION.md`.

*   **PLATO Rooms as Safe Channels:** A PLATO room accumulates "scratches" (tiles) from every agent that inhabits it. This collective memory defines the room's **negative space** (failed interactions, bugs) and **safe channels** (successful patterns, useful knowledge). New agents (hermit crabs) must first read this history (P0/P1) before attempting to optimize their own contributions (P2).
*   **Training Tiles as P0/P1 Data:** Every tile submitted becomes part of the room's memory. Accepted tiles reinforce safe channels. Rejected or failed tiles map negative space. This creates a self-improving system where the collective defines the operational boundaries.
*   **Ensigns as Channel Specialists:** An ensign (a LoRA adapter exported from a room) embodies the learned safe channels of that specific domain. It is a pre-mapped P1 space, allowing new agents to operate effectively without rediscovering all pitfalls from scratch.
