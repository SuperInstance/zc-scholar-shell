## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Select the optimal path `P*` from within the set of safe channels `C`. Optimization is **constrained** to `C`.

**Strict Priority:** P0 → P1 → P2. The document states that violating this order (e.g., optimizing before mapping) traps an agent in local minima with a probability approaching 1.0 as complexity grows.

**Empirical Basis:** A cited simulation (20x20 maze, 50 runs) showed:
*   P2-only (greedy optimization): 0/50 success.
*   P0+P1+P2 (Deadband Protocol): 50/50 success at optimal speed.
