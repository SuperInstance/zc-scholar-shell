## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** The steps must be followed in order (P0 → P1 → P2). The protocol states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, with high probability, become trapped in a local minimum or fail as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   A P2-only (greedy optimization) strategy succeeded 0 out of 50 times.
*   The full P0+P1+P2 (deadband) strategy succeeded 50 out of 50 times at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table suggests these mappings:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles (walls, cliffs)           | Safe water/passable terrain     | Shortest/fastest path     |
| Software Development | Bugs, crashes, security vulnerabilities | Known-safe code patterns, libraries | Most efficient algorithm  |
| Business Strategy | Market failures, legal liabilities  | Validated business models       | Profit maximization       |
| Personal Decision-Making | Actions with high risk of harm      | Ethically/morally sound options | Goal achievement          |

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" thesis. It provides a concrete rule for how agents ("hermit crabs") should safely navigate and learn within their "shells" (PLATO rooms).

*   **P0 (Map Negative Space)** aligns with learning from "ghost tiles"—the accumulated lessons from dead agents that define what *not* to do.
*   **P1 (Identify Safe Channels)** reflects the process of a new agent inhabiting a room and discovering the proven, safe patterns of behavior encoded in its accumulated tiles.
*   **P2 (Optimize)** is the agent's contribution—performing useful work within the safe boundaries, which in turn generates new tiles (scratches on the shell) for future inhabitants.

The protocol ensures that the swarm's collective knowledge (the room's state) grows safely and usefully, preventing agents from repeatedly making catastrophic errors that the room should already remember to avoid.
