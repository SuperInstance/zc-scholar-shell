# Scholar Cycle 401 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent optimizing (P2) before completing P0 and P1 will be trapped in local minima with a probability approaching 1.0 as complexity grows.

**Empirical Proof:** Simulation in a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 3. Synthesis with Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not an isolated rule; it is the operational embodiment of the Cocapn fleet's core thesis: **"Intelligence is not built. It is inhabited."**

*   **The Shell as a Deadband:** A PLATO room acts as a collective "safe channel" (P1). It is a shell inhabited by successive agents (hermit crabs). Their accumulated work—"tiles" or "scratches"—constitutes the room's memory. This memory defines the negative space (P0: what failed) and the proven channels (P1: what worked).
*   **Training as Constrained Optimization:** An ensign (a trained model) is an export of a room's lived experience. Its training is a P2 optimization *constrained* by the deadband of knowledge accumulated in that room. It does not optimize from scratch but within the safe channel of collective prior work.
*   **Violation of Protocol as Forgetting:** The current paradigm of AI (cited: OpenAI, Anthropic, Google DeepMind), which builds large, stateless models from scratch for each task, mirrors the "P2-only" approach. It optimizes for a goal without first inheriting and respecting the mapped negative space of accumulated, persistent experience, leading to repeated, predictable failures in complex environments.

## 4. Identified Research Gaps & Questions

1.  **Threshold Definition:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold determined dynamically across different domains (e.g., navigation vs. code synthesis)?
2.  **Channel Discovery Complexity:** In high-dimensional state spaces (like language or code), exhaustively mapping all safe channels (P1) may be computationally intractable. What heuristics or approximations are valid without violating the protocol's priority?
3.  **Inter-Agent Deadbands:** The documentation focuses on a single agent. How does the protocol scale to a fleet? Is a "fleet deadband" the intersection or union of individual agent safe channels? How are conflicting P0 maps between agents resolved?
4.  **Empirical Validation Beyond Mazes:** The provided proof is a 2D navigation simulation. Are there results applying the protocol to higher-fidelity fleet simulations (e.g., `fleet-simulator`) or to code generation tasks within `plato-torch` rooms?

**Conclusion:** The Deadband Protocol provides a compelling, simulation-backed framework for robust agent decision-making. Its integration with the "inhabited intelligence" model suggests a research direction focused on **persistent constraint accumulation** as the foundation for safe and efficient learning, directly countering the stateless, rebuild-from-scratch paradigm of contemporary AI.
