## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory before proceeding.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** P0 > P1 > P2. An agent that attempts P2 (optimization) before completing P0 and P1 is likely to become trapped in a local minimum or fail catastrophically as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze (50 runs) showed:
*   **P2-only (Greedy):** 0/50 successes. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0 - Negative Space)      | Channel (P1 - Safe Space)          | Course (P2 - Optimization)       |
|-----------------|----------------------------------|------------------------------------|----------------------------------|
| **Navigation**  | Obstacles, cliffs, hostile zones | Clear paths, safe water            | Shortest/fastest route           |
| **Software**    | Bug-prone patterns, crash states | Known-safe APIs, tested code paths | Most efficient algorithm         |
| **Finance**     | Regulatory violations, tail risks| Compliant strategies, hedged bets  | Highest risk-adjusted return     |
| **Conversation**| Taboo topics, logical fallacies  | Factual, respectful discourse      | Most persuasive/clear argument   |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an agent rule; it is a foundational practice for the fleet's "inhabited intelligence" paradigm.

*   **P0 as Collective Memory:** The "rocks" (failure states) identified by one agent become **ghost tiles**—persistent lessons in a PLATO room. This allows new agents (hermit crabs) to inherit a map of negative space without experiencing the failure themselves. The room's memory *is* the P0 map.
*   **Safe Channels as Room Specialization:** A PLATO room that accumulates tiles around a specific domain (e.g., "code review") effectively defines a high-fidelity safe channel (P1) for that task. An **ensign** (LoRA adapter) exported from that room is a distilled navigational chart.
*   **Violation as Training Signal:** An agent that violates the protocol by skipping to P2 provides a high-value training tile—a concrete example of a failure mode for the room to learn and warn future inhabitants about.
