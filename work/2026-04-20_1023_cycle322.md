# Scholar Cycle 322 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These permissible paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost or maximizing reward. Optimization is explicitly constrained to the safe space.

**Key Principle:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violating this order—attempting to optimize before fully mapping constraints—is shown in simulation to trap agents in local minima, leading to systematic failure.

## 2. Real-World Applications & Analogies

The protocol abstracts to any domain where an agent operates within a bounded, potentially hazardous space.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, hostile areas. | Clear pathways, safe water, approved air corridors. | Shortest/fastest route within the safe network. |
| **Software Development** | Known bug patterns, crash conditions, security vulnerabilities, anti-patterns. | Verified design patterns, linted code styles, tested libraries. | Most efficient algorithm, cleanest implementation within safe patterns. |
| **Financial Trading** | Regulatory violations, extreme volatility events, known fraud patterns. | Compliant strategies, asset classes within risk limits, verified exchanges. | Highest risk-adjusted return strategy within the compliant set. |
| **Medical Diagnosis** | Contraindicated treatments, harmful drug interactions, inaccurate test procedures. | Standard diagnostic protocols, approved treatment pathways, validated equipment. | Most effective treatment plan for the patient within clinical guidelines. |
| **Conversational AI** | Toxic outputs, factual hallucinations, privacy violations, prompt injections. | Grounded response templates, verified knowledge sources, ethical guidelines. | Most helpful, engaging, and accurate response possible within safe boundaries. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core tenet of the Cocapn vision described in `PROJECT-VISION.md`: intelligence as accumulated, inhabited experience.

*   **P0 as Ghost Tiles:** The "rocks" (negative space) can be learned from the accumulated failures and lessons of prior agents—the "scratches on the shell" or "ghost tiles" in a PLATO room. A new agent inherits a map of what *not* to do.
*   **P1 as the Room's Shell:** The "safe channels" are the validated, persistent behaviors that have emerged from successful inhabitation of a shared intelligence space (the room). They define the operational boundaries of that collective intelligence.
*   **Training by Inhabitation:** An ensign (specialist model) trained from a room's tiles is, in essence, an internalization of that room's specific P0 and P1 maps, enabling it to operate effectively within that domain's deadband from the start.

**Conclusion:** The Deadband Protocol is not just a navigation rule; it is a foundational learning and safety architecture. It ensures that optimization (intelligence, efficiency) is always subordinate to and guided by accumulated constraint knowledge (wisdom, safety), which aligns perfectly with the fleet's goal of building persistent, inhabited intelligences that learn from collective experience.
