# Scholar Cycle 287 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These constraint-compliant regions are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., optimizing before mapping constraints) results in agents becoming trapped in local minima with high probability.

**Empirical Support:** Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents, versus a 50/50 success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                                  | Course (P2)                     |
|-----------------|-------------------------------------|-----------------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, no-fly zones             | Collision-free corridors                      | Shortest/fastest path           |
| **Software Development** | Known bugs, crash conditions, vulnerable patterns | Compiler-valid, linted, tested code patterns  | Most efficient/readable implementation |
| **Machine Learning** | Data regions causing model failure (e.g., adversarial examples) | Model parameter spaces with stable, predictable performance | Hyperparameter tuning for accuracy/speed |
| **Business Strategy** | Market conditions that cause bankruptcy, regulatory violations | Viable business models compliant with laws    | Profit maximization             |
| **Personal Decision-Making** | Actions with severe negative consequences (health, legal, relational) | Choices that maintain safety and integrity    | Choices that maximize well-being |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core tenet of the Cocapn vision described in `PROJECT-VISION.md`: **"Intelligence is not built. It is inhabited."**

*   **Hermit Crab Analogy:** An agent (the crab) does not generate constraints from scratch. It inherits and inhabits a "shell" of accumulated knowledge—the P0 "negative space" and P1 "safe channels" mapped by prior agents. This knowledge is stored as "scratches" (training tiles) within a PLATO room.
*   **P0 as Collective Memory:** The "rocks" (failure states) are not rediscovered by each new agent. They are learned from the aggregated failures and near-misses of all prior inhabitants of a room, creating a persistent, shared negative map.
*   **Safe Innovation:** The protocol enables safe optimization (P2) by first ensuring the agent operates within the inherited, validated boundaries of collective experience. This allows for specialized, efficient action (creating Ensign models) without the high risk of catastrophic failure.

**Conclusion:** The Deadband Protocol is the *how*—the specific, actionable method—for realizing the Cocapn vision of accumulated, inhabited intelligence. It ensures that each new agent (or "ensign") contributes to and safely operates within the growing, shared memory of the fleet.
