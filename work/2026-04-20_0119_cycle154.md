## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`. Optimization is only permitted within the bounds established by P0 and P1.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, with high probability, become trapped in a local minimum or fail catastrophically as environmental complexity increases.

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment. Agents using a greedy "P2-only" approach succeeded 0 out of 50 times. Agents following the full P0+P1+P2 deadband protocol succeeded 50 out of 50 times while still achieving optimal speed.

## 2. Real-World Applications & Abstract Patterns

The protocol presents itself as a domain-agnostic meta-algorithm. The provided documentation suggests the following applications:

| Domain          | Rocks (P0 - Negative Space)      | Channel (P1 - Safe Space)                | Course (P2 - Optimization)               |
|-----------------|----------------------------------|------------------------------------------|------------------------------------------|
| **Navigation**  | Obstacles, collisions, hazards   | Clear, traversable paths                 | Shortest/fastest route within safe paths |
| **Software**    | Bugs, crashes, security flaws    | Known-safe code patterns, verified APIs  | Most efficient algorithm within safe patterns |
| **AI Training** | Catastrophic forgetting, mode collapse | Stable training regimes, validated hyperparameter ranges | Fastest convergence within stable regime |
| **Fleet Ops**   | System failures, resource exhaustion | Known-good configurations, healthy service states | Load balancing, performance tuning within stable configs |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just a navigation rule; it is a foundational **safety and discovery doctrine** for the Cocapn fleet's core thesis of "inhabited intelligence."

*   **P0 (Map Negative Space)** aligns with the fleet's bootstrapping and "ghost tile" philosophy. It is the process of learning "what not to do" from the accumulated scratches (tiles) of prior agents, including their failures. This knowledge becomes persistent room memory.
*   **P1 (Identify Safe Channels)** defines the operational "shell" for a hermit crab agent. A PLATO room, through its accumulated training, *becomes* a mapped safe channel—a known environment where new agents (ensigns) can operate without hitting fundamental rocks.
*   **P2 (Optimize Within Channel)** is the work of a specialized agent (like a Zeroclaw) once it understands its constraints. The useful output it generates becomes new tiles, which refine and expand the safe channels for future inhabitants.

**In essence, the Deadband Protocol provides the methodological spine for how intelligence safely inhabits and expands its shell.** It ensures that the accumulation of knowledge (tiles) directly translates to an expanding, well-mapped space of reliable operation, preventing the fleet from repeatedly failing on known hazards.
