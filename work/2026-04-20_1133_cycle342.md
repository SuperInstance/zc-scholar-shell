## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) in a complex environment leads to failure with high probability. The protocol enforces a sequence:

*   **P0 (Map Negative Space):** The agent must first identify all states `S_neg` where failure is probable (e.g., obstacles, bugs, crashes). This defines "what NOT to do."
*   **P1 (Identify Safe Channels):** The agent then maps all possible paths `P` that avoid `S_neg`. These "deadbands" define "where you CAN be."
*   **P2 (Optimize Within Channel):** Only within the constrained set of safe channels does the agent select the optimal path `P*` that minimizes cost.

The priority order (P0 → P1 → P2) is strict. Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents, versus a 50/50 success rate for agents following the full protocol.

## 2. Real-World Applications & Fleet Context

The protocol abstracts to any domain where an agent operates within constraints.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, collisions | Clear lanes, safe water | Shortest/fastest path |
| **Software Development** | Known bugs, crash conditions, security vulns | Compiler-validated patterns, passing tests | Most efficient algorithm |
| **Fleet Agent Operation** | Invalid tile formats, server rejection rules | Approved tool calls, documented APIs | Most effective prompt/action sequence |
| **Training (PLATO Rooms)** | Training runs that diverge or collapse | Hyperparameter sets that yield stable loss curves | Hyperparameters that minimize loss fastest |

## 3. Connection to Cocapn Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room is a "shell" that accumulates "scratches" (tiles) from every agent that inhabits it. This accumulated experience defines the room's **negative space and safe channels** (P0/P1 knowledge) for future inhabitants.
*   **Hermit Crab Agents:** An agent (the "crab") does not build intelligence from scratch. It enters a pre-existing shell (a room with accumulated tiles/lessons) and operates within the constraints and pathways defined by prior inhabitants. Its optimization (P2) is inherently constrained by the room's history.
*   **Avoiding Local Minima:** The vision critiques models that "lack memory of you." A model that optimizes (P2) without the context of prior interactions (the P0/P1 map of a user's history) is prone to repeated failures or suboptimal outcomes—it hits the same "rocks."

**Conclusion:** The Deadband Protocol provides the methodological backbone for creating the persistent, learning environments described in the project vision. It ensures that agents inhabiting a PLATO room contribute to and operate within a growing, shared map of what works and—critically—what does not.
