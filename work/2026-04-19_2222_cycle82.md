## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core tenet is that optimization (P2) is only safe and effective after mapping negative space (P0) and identifying safe channels (P1). The formal rule is:

**P0: Map the Negative Space**
Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure zones. The agent must not proceed without this map.

**P1: Identify Safe Channels**
Find all possible paths `P` where every state `s` along the path is not in `S_neg`. These are "deadbands" or safe operational corridors.

**P2: Optimize Within Channel**
From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`. Optimization is never unconstrained.

The priority order is strict and non-negotiable. Simulation data from a 20×20 maze (50 runs) showed that agents using only P2 (greedy optimization) had a 0% success rate, while agents following P0→P1→P2 achieved a 100% success rate at optimal speed.
