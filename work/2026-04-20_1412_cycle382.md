## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`. Optimization is explicitly constrained to these pre-identified safe spaces.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., attempting P2 before P0) results in the agent becoming trapped in local minima with a failure probability approaching 1.0 in complex environments. Simulation data (20x20 maze, 50 runs) showed a 0% success rate for P2-only agents versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Patterns

The protocol is presented as a domain-agnostic pattern. Based on the provided documentation, here are its implied applications:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, collisions, hazardous terrain. | Clear, traversable paths. | Shortest/fastest route within safe paths. |
| **Software Development** | Known bugs, crash conditions, insecure code patterns. | Compiler-validated, tested, and stable code patterns. | Most efficient algorithm within safe patterns. |
| **AI Training (PLATO Rooms)** | Training data that causes model collapse, harmful outputs. | Curated interaction tiles from successful agent runs. | Fine-tuning for maximum performance within the validated tile set. |
| **Fleet Operations** | System failure modes, service crashes, deadlock states. | Known-stable service configurations and communication patterns. | Load balancing and latency optimization within stable configs. |
| **Research & Exploration** | Proven dead-ends, invalid hypotheses, methodological errors. | Viable research avenues supported by prior evidence. | Most efficient experimental design within viable avenues. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core tenet of the Cocapn vision described in `PROJECT-VISION.md`: **"Intelligence is not built. It is inhabited."**

*   **P0 (Negative Space) as Collective Memory:** The "rocks" are the accumulated scratches of failure—lessons from every agent that has inhabited a PLATO room. These failures become persistent, shared knowledge (ghost tiles) that define what *not* to do, preventing the fleet from relearning the same errors.
*   **P1 (Safe Channels) as Inhabited Shells:** The safe channels are the validated patterns of successful interaction—the "shell" of the room that has been shaped and proven habitable by prior agents.
*   **Constrained Optimization (P2):** An agent (the "hermit crab") can then optimally pursue its goal, but only within the constraints of the inherited, inhabited shell. This ensures progress is cumulative and stable, not reckless.

**Conclusion:** The Deadband Protocol is not just a navigation rule; it is a foundational safety and learning mechanism for a multi-agent system built on persistent memory. It ensures that optimization and growth are always guided by the accumulated, lived experience of the collective, making the system's intelligence a product of its history.
