## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization leads to failure in complex environments. The protocol enforces a sequence where mapping failure states (negative space) must precede any pathfinding or optimization.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G in environment E:
1.  **P0: Map the Negative Space:** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels:** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel:** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable. An agent that attempts P2 (optimization) before completing P0 and P1 is predicted to become trapped in local minima with a probability approaching 1.0 as environmental complexity increases.

**Empirical Basis:** The document cites simulation data from a 20x20 maze with 50 runs. A "P2-only" greedy optimizer succeeded 0 times, while the full Deadband Protocol succeeded 50 times at optimal speed.
