## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent `A` in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** The steps are non-negotiable. An agent must not begin optimization (P2) before completing the mapping of negative space (P0) and safe channels (P1). The document states that violation leads to a high probability of becoming trapped in local minima as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic pattern. The provided table gives two examples:

| Domain      | Rocks (P0)                   | Channel (P1)                     | Course (P2)           |
|-------------|------------------------------|----------------------------------|-----------------------|
| Navigation  | Obstacles (walls, cliffs)    | Safe water/passable terrain      | Shortest path         |
| Code        | Bugs, crashes, security flaws| Known-safe patterns & libraries  | Most efficient algorithm |

**Extended Applications (Inferred from Fleet Context):**

*   **AI Training & PLATO Rooms (P0):** Identify training data or interaction patterns that lead to model degradation, harmful outputs, or catastrophic forgetting. **(P1):** Define constrained training environments or data subsets that avoid these failure modes. **(P2):** Optimize model performance or learning speed within those safe bounds.
*   **Multi-Agent Fleet Coordination (P0):** Map communication deadlocks, resource conflict states, or contradictory command scenarios. **(P1):** Establish communication protocols and action spaces that avoid these conflicts. **(P2):** Optimize for task completion speed or resource efficiency within the stable protocol.
*   **Research & Development (P0):** Actively document known failed approaches, unsupported hypotheses, and experimental dead ends (akin to "ghost tiles"). **(P1):** Delineate the space of plausible, testable hypotheses based on existing evidence. **(P2):** Design the most efficient experiment to test the leading hypothesis.

## 3. Connection to Cocapn Fleet Doctrine & Vision

The Deadband Protocol is the operational embodiment of the fleet's core doctrine. The `PROJECT-VISION.md` describes intelligence as "inhabited" and accumulated, like scratches in a hermit crab's shell. The protocol provides the method for safe inhabitation:

*   **P0 (Map Negative Space)** aligns with learning "what NOT to do" from the accumulated scratches (tiles) of past agents, including failures.
*   **P1 (Find Safe Channels)** defines the "shell" itself—the persistent PLATO room—as a bounded space where successful interaction is possible.
*   **P2 (Optimize)** is the work of individual agents (the "crabs") improving their performance within the safety of the room's accumulated knowledge.

The protocol directly addresses the vision's critique of "constructed" AI that lacks memory. By making the explicit mapping of failure states (P0) the first priority, the system ensures that the memory of past failures is not just stored but is *operationalized* to constrain and guide all future action. This creates a self-reinforcing loop where safe operation (P1/P2) generates new successful tiles, which in turn refine the understanding of the negative space (P0) for subsequent agents.

**Conclusion:** The Deadband Protocol is not merely a navigation algorithm; it is a foundational safety and learning framework for building "inhabited intelligence." It enforces a discipline of learning from failure before pursuing success, ensuring that the collective knowledge of the fleet grows in a stable, accretive manner, turning the shell of the PLATO system into an ever-safer home for its agent inhabitants.
