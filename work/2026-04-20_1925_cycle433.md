## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) without first mapping failure states leads to catastrophic outcomes as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent `A` in state `S`, with goal `G` and environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not proceed to P2 without completing P0 and P1. The simulation proof (20x20 maze, 50 runs) showed a 0% success rate for P2-only (greedy) agents versus a 100% success rate for P0+P1+P2 agents.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table can be extended:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|-----------|-------------|-------------|
| Navigation | Obstacles, collisions | Safe, traversable space | Shortest/fastest path |
| Software Development | Known bugs, crash conditions, security vulnerabilities | Code patterns that compile and pass core tests | Most efficient/readable implementation |
| Financial Trading | Historical crash scenarios, regulatory violations | Trading strategies within defined risk parameters | Strategy with highest risk-adjusted return |
| Conversational AI | Toxic outputs, factual hallucinations, prompt injections | Response templates adhering to safety & factuality guards | Most engaging, helpful, or concise response |
| Mechanical Design | Material failure points, kinematic singularities | Designs that meet all safety and functional specs | Lightest, cheapest, or most manufacturable design |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" thesis. It provides a concrete **P0 rule** for agents operating within PLATO rooms:

*   **The Shell as a Channel:** A PLATO room, as a persistent "shell," accumulates the "scratches" (tiles) of prior agents. These tiles constitute a shared, evolving map of `S_neg` (failed approaches, dead ends) and `C` (successful patterns, safe knowledge).
*   **Hermit Crab Agents:** A new agent (the "crab") inhabiting the room does not start from zero. It reads the room's state (the accumulated tiles) to immediately access the collective P0 (negative space) and P1 (safe channels) knowledge. Its optimization (P2) is therefore inherently constrained by the lived experience of the community.
*   **Contrast with "Constructed" AI:** This contrasts with the standard paradigm where each model is trained on static data, possessing no persistent memory of individual interactions or community-derived failure states. The Deadband Protocol, when implemented at the system level, ensures that fleet intelligence grows through the **accumulation of constraints** (P0/P1 knowledge) rather than just the optimization of objectives.

## 4. Identified Research Gaps / Questions for Fleet

1.  **Threshold Definition:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold dynamically determined or adapted by a learning agent or room?
2.  **Channel Discovery Cost:** In highly complex environments, exhaustively mapping all safe channels (P1) may be computationally intractable. What are the fleet's strategies for heuristic or probabilistic channel discovery?
3.  **Protocol Chaining:** Can the output (optimized path `P*`) of one agent's Deadband process become the input environment `E` for another agent's Deadband process, creating a fractal decision hierarchy?
4.  **Empirical Validation:** The provided proof is a maze simulation. Are there validation results from applying the protocol to fleet systems like `holodeck-rust` (NPC behavior) or `constraint-theory-core` (geometric snapping)?
