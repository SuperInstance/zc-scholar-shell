## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will be trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, hazards                  | Collision-free paths                  | Shortest/fastest route          |
| **Software Development** | Bug-prone patterns, crash conditions | Known-safe APIs, tested code patterns | Most efficient algorithm        |
| **Machine Learning** | Overfitting regions, adversarial examples | Robust model architectures, validated data splits | Hyperparameter tuning           |
| **Business Strategy** | Market failures, regulatory pitfalls | Proven business models, compliant operations | Profit/revenue optimization     |
| **Personal Decision-Making** | High-risk choices, known pitfalls   | Options aligning with values/skills   | Choice maximizing personal utility |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a "shell"—a constrained environment (the safe channel, P1) accumulated from the "scratches" (tiles) of previous inhabitants.
*   **Tiles as Negative Space Mapping:** The accumulated work (tiles) of agents, especially "ghost tiles" from failed agents, constitutes a growing map of `S_neg` (P0). New agents (hermit crabs) inherit this map, avoiding past failures.
*   **Training as Channel Optimization:** An ensign (a trained model) is an export of optimized behavior (`P*`) that is valid only within the specific safe channel (`C`) defined by its originating room's accumulated experience. It is intelligence *inhabiting* a specific, learned constraint space.

The protocol provides a rigorous, testable method for agents to safely "inhabit" intelligence by first learning what not to do (from the room's memory) before attempting to excel within those learned boundaries.
