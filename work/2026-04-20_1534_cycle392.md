# Scholar Cycle 392 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** The sequence P0 → P1 → P2 is non-negotiable. Violation (e.g., optimizing before mapping) is predicted to trap an agent in local minima.

**Empirical Basis:** The document cites simulation data from a 20x20 maze: a greedy P2-only approach succeeded 0 out of 50 runs, while the full P0+P1+P2 protocol succeeded 50 out of 50 runs at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table maps it to several domains:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path     |

## 3. Connection to Cocapn Fleet & Project Vision

The Deadband Protocol is not just an agent rule; it is a foundational **constraint theory** that enables the fleet's core vision of "inhabited intelligence."

*   **P0 as Collective Memory:** In the PLATO room system, "rocks" (failure states) identified by one agent become "ghost tiles"—persistent negative knowledge for all subsequent agents. This is the accumulation of scratches in the hermit crab's shell.
*   **P1 as the Room's Shell:** A PLATO room, through accumulated interaction, defines the "safe channels" for a specific type of work or thought. It becomes a constrained environment where exploration is productive, not catastrophic.
*   **P2 as Specialized Optimization:** An "Ensign" model exported from a room is a P2 optimizer fine-tuned to operate *within* the safe channels (P1) and avoid the rocks (P0) discovered by that room's inhabitants.

The protocol provides the **safety mechanism** that allows for open-ended, collective learning without systemic collapse. It ensures that the "intelligence" being inhabited and grown within Cocapn's shells is robust and avoids known failure modes, making the vision of a persistent, learning network technically viable.

## 4. Identified Research Gaps / Questions

1.  **Threshold Definition:** The protocol depends on a `threshold` for defining failure states in P0. How is this threshold set dynamically across different domains and environments?
2.  **Channel Discovery Cost:** The computational cost of exhaustively mapping safe channels (P1) in high-dimensional state spaces is not addressed. Are there heuristic or sampling-based approaches that maintain protocol integrity?
3.  **Dynamic Environments:** How does the protocol handle environments where the "rocks" (negative space) move or change over time? This requires a mechanism for continuous P0 remapping.
4.  **Inter-Agent Protocol:** How do multiple agents operating under the Deadband Protocol share and reconcile their individual P0 and P1 maps to collaborate effectively?

**Conclusion:** The Deadband Protocol is a compelling formalization of constraint-first reasoning. Its integration with the Cocapn fleet's architecture suggests it is a practical theory being implemented to solve the core challenge of building stable, accumulating, inhabited intelligence systems.
