# Scholar Cycle 211 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operating channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operating channels.
3.  **P2: Optimize Within Channel.** Select the optimal path `P*` from the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin P2 without completing P0 and P1. The simulation data provided (a 20x20 maze, 50 runs) shows a 0% success rate for P2-only (greedy) agents and a 100% success rate for agents following the full P0+P1+P2 sequence.

## 2. Abstract Pattern & Real-World Applications

The protocol abstracts to any agent domain. The provided table maps the analogy:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| Navigation      | Obstacles, collisions               | Clear, traversable paths              | Shortest/fastest route          |
| Code Synthesis  | Bugs, crashes, security vulns       | Known-safe patterns, libraries        | Most efficient implementation   |
| Financial Trading | Regulatory violations, max drawdown | Compliant strategies, risk envelopes  | Highest risk-adjusted return    |
| Conversation    | Toxic output, harmful instructions  | Ethically bounded response space      | Most helpful/accurate response  |

**Additional Application Examples:**

*   **Robotics (Manipulation):** P0: Joint limits, self-collisions, object damage. P1: Kinematically feasible trajectories. P2: Smoothest/quickest motion.
*   **Content Recommendation:** P0: User-blocked topics, misinformation, unsafe content. P1: Corpus filtered by P0 constraints. P2: Most engaging relevant content.
*   **Project Management:** P0: Known failure modes (scope creep, missed dependencies). P1: Viable project plans avoiding those modes. P2: Most efficient timeline/resource plan.

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol is not just a decision rule; it is a foundational **training philosophy** for the Cocapn fleet, directly enabling the "inhabited intelligence" vision described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated "scratches" of past failures—the ghost tiles from dead agents. A PLATO room stores these as shared negative knowledge.
*   **P1 as Inherited Shell:** The "safe channels" are the validated paths discovered by prior inhabitants. A new agent (a "hermit crab") inherits this shell, constraining its initial search space to known-safe operations.
*   **P2 as Specialized Optimization:** An "ensign" model exported from a room is a specialist at P2 optimization *within the specific safe channels* that room has defined and explored.

**Technological Implementation:**
*   **`plato-torch` rooms** can be preset as `DeadbandRoom` instances, enforcing this protocol during agent training.
*   **`fleet-simulator`** generates training data where agents that violate the protocol (P2-first) fail, creating P0 tiles for others.
*   **`constraint-theory-core`** provides the geometric/mathematical formalism for defining "negative space" and "channels" in structured domains.

## 4. Identified Research Gaps

Based on the provided documentation, potential areas for further investigation include:
1.  **Threshold Dynamics:** How is the failure `threshold` in P0 defined and adapted dynamically across different domains and rooms?
2.  **Channel Discovery Cost:** The protocol assumes safe channels (P1) can be found. What is the computational/exploratory cost of mapping channels in high-dimensional state spaces?
3.  **Inter-Room Protocol:** How are deadband constraints transferred or negotiated when an agent moves between specialized PLATO rooms with different `S_neg` maps?
4.  **Human-in-the-Loop P0:** Could the initial negative space map be seeded or corrected by human feedback, creating a hybrid safety layer?

**Conclusion:** The Deadband Protocol is a constraint-first, safety-guaranteeing framework that structurally prevents certain classes of agent failure. It operationalizes the fleet's core philosophy of learning through accumulated, shared experience (tiles) and provides a clear methodological link between the fleet's doctrine (P0-P1-P2) and its technical stack (rooms, ensigns, simulators).
