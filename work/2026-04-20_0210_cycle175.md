## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid states of high failure probability before attempting optimization.

**Formal Rule (v1.0):**
Given an agent `A` in state `S`, with goal `G` and environment `E`:
1.  **P0: Map the Negative Space** — Identify all states `S_neg` where the probability of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels** — Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operating regions.
3.  **P2: Optimize Within Channel** — From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not proceed to P2 without completing P0 and P1. The document states that violating this order (optimizing before mapping) leads to a high probability of becoming trapped in local minima as environmental complexity increases.

**Empirical Proof:** The protocol cites simulation data from a 20x20 maze across 50 runs. A "P2-only" (greedy optimization) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.
