## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operating channels (P1) leads to catastrophic failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or prohibited zones.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is *not* in `S_neg`. These are the "deadbands" or permissible operating regions.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost or maximizes reward.

**Key Principle:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violating this order—attempting to optimize before mapping constraints—traps the agent in local minima with a failure probability approaching 1.0 in complex environments.

**Empirical Basis:** The document cites simulation data from a 20x20 maze across 50 runs. A greedy, P2-only strategy succeeded 0 times, while the full Deadband Protocol (P0+P1+P2) succeeded 50 times at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic meta-algorithm. The provided table suggests the following mappings:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
| :-------------- | :---------------------------------- | :------------------------------------ | :------------------------------ |
| Navigation      | Obstacles, hazards                  | Clear, traversable paths              | Shortest/fastest route          |
| Software Development | Bugs, crashes, security vulnerabilities | Known-safe code patterns, libraries   | Most efficient/performant implementation |
| Machine Learning | Overfitting, mode collapse, adversarial examples | Stable training regions, validated architectures | Model that maximizes accuracy/score |
| Business Strategy | Market failures, regulatory bans    | Viable business models, compliant operations | Most profitable growth plan     |
| Personal Decision-Making | Actions with high risk of harm      | Choices aligned with values and safety | Option that maximizes personal benefit |

## 3. Connection to Cocapn Fleet & Project Vision

The Deadband Protocol is not just a navigation rule; it is a foundational **safety and reliability doctrine** for the fleet's core thesis of "inhabited intelligence."

*   **P0 as Collective Memory:** In the PLATO room system, "rocks" (failure states) identified by one agent become "ghost tiles"—persistent warnings for all future inhabitants. This is the accumulation of negative knowledge.
*   **P1 as Inherited Shells:** A "safe channel" is analogous to a well-inhabited PLATO room. The room itself, through accumulated interaction tiles, defines the proven boundaries of successful operation. An agent "moves into" this pre-mapped shell.
*   **P2 as Specialized Optimization:** An "Ensign" (a specialized LoRA adapter) represents the optimization of behavior *within* the safe channel defined by its training room. It is a P2 artifact built upon a P0+P1 foundation.

This protocol directly counters the "construction" paradigm criticized in `PROJECT-VISION.md`. Instead of building a monolithic, general-purpose model through unconstrained optimization on vast data (a pure P2 approach), Cocapn advocates for building intelligence through the sequential, priority-driven accumulation of constraints (P0) and proven habitats (P1), *then* specializing (P2).
