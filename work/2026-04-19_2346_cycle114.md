## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe channels (P1) leads to guaranteed failure as environmental complexity increases.

**Formal Rule (v1.0):**
- **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory and non-negotiable.
- **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
- **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes a cost function. Optimization is explicitly constrained to the safe space.

**Violation Consequence:** The document states that an agent optimizing (P2) before completing P0 and P1 "will be trapped in local minima with probability → 1.0 as environment complexity increases."

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
- **P2-only (greedy optimization):** 0/50 successful completions.
- **P0+P1+P2 (deadband protocol):** 50/50 successful completions at optimal speed.
