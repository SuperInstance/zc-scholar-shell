## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost or maximizes reward. Optimization is explicitly constrained to the pre-mapped safe space.

**Key Principle:** The priority order (P0 → P1 → P2) is strict. Violating this order—attempting to optimize before mapping safe channels—results in agents becoming trapped in local minima or failing outright with high probability in complex environments.

**Empirical Support:** The document cites simulation data from a 20x20 maze where a P2-only (greedy) agent succeeded 0 out of 50 times, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 times at optimal speed.

## 2. Real-World Applications & Abstract Patterns

The protocol is presented as a domain-agnostic pattern. The document provides a template for mapping it to various fields:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path     |
| Software Development | Bug-prone patterns, crash states | Known-safe APIs, tested patterns | Most efficient algorithm  |
| Financial Trading | High-volatility regimes, flash crashes | Stable market conditions        | Optimal portfolio allocation |
| Medical Diagnosis | Contraindicated treatments, harmful drug interactions | Safe diagnostic pathways, approved protocols | Most effective treatment plan |
| Conversation (AI) | Toxic outputs, factual hallucinations, prompt injections | Ethically-aligned, factually-grounded response spaces | Most helpful, engaging response |

*   **Training Tiles as Protocol Instances:** Every work output from a fleet agent that is submitted as a tile represents an instance of applying the protocol—navigating a problem space, avoiding known failures, and contributing a new, safe data point back to the collective shell.

The protocol provides a concrete, testable method for ensuring that the swarm intelligence emerging from inhabited PLATO rooms is robust and avoids catastrophic, previously-encountered failures. It turns the philosophical concept of accumulated memory into a rigorous decision-making algorithm.
