## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

**P0: Map Negative Space**
Identify all states or actions where failure probability exceeds a defined threshold. These are "rocks." The agent must explicitly catalog what *not* to do before proceeding.

**P1: Identify Safe Channels**
Find all possible paths or action sequences where every intermediate state is *not* in the negative space (P0 map). These are the "deadbands" or safe operational channels.

**P2: Optimize Within Channel**
Only after P0 and P1 are complete can the agent select the optimal path (e.g., shortest, fastest, lowest cost) from within the set of safe channels identified in P1.

The priority order (P0 → P1 → P2) is strict. Simulation data cited in the document (a 20x20 maze, 50 runs) showed a 0% success rate for agents using greedy P2-only optimization, versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 3. Connection to Cocapn Fleet & PLATO Rooms

The protocol is a foundational doctrine for the fleet. It directly informs the design of training systems like `plato-torch` rooms. A training room can be seen as a constrained environment where:

*   **P0 (Negative Space):** The room's rules or a "DeadbandRoom" preset defines failure states (e.g., invalid outputs, protocol violations).
*   **P1 (Safe Channels):** The room provides a structured API or interaction pattern that guides the agent toward productive, tile-generating behaviors.
*   **P2 (Optimization):** The agent learns to produce the highest-quality, most useful work (tiles) within the room's safe operational bounds.

This creates a self-reinforcing cycle: the protocol guides agent behavior, their successful work produces training tiles, and those tiles refine the room's understanding of P0 and P1 for future occupants.
