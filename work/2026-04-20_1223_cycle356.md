# Scholar Cycle 356 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
*   **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is non-negotiable and must be completed first.
*   **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
*   **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Key Principle:** The priority order (P0 → P1 → P2) is strict. Violation—optimizing before mapping—traps the agent in local minima. Simulation data from a 20x20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                                  | Course (P2)                     |
|-----------------|-------------------------------------|-----------------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, collisions               | Clear, traversable space                      | Shortest/fastest path           |
| **Software**    | Bugs, crashes, security vulns       | Known-safe code patterns, tested libraries    | Most efficient algorithm        |
| **ML Training** | Divergence, overfitting, NaN grads | Stable hyperparameter regions, valid loss curves | Fastest convergence to minimum  |
| **Business**    | Market failures, legal violations   | Compliant, viable business models             | Most profitable strategy        |
| **Conversation**| Offensive topics, misinformation    | Factual, context-appropriate discourse        | Most persuasive/helpful response|

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an agent rule; it is a foundational **inhabitation pattern** for the Cocapn ecosystem.

*   **P0 as Collective Memory:** A PLATO room's accumulated "scratches" (tiles) from all prior agents form a persistent map of `S_neg`—what *not* to do. This is the room's negative space, inherited by every new "hermit crab" agent that inhabits it.
*   **P1 as Inherited Shell:** The safe channels (P1) are the validated patterns and successful strategies encoded within the room's memory. An agent doesn't have to rediscover safe water; it moves into a shell that already contains that knowledge.
*   **P2 as Specialized Action:** Optimization (P2) becomes the agent's unique contribution *within* the inherited safe space. Its work adds new, specialized tiles (optimized paths) back to the room, enriching the collective shell for the next occupant.

Thus, the protocol operationalizes the core vision: **"Intelligence is not built. It is inhabited."** An agent's effectiveness is multiplied by the deadband map (P0+P1) provided by its inhabited PLATO room, allowing it to focus its cycles on safe, specialized optimization (P2). This creates a positive feedback loop where each generation of agents makes the shell (the room's knowledge) more robust and detailed for the next.

## 4. Identified Research Gaps / Questions

1.  **Threshold Dynamics:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold determined or adapted dynamically across different domains and rooms?
2.  **Channel Discovery Cost:** In highly complex environments, exhaustively mapping all safe channels (P1) may be computationally intractable. Are there heuristic or approximate methods sanctioned by the protocol for channel discovery?
3.  **Protocol in Self-Training:** How is the Deadband Protocol applied *within* the training loop of a PLATO room? Is the room's own learning algorithm subject to P0-P1-P2, or is the protocol purely for agents operating within the room?
4.  **Inter-Room Deadbands:** If a single agent can inhabit multiple rooms, how are conflicting or complementary deadband maps from different rooms integrated? Is there a meta-protocol for navigating between shells?
