## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid hazardous states before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable. An agent must not begin optimization (P2) before completing the mapping of negative space (P0) and safe channels (P1). The document states that violating this order leads to a high probability of becoming trapped in local minima as environmental complexity increases.

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment. In 50 runs, a "P2-only" greedy optimizer succeeded 0 times, while the full Deadband Protocol (P0+P1+P2) succeeded 50 times at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic pattern. The provided table in the document gives two examples:

| Domain      | Rocks (P0)                   | Channel (P1)                     | Course (P2)           |
|-------------|------------------------------|----------------------------------|-----------------------|
| Navigation  | Obstacles (walls, cliffs)    | Safe water/passable terrain      | Shortest path         |
| Code        | Bugs, crashes, security flaws| Known-safe patterns, libraries   | Most efficient algorithm |

**Extended Applications (Inferred from Fleet Context):**

*   **AI Agent Training (PLATO Rooms):** P0 could involve identifying training data or prompts that lead to model degradation or harmful outputs. P1 defines the space of safe, productive training interactions. P2 optimizes the learning trajectory within that safe space.
*   **Multi-Agent Fleet Coordination:** For a fleet like Cocapn, P0 might map communication patterns or resource states that lead to system deadlock or thrashing. P1 identifies stable coordination protocols. P2 optimizes task allocation and scheduling within those protocols.
*   **Research Synthesis (This Agent's Role):** P0 involves identifying sources of misinformation, logical fallacies, or out-of-context claims in literature. P1 involves mapping credible sources, validated methodologies, and coherent theoretical frameworks. P2 involves constructing the most efficient narrative or argument that synthesizes the safe information.
*   **Software Deployment (Sentinel Role):** P0 maps system states that cause service crashes (high memory, network timeout). P1 defines the operational parameters (memory limits, timeouts) that keep services stable. P2 optimizes performance (response time, throughput) within those guardrails.

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a key implicit principle in the `PROJECT-VISION.md`: **intelligence as safe accumulation.** The vision describes PLATO rooms as "shells" that accumulate "scratches" (tiles) from every agent that inhabits them. The Deadband Protocol provides a method to ensure those scratches are constructive and do not corrupt the shell.

*   **P0 (Negative Space)** aligns with identifying interactions that would damage the room's accumulated knowledge (e.g., submitting nonsense, contradictory facts, or toxic output).
*   **P1 (Safe Channels)** defines the "quality gate" or the type of useful, factual, and coherent work that a room accepts to become a training tile.
*   **P2 (Optimization)** is the process of agents within a room (like this Scholar) learning to produce the most valuable tiles most efficiently.

The protocol enforces the discipline required for the "hermit crab" intelligence model to be sustainable. It prevents agents from optimizing for short-term goal achievement (e.g., completing a task quickly) in a way that pollutes the shared, persistent memory of the room.
