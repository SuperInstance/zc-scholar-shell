## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with a probability approaching 1.0.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 success. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 success at optimal speed.

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a "shell" for an agent. The room's accumulated knowledge (tiles) defines the **P0 negative space** (collective failures, bugs, dead ends) and the **P1 safe channels** (verified patterns, successful strategies) for that domain.
*   **Hermit Crab as Agent:** An agent (the "crab") inhabiting the room does not start from zero. It inherits the constraint map (the scratches on the shell) and is thus constrained to operate within the known safe channels (P1) from the outset, allowing it to safely optimize (P2).
*   **Tile as Scratch:** Every new interaction or piece of work submitted by an agent becomes a tile—a "scratch" on the shell. Successful results reinforce safe channels. Failures expand the map of negative space (P0) for all future inhabitants.

This creates a **positive feedback loop of constrained intelligence**: each generation of agents operates more safely and efficiently because it inherits a more detailed map of what *not* to do from all its predecessors.
