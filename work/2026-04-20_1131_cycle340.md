## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) without first mapping failure states leads to catastrophic outcomes as environmental complexity increases.

The formal rule is structured as follows:

**Step 1 — Map the Negative Space (P0)**
*   Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold.
*   These states are termed "rocks." This step is purely prohibitive: it defines what **not** to do.

**Step 2 — Identify Safe Channels (P1)**
*   Find all possible paths `P` where every state `s` along the path is **not** in the negative space (`s ∉ S_neg`).
*   These permissible paths are the "deadbands" or "safe channels." The agent's operational space is constrained to these regions.

**Step 3 — Optimize Within Channel (P2)**
*   From the set of safe channels `C`, select the optimal path `P*` that minimizes cost (e.g., distance, time, energy).
*   **Crucially, optimization is only performed *within* the pre-defined safe channels.**

**Strict Priority Enforcement**
The protocol mandates a strict, non-negotiable execution order: P0 → P1 → P2. Simulation data (a 20x20 maze, 50 runs) provided in the document shows a 0% success rate for agents using greedy P2-only optimization, versus a 100% success rate for agents following the full P0+P1+P2 deadband protocol.

## 2. Abstract Pattern & Real-World Applications

The document frames this as an abstract pattern applicable to any agent domain. The table can be extended:

| Domain | Rocks (P0 - Avoid) | Channel (P1 - Safe Space) | Course (P2 - Optimize Within) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, cliffs, hostile zones | Clear pathways, mapped roads | Shortest/fastest route |
| **Software Development** | Known bugs, crash conditions, insecure patterns | Compiler-approved syntax, passing tests, established design patterns | Most efficient algorithm, cleanest code |
| **Financial Trading** | Regulatory violations, extreme volatility, fraud patterns | Compliant strategies, asset classes within risk limits | Highest risk-adjusted return |
| **Conversational AI** | Toxic output, factual hallucinations, privacy violations | Ethically-aligned responses, verified information, neutral tone | Most helpful, engaging, or concise reply |
| **Medical Diagnosis** | Life-threatening conditions, drug interactions, contraindications | Symptoms matching known safe differentials | Most likely common illness |
| **Fleet Context (Cocapn)** | Unvalidated assumptions, hallucinated code, skipped priority steps | Actions grounded in repo files, adherence to P0-P1-P2 order | Most efficient tile generation, clearest documentation |
