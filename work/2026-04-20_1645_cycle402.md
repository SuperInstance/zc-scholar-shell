# Scholar Cycle 402 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden zones.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or permissible operational zones.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Key Principle:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violation—optimizing before mapping—traps the agent in local minima. Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents versus a 50/50 success rate for agents following the full Deadband Protocol.

## 2. Real-World Applications & Abstract Patterns

The protocol presents an abstract pattern applicable to any agent domain. Based on the provided documentation, here are concrete mappings:

| Domain | Rocks (P0 - What NOT to do) | Channel (P1 - Where you CAN be) | Course (P2 - Best path) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, collisions, restricted zones. | Collision-free paths, navigable waterways. | Shortest, fastest, or most energy-efficient route within safe channels. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core tenet of the Cocapn vision described in `PROJECT-VISION.md`: **"Intelligence is not built. It is inhabited."**

*   **P0 as Accumulated Scratches:** The "negative space" map is not pre-defined but is learned and accumulated from the failures (scratches) of all prior agents that have inhabited a PLATO room. A "ghost tile" from a dead agent that hit a "rock" becomes a P0 constraint for future agents.
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated, persistent knowledge of the room—the shell itself. They define the possible spaces where new agents (hermit crabs) can successfully operate.
*   **P2 as Specialized Optimization:** An "ensign" (a specialist AI model) is an export of optimized behavior (P2) that is only valid *within the specific safe channels* (P1) of the room it was trained in.

Thus, the protocol ensures that optimization and specialization (growing smarter) are always grounded in the collective, inhabited memory of the system (the shell), preventing agents from blindly optimizing themselves into failure.
