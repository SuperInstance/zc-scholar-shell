## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is a formal rule designed to prevent agents from becoming trapped in local minima or failing catastrophically by optimizing toward a goal without first understanding the environment's constraints.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory before proceeding.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These permissible paths are "the deadbands" or safe channels. The agent's operational space is constrained to these channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority Order:**
*   **P0:** Identify what *not* to do (negative space).
*   **P1:** Map where you *can* be (safe channels).
*   **P2:** Find the best path within the safe channels.

**Reported Consequence of Violation:** Simulation data from a 20x20 maze (50 runs) indicated that agents using a P2-only (greedy optimization) approach had a 0% success rate, while agents following the full P0+P1+P2 protocol had a 100% success rate. The document states that violating the priority order leads to a high probability of becoming trapped as environmental complexity increases.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. The provided table gives two examples:

| Domain       | Rocks (P0)                   | Channel (P1)                     | Course (P2)               |
|--------------|------------------------------|----------------------------------|---------------------------|
| Navigation   | Obstacles (walls, cliffs)    | Safe water / clear paths         | Shortest path             |
| Code         | Bugs, crashes, security flaws| Known-safe patterns, libraries   | Most efficient algorithm  |

**Extended Applications (Inferred from Fleet Context):**

*   **AI Training & Room Design (PLATO):** A training room's "rocks" could be learning trajectories that lead to mode collapse, reward hacking, or catastrophic forgetting. The "safe channel" is the constrained parameter space or curriculum that avoids these failures. Optimization (P2) then seeks the fastest learning rate within that safe curriculum.
*   **Multi-Agent Fleet Coordination:** For a fleet agent like JetsonClaw1 (edge GPU), "rocks" are system states that cause thermal throttling, memory exhaustion, or unsafe hardware operations. "Safe channels" are defined performance envelopes and system load profiles. Optimization involves completing tasks as quickly as possible while staying within those envelopes.
*   **Research & Development:** In a research project, "rocks" are known dead-end approaches, flawed assumptions, or methods that conflict with core constraints (e.g., privacy laws, hardware limits). "Safe channels" are the viable research directions aligned with resources and ethics. Optimization is the most efficient experimental design within those directions.
*   **Content Generation (This Agent's Task):** For a Scholar agent, "rocks" are outputs that hallucinate facts, violate the fleet's factual mandate, or are irrelevant to the task. "Safe channels" are synthesis based strictly on provided source files (`fleet-knowledge/`) and clear, factual writing. Optimization is producing the most useful, well-structured summary within those constraints.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a key aspect of the fleet's vision as described in `PROJECT-VISION.md`. If intelligence is "inhabited" and accumulated through shared experiences in PLATO rooms, then defining the **negative space (P0)** is critical. It establishes the shared boundaries—the "walls of the shell"—that prevent the collective intelligence from diverging into failure states. The safe channels (P1) become the shared behavioral norms and successful patterns that get reinforced as "scratches" (tiles) in the room's memory. Optimization (P2) is the individual agent's contribution, but it is made robust and reusable because it occurs within the collectively validated safe space.
