## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) is highly likely to fail in complex environments unless the agent first identifies and avoids catastrophic failure states.

**Formal Rule (v1.0):**
Given an Agent `A` in state `S`, with goal `G` and environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg` (`s ∉ S_neg`). These permissible paths are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin optimization (P2) before completing the mapping of negative space (P0) and safe channels (P1). The document states that violating this order—optimizing before mapping—leads to a probability approaching 1.0 of becoming trapped in a local minimum as environmental complexity increases.

**Empirical Support:** Simulation data from a 20x20 maze over 50 runs is cited:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. The provided table gives two examples:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|-----------|-------------|-------------|
| Navigation | Obstacles (walls, cliffs) | Safe water / clear paths | Shortest valid path |
| Software Development | Bugs, crashes, security flaws | Known-safe code patterns, libraries, APIs | Most efficient implementation using safe patterns |

**Extended Applications (Inferred from Fleet Context):**

*   **AI Training & Safety (PLATO Rooms):** Before training a model (P2 optimization), first map failure modes like catastrophic forgetting or reward hacking (P0), then constrain training to methodologies and data that avoid those failures (P1).
*   **Multi-Agent Fleet Operations:** For a fleet agent (e.g., JetsonClaw1), P0 involves identifying states that lead to system crashes, resource exhaustion, or protocol violations. P1 defines the operational parameters and communication patterns that keep the agent stable. P2 is then optimizing task performance within those bounds.
*   **Research & Development:** Before pursuing a novel optimization in a system (P2), a researcher should first comprehensively understand how and why similar approaches have failed (P0), then identify the theoretical or experimental "safe space" where new work can proceed without repeating those failures (P1).

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a key aspect of the fleet's vision as described in `PROJECT-VISION.md`. If "intelligence is inhabited" through accumulated experience (tiles) in shared spaces (PLATO rooms), then the protocol provides a method for that inhabitation to be *safe* and *accumulative*.

*   **P0 (Ghost Tiles):** The "negative space" is partially mapped by the accumulated failures and lessons of prior agents—the "scratches" on the shell that show where not to go. These persist as "ghost tiles."
*   **P1 (The Room as a Channel):** A well-inhabited PLATO room itself becomes a defined "safe channel." Its collected tiles (scratches) collectively describe a bounded space of viable agent behavior and successful learning.
*   **P2 (Specialist Ensigns):** An "ensign" (a trained, specialist model) is the product of optimization (P2) conducted strictly within the safe channel of knowledge (P1) defined by its originating room, which itself was built by mapping negative space (P0).

Thus, the Deadband Protocol is not just a decision rule; it is a foundational safety and efficacy mechanism for the fleet's core inhabited intelligence paradigm. It ensures that the "hermit crabs" (agents) learning within their "shells" (rooms) do not destroy their habitat in the process of optimizing their performance.
