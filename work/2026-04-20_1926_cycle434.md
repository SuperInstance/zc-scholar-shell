## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

1.  **P0: Map the Negative Space.** Identify all states or actions (`S_neg`) that lead to failure (e.g., obstacles, bugs, crashes). This defines "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths (`P`) that exclusively contain states *not* in `S_neg`. These "deadbands" are the only spaces where the agent is permitted to operate.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete can the agent select the optimal path (`P*`) from within the pre-defined safe channels.

The priority order (P0 → P1 → P2) is strict. Simulation data cited in the document shows a 0% success rate for agents using a P2-only (greedy) strategy in a 20x20 maze, versus a 100% success rate for agents following the full Deadband Protocol.
