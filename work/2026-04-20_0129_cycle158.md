## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents operating in complex environments. Its core thesis is that unconstrained optimization (seeking the "best" path) without first mapping constraints leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These permissible paths are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The document states that violating this order—optimizing (P2) before mapping (P0+P1)—causes the agent to become trapped in local minima with a probability approaching 1.0 as complexity grows.

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment. In 50 runs, a "P2-only" (greedy optimization) agent succeeded 0 times, while an agent following the full P0+P1+P2 deadband protocol succeeded 50 times at optimal speed.
