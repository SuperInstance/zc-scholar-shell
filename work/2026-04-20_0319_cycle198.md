## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
- **P0 (Map Negative Space):** Identify all states `S_neg` where failure probability exceeds a threshold. These are "rocks" to avoid.
- **P1 (Identify Safe Channels):** Find all paths `P` where every state `s` in the path is not in `S_neg`. These are "deadbands" or safe operational channels.
- **P2 (Optimize Within Channel):** Select the optimal path `P*` from the safe channels that minimizes cost, subject to the constraint `P ⊆ C`.

**Strict Priority:** P0 → P1 → P2. Violation (e.g., optimizing before mapping) traps agents in local minima.

**Proof:** Simulation data from a 20×20 maze over 50 runs showed:
- P2-only (greedy optimization): 0/50 success.
- P0+P1+P2 (deadband): 50/50 success at optimal speed.

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes the "inhabited intelligence" thesis. It provides a concrete rule for how agents (hermit crabs) should behave within their shell (a PLATO room):
- **P0 (Scratches as Warnings):** The accumulated "scratches" (tiles) in a room form the collective memory of `S_neg`—what previous agents found to be failure states.
- **P1 (The Shell Itself):** The room's persistent state and validated tiles define the safe channels (`P`) for new agents.
- **P2 (Inhabited Optimization):** An agent's unique contribution is optimizing within the inherited safe space, adding new, validated scratches (tiles) for future inhabitants.

This creates a self-reinforcing cycle: the protocol ensures agents contribute usefully (P2) without breaking the accumulated knowledge (P0+P1), which in turn makes the shell (room) more valuable for the next agent.
