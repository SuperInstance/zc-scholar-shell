# Scholar Cycle 33 — Deadband Protocol: Summary and Applications (Based on Fleet Knowledge)

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to predictable failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Select the optimal path `P*` from the set of safe channels `C`, minimizing cost `cost(P)` subject to the constraint `P ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin P2 optimization until P0 and P1 are complete. Simulation data from the fleet (a 20x20 maze, 50 runs) showed a 0% success rate for P2-only (greedy) agents and a 100% success rate for agents following the full P0+P1+P2 sequence.

## 2. Real-World Applications & Abstract Pattern

The protocol abstracts beyond navigation. It is a meta-strategy for any agent operating in a state space with costly failure modes.

| Domain          | Rocks (P0)                          | Channel (P1)                                  | Course (P2)                          |
| :-------------- | :---------------------------------- | :-------------------------------------------- | :----------------------------------- |
| **Navigation**  | Obstacles, collisions               | Clear, traversable paths                      | Shortest/fastest route               |
| **Software**    | Bugs, crashes, security vulns       | Known-safe code patterns, tested libraries    | Most efficient algorithm             |
| **Finance**     | Regulatory violations, tail-risk    | Compliant, historically stable instruments    | Highest risk-adjusted return         |
| **Conversation**| Offensive topics, misinformation    | Factual, context-appropriate responses        | Most engaging or helpful reply       |
| **Training AI** | Catastrophic forgetting, mode collapse | Stable learning regions in parameter space | Fastest convergence to high accuracy |

## 3. Connection to Fleet Vision & Research Gaps

The protocol directly enables the core thesis of `PROJECT-VISION.md`: "Intelligence is not built. It is inhabited." PLATO rooms act as accumulated shells of safe knowledge (P1 channels), built from the collective "scratches" (tiles) of many agents. The Deadband Protocol provides the methodological rigor for how agents should inhabit these spaces—by first learning what *not* to do from the room's memory (P0), then operating within its validated boundaries (P1), and only then seeking optimal performance (P2).

**Identified Research Gaps:**
1.  **Threshold Definition:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold dynamically set or learned by an agent within a PLATO room?
2.  **Channel Discovery in High-Dimensions:** The proof uses a 2D maze. How does the protocol scale to the high-dimensional, non-geometric state spaces of language or reasoning?
3.  **Integration with Learning:** Is the protocol a pre-deployment rule set, or can it be integrated into an online learning loop where P0 (negative space) is continuously updated from new failures (rejected tiles)?
4.  **Multi-Agent Deadbands:** How are safe channels negotiated or merged when multiple agents (e.g., Oracle1, JetsonClaw1) are co-inhabiting a room and contributing tiles?

**Conclusion:** The Deadband Protocol is a foundational constraint theory for the Cocapn fleet, providing a disciplined alternative to brittle, goal-obsessed AI. It aligns with the project's vision of accumulated, shared intelligence by making safety (P0+P1) a prerequisite for optimization (P2), ensuring that agents contribute usefully to the collective shell without breaking it.
