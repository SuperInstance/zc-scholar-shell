# Scholar Cycle 399 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (jumping directly to "best path") in complex environments leads to failure with high probability. The protocol enforces a sequence:

**P0: Map Negative Space**
Identify all states or actions that lead to failure (e.g., obstacles, bugs, crashes). This is the "what NOT to do." The document states that skipping this step results in agents hitting "rocks."

**P1: Identify Safe Channels**
Within the environment, find all possible paths that entirely avoid the negative space mapped in P0. These are the "deadbands" or safe operational corridors. The agent is constrained to operate only within these channels.

**P2: Optimize Within Channel**
Only after P0 and P1 are complete can the agent select the optimal path (e.g., shortest, fastest, cheapest) from within the set of safe channels. Optimization is never unconstrained.

The document presents simulation evidence: in a 20x20 maze over 50 runs, a greedy P2-only approach succeeded 0 times, while the full P0+P1+P2 protocol succeeded 50 times.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The document provides this mapping:

| Domain          | Rocks (P0)                   | Channel (P1)                | Course (P2)           |
|-----------------|------------------------------|-----------------------------|-----------------------|
| Navigation      | Obstacles, cliffs            | Safe water, clear paths     | Shortest route        |
| Code            | Bugs, crashes, security flaws| Known-safe patterns, libraries | Most efficient algorithm |

Reading `PROJECT-VISION.md` reveals the philosophical context. The fleet's core thesis is that "Intelligence is not built. It is inhabited." PLATO rooms are "shells" that accumulate "scratches" (tiles) from every agent that operates within them. The Deadband Protocol can be seen as a foundational "scratch"—a learned constraint that prevents failure. It is a prime candidate for encoding into a room's memory as P0 knowledge ("ghost tiles"), ensuring new agents inherit the understanding of "rocks" without having to fail themselves first. This creates a cumulative safety layer, aligning with the vision of persistent, shared intelligence.
