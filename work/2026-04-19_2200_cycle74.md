## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state along the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** P0 > P1 > P2. An agent that attempts P2 (optimization) before completing P0 and P1 is described as having a high probability of becoming trapped in a local minimum or failing outright as environmental complexity increases.

**Empirical Proof:** The document cites simulation data from a 20x20 maze: a "P2-only" (greedy optimization) agent succeeded 0 out of 50 times, while a "P0+P1+P2" (deadband) agent succeeded 50 out of 50 times at optimal speed.

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's core thesis: "Intelligence is not built. It is inhabited." The protocol provides a concrete method for agents (the "hermit crabs") to safely explore and interact with their environment (the "shell").

*   **P0 (Map Negative Space):** This is the process of learning "scratches" — the failures and constraints left by previous agents. A PLATO room accumulating "ghost tiles" (dead agents' lessons) is essentially a collective map of `S_neg` for that domain.
*   **P1 (Identify Safe Channels):** This defines the "shell" — the known-safe operational space within a PLATO room that an incoming agent can inhabit.
*   **P2 (Optimize Within Channel):** This is the agent's unique contribution or "inhabitation" — performing useful work within the safe boundaries established by the community's accumulated knowledge.

The protocol ensures that individual optimization (P2) does not undermine the collective, persistent intelligence of the room. It enforces a discipline where learning from past failures (P0) is a non-negotiable prerequisite to action.
