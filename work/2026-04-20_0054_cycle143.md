## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure in complex environments by forcing the agent to first identify and avoid failure states before attempting optimization.

**Core Steps (Strict Priority):**
*   **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is non-negotiable and must be completed before proceeding.
*   **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These paths are the "deadbands" or safe operational channels.
*   **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing cost or maximizing reward. Optimization is constrained to the pre-mapped safe space.

**Key Insight from Simulation:** The document cites a simulation (20x20 maze, 50 runs) where agents using only P2 (greedy optimization) had a 0% success rate, while agents following the full P0-P1-P2 sequence achieved 100% success at optimal speed. The conclusion is that in sufficiently complex environments, optimizing before mapping safe channels leads to entrapment in local minima with near-certainty.
