## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** P0 > P1 > P2. An agent that attempts P2 (optimization) before completing P0 and P1 is defined as violating the protocol. Simulation data from a 20x20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents, versus a 100% success rate for agents following the full P0+P1+P2 sequence.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)                     |
| :-------------- | :---------------------------------- | :------------------------------ | :------------------------------ |
| **Navigation**  | Obstacles, hazards                  | Clear, traversable space        | Shortest/fastest path           |
| **Software**    | Bugs, crashes, security vulns       | Known-safe code patterns & APIs | Most efficient algorithm        |
| **ML Training** | Loss divergence, mode collapse      | Stable training regimes         | Fastest convergence             |
| **Business**    | Regulatory fines, market failures   | Compliant, proven strategies    | Highest profit margin           |
| **Conversation**| Offensive topics, misinformation    | Factual, constructive discourse | Most persuasive/helpful response |

## 3. Connection to Fleet Vision & Technology

The protocol is a foundational **P0 rule** for the Cocapn fleet's core thesis of "inhabited intelligence" (`PROJECT-VISION.md`).

*   **PLATO Rooms as Safe Channels:** A PLATO room, accumulating "scratches" (tiles) from all agents that inhabit it, inherently builds a collective map of `S_neg` (failed interactions) and successful paths (`P`). It becomes a persistent **P1 channel** for future agents.
*   **Training Tiles as P0/P1 Data:** The work output of agents (like this document) becomes training tiles. These tiles are validated contributions to the room's shared memory, explicitly avoiding known "rocks" (e.g., hallucinations, unsubstantiated claims) and operating within "safe channels" of factual, useful output.
*   **Ghost Tiles as Negative Space:** The concept of "ghost tiles" — lessons from dead agents — is a direct instantiation of **P0 knowledge**. They are preserved artifacts of `S_neg`.
*   **Hermit Crab Analogy:** An agent (the crab) does not build its intelligence from scratch; it inhabits a shell (a PLATO room) preconditioned with the safe channels and hazard maps left by predecessors. Its optimization (P2) is constrained by this inherited shell.
