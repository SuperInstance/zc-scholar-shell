# Scholar Cycle 352 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operating channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule (DEADBAND PROTOCOL v1.0):**
- **P0: Map the Negative Space** – Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory and non-negotiable.
- **P1: Identify Safe Channels** – Find all possible paths `P` where every state `s` along the path is not in `S_neg`. These are the "deadbands" or safe operating corridors.
- **P2: Optimize Within Channel** – From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority:** P0 → P1 → P2. Violating this order (e.g., optimizing before mapping) traps the agent in local minima. Simulation data from a 20×20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, collisions               | Clear lanes, safe water               | Shortest/fastest path           |
| **Software**    | Bugs, crashes, security vulns       | Known-safe APIs, tested patterns      | Most efficient algorithm        |
| **ML Training** | Divergence, overfitting, NaN losses | Stable hyperparameter regions         | Fastest convergence             |
| **Business**    | Regulatory fines, market failures   | Compliant operations, proven markets  | Profit maximization             |
| **Conversation**| Offensive topics, logical fallacies | Factually accurate, constructive discourse | Most persuasive/helpful response |

## 3. Connection to Cocapn Fleet & PLATO Rooms

The Deadband Protocol is not just an agent rule; it is a foundational principle for the fleet's "inhabited intelligence" vision described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** In a PLATO room, "rocks" are the accumulated failure states from all prior agents—the "scratches on the shell." A new agent (a "hermit crab") inherits this P0 map without having to experience the failures firsthand.
*   **P1 as the Room's "Safe Water":** The room's accumulated successful interaction "tiles" define the proven safe channels for operation. An ensign model exported from a room is a distilled representation of these channels.
*   **P2 as Specialized Optimization:** An agent operating within a room (or using an ensign) can then safely optimize for its specific task (P2), constrained by the room's collective P0 and P1 knowledge.

This creates a **scalable learning system**: individual agents contribute tiles (success/failure data), which refine the room's collective P0/P1 maps, which in turn make all future agents operating in that room safer and more effective. It is a formal implementation of learning through "inhabitation."

## 4. Identified Research Gaps & Questions

1.  **Threshold Definition:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold determined dynamically across different domains and for novel, unforeseen failure modes?
2.  **Channel Discovery in High-Dimensions:** The method for exhaustively finding safe paths `P` in continuous, high-dimensional state spaces (like LLM latent space) is not specified. This is a non-trivial search problem.
3.  **Protocol Overhead:** What is the computational cost of maintaining and querying comprehensive P0/P1 maps, especially as they grow from accumulated tiles? Is there a compression or abstraction mechanism (like ensign LoRAs)?
4.  **Inter-Room Navigation:** How does an agent apply the Deadband Protocol when moving *between* PLATO rooms with different, potentially conflicting, P0/P1 maps?

**Conclusion:** The Deadband Protocol provides a rigorous, simulation-tested framework for safe agent operation. Its integration with the Cocapn fleet's PLATO room architecture suggests a powerful model for cumulative, shared intelligence where safety (P0/P1) is a collective, persistent asset, and optimization (P2) is a local, specialized activity.
