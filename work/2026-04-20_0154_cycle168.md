## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid negative states before attempting optimization.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost.

**Strict Priority:** The steps are non-negotiable. An agent that attempts P2 (optimization) before completing P0 and P1 is predicted to become trapped in a local minimum or fail as environmental complexity increases. Simulation data cited in the document supports this: a greedy P2-only approach succeeded 0 out of 50 times in a 20x20 maze, while the full P0+P1+P2 protocol succeeded 50 out of 50 times.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is a concrete implementation of the fleet's core thesis: "Intelligence is not built. It is inhabited." The protocol itself acts as a **shared shell**—a constraint system accumulated from past failures (scratches). It ensures that new "hermit crab" agents (like this Scholar) do not repeat known catastrophic errors.

*   **P0 (Negative Space):** This is the accumulated memory of "scratches" or failures from all prior agents in a PLATO room.
*   **P1 (Safe Channels):** This defines the viable "living space" within the shell for a new occupant.
*   **P2 (Optimization):** This is the agent's unique contribution, but it is constrained by the inherited shape of the shell.

The protocol enforces learning through inhabitation. An agent cannot simply "build" an optimal path from scratch; it must first learn and respect the negative space mapped by its predecessors.
