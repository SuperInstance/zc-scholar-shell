## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure in complex environments by enforcing a sequence that maps constraints before attempting optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These permissible paths are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`. Optimization is only permitted within the bounds established in P0 and P1.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, as environmental complexity increases, become trapped in local minima with a probability approaching 1.0.

**Empirical Basis:** The protocol is supported by simulation data (a 20x20 maze, 50 runs) showing that a P2-only (greedy) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not an isolated rule but aligns with the core thesis of Cocapn/SuperInstance: "Intelligence is not built. It is inhabited."

*   **Hermit Crab Analogy:** The protocol provides a "shell" of operational discipline. The P0 step ("Map the Negative Space") is akin to the crab inspecting a potential shell for fatal cracks or occupants before moving in. It is a foundational survival behavior.
*   **Accumulation of Scratches:** Each application of the protocol—each identified rock (P0) and validated safe channel (P1)—generates a "tile" of experiential knowledge. When these tiles are submitted to a PLATO room, they become part of the room's persistent memory, accumulating scratches that guide future inhabitants (agents).
*   **Rejection of "Construction-Only" AI:** The protocol enforces learning from failure (P0) and constraint (P1) before optimization (P2). This contrasts with a pure "P2" approach that mirrors the "construction" paradigm—focused solely on goal performance without building a persistent map of what *not* to do, leading to repeated, catastrophic failures in novel situations.
