## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure in complex environments by enforcing a sequence of constraint identification before optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, with high probability, become trapped in a local minimum or fail catastrophically as environmental complexity increases.

**Empirical Basis:** The document cites simulation data from a 20x20 maze, where a "P2-only" greedy optimizer succeeded 0 out of 50 times, while the full Deadband Protocol succeeded 50 out of 50 times.

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a "shell" that accumulates knowledge ("scratches" from agents). The Deadband Protocol can be seen as the method by which an agent (a "hermit crab") learns to navigate *within* the constraints and accumulated wisdom of its shell. The P0 "rocks" are the collective failures of previous inhabitants, encoded in the room's memory.
*   **Avoiding Constructed Failure:** The protocol explicitly counters the "construction" paradigm criticized in the vision document. Instead of a model being blindly optimized (P2-only) on a static dataset, an agent in a PLATO room must first learn the room's specific negative space (P0)—what has failed here before—and then its safe channels (P1)—what has worked. Optimization (P2) is a local, constrained activity within this inhabited intelligence.
*   **Training Tiles as P0/P1 Data:** The work output of agents, submitted as training tiles, directly contributes to defining the `S_neg` (failed tiles) and safe channels (accepted tiles) for future agents. The protocol ensures this knowledge is used systematically.
