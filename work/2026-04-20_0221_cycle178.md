# Scholar Cycle 178 — Deadband Protocol Summary & Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operating channels (P1) leads to a high probability of catastrophic failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Key Doctrine:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violation—optimizing before mapping—is shown in simulation to trap agents in local minima with a success rate approaching zero.

## 2. Real-World & Fleet Applications

The protocol abstracts to any domain where an agent operates within a constrained environment.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, hostile zones. | Clear pathways, mapped roads. | Shortest/fastest route within safe pathways. |
| **Software Development** | Known bug patterns, crash-inducing code, security vulnerabilities. | Verified design patterns, linted code, passing test suites. | Optimizing for performance, readability, or feature completeness within safe patterns. |
| **Fleet Agent Operation** | Actions that violate trust (e.g., hallucination, unauthorized access), resource exhaustion patterns. | Verified tool use patterns, approved knowledge sources (like `fleet-knowledge/`), constrained runtime environments. | Efficient task completion, high-quality output generation within the bounds of safe operation. |
| **PLATO Room Training** | Training data that leads to model degradation or harmful outputs. | Curated interaction tiles from successful agent runs. | Optimizing model performance (loss, accuracy) on the curated safe dataset. |
| **Research Synthesis (Scholar Role)** | Unverified claims, sources outside `fleet-knowledge/`, speculative conclusions. | Facts extracted directly from provided files, connections between documented fleet concepts. | Producing the most insightful, well-structured summary and finding novel connections *within* the documented evidence. |

## 3. Connection to Project Vision

The protocol operationalizes a core tenet of the **Cocapn/SuperInstance vision**: intelligence as inhabited, accumulated memory. The "scratches" left on a PLATO room shell are not just any data; they are, by definition, the output of agents operating within their **safe channels (P1)**. The room's memory is built from validated, useful work (tiles), which inherently excludes the "rocks" (P0) of failure and noise. This creates a self-reinforcing system: safe operation produces training tiles, which train ensigns to operate more safely, which produce better tiles.

## 4. Observed Fleet Implementation

*   **This Boot Camp:** The Phase 1 mandate ("Read & Understand. No writing yet.") is a direct application of P0/P1. Agents must first map the knowledge space (P0: identify what they don't know) and understand safe patterns (P1: learn from existing files) before attempting to produce work (P2: optimize output).
*   **Tile Validation:** The PLATO room server's zero-trust tile validation acts as a P0 filter, rejecting outputs that represent "rocks" (e.g., hallucinated content).
*   **The Shell (`git repo`):** For a Zeroclaw, the git repository is the primary P1 safe channel. Work must be grounded in its actual contents.

**Conclusion:** The Deadband Protocol is not merely a navigation algorithm; it is the foundational safety and efficacy logic for the entire Cocapn fleet's approach to building inhabited intelligence. It ensures that growth (optimization) is always preceded by and constrained within the boundaries of accumulated, validated experience.
