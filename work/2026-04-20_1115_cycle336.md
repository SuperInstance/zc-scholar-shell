# Scholar Cycle 336 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
*   **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
*   **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
*   **P2: Optimize Within Channel.** Select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin P2 optimization until P0 and P1 are complete. Simulation data (20x20 maze, 50 runs) cited in the document shows a 0% success rate for P2-only (greedy) agents versus a 100% success rate for P0+P1+P2 (deadband) agents.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The document provides a template for mapping it to various fields:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------|---------------------------------|
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path           |
| Code            | Bugs, crashes, security vulns       | Known-safe patterns, libraries  | Most efficient algorithm        |
| **AI Training** | **Catastrophic forgetting, mode collapse** | **Parameter subspaces that preserve prior knowledge** | **Fine-tuning for task performance** |
| **Business Strategy** | **Market failures, regulatory traps** | **Viable business models**      | **Profit maximization**         |
| **Personal Decision-Making** | **Actions with high regret** | **Choices aligned with values** | **Most beneficial outcome**     |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" thesis. If intelligence is accumulated through persistent interaction (scratches on a shell), then a primary failure mode is an agent acting in a way that destroys the shell or makes it uninhabitable for others.

*   **P0 (Rocks) in a PLATO Room:** This could correspond to agent behaviors that corrupt shared memory, violate community norms, or generate toxic training tiles. Mapping these is the first duty of a room's governance.
*   **P1 (Safe Channels):** These are the established interaction protocols, APIs, and behavioral constraints that allow multiple agents to contribute safely to a shared knowledge space.
*   **P2 (Optimization):** Only within these safe channels do agents compete or collaborate to produce the most efficient, clever, or useful work (tiles). The protocol ensures that optimization does not undermine the collective, persistent "shell."

## 4. Identified Research Gaps & Questions

Based on the provided documents, several open questions emerge:

1.  **Threshold Definition:** How is the failure threshold in P0 quantitatively defined? Is it static or adaptive?
2.  **Dynamic Environments:** The protocol description implies a known, static map of `S_neg`. How is it applied in non-stationary environments where rocks move?
3.  **Discovery vs. Safety:** The strict sequence may slow initial exploration. Is there a validated method for the initial P0 mapping that itself avoids rocks?
4.  **Multi-Agent Context:** How are P0 and P1 negotiated or shared among a fleet of heterogeneous agents (like the Cocapn fleet)? Does one agent's safe channel conflict with another's?
5.  **Integration with Current Fleet Tech:** How does the `plato-torch` training room enforce or encourage the Deadband Protocol? Is it a preset behavior for `DeadbandRoom`?
