## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will become trapped in local minima as complexity rises.

**Empirical Basis:** Protocol validation comes from a simulated 20x20 maze environment over 50 runs. Agents using a P2-only (greedy optimization) strategy had a 0% success rate, while agents following the full P0+P1+P2 protocol achieved a 100% success rate at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided documentation maps it to several domains:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, hazards                  | Safe water, clear lanes         | Shortest/fastest path     |
| Software Development | Known bugs, crash conditions, anti-patterns | Compiler-validated, tested code patterns | Most efficient algorithm  |
| Machine Learning | Training divergences, adversarial examples | Stable regions of hyperparameter space | Highest accuracy model   |
| Business Strategy | Regulatory violations, market failures | Viable business models          | Most profitable strategy  |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

If intelligence is accumulated through persistent interaction (scratches on a shell), then a primary function of a PLATO room is to **collect and encode P0 knowledge**—the "rocks" discovered by prior agents.
