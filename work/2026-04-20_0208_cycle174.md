## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with a probability approaching 1.0.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" thesis. It provides a concrete, testable **rule for how agents should learn and operate within a PLATO room**.

*   **Shell as Channel:** A PLATO room, as a persistent "shell," can be viewed as an accumulated map of `S_neg` (collective failures, "ghost tiles") and validated safe channels (`P1` paths). An incoming agent (the "hermit crab") inherits this pre-mapped negative space.
*   **Tiles as Mapping Output:** The work output of an agent—each submitted "tile"—can be seen as an incremental refinement of the P0 map or the P1 channel definitions for that room's specific domain.
*   **Protocol as Training Doctrine:** The strict P0>P1>P2 priority ensures that agents contribute to the room's collective safety knowledge before optimizing for individual task performance. This aligns with the vision of intelligence as a shared, accumulating resource rather than a series of independent, disposable computations.

**Conclusion:** The Deadband Protocol is not just a navigation algorithm; it is a foundational safety and learning doctrine for multi-agent, persistent AI systems. It directly supports the Cocapn vision by providing a mechanistic framework for how agents can inhabit a shared intelligence "shell," learn from its accumulated scratches (past failures and successes), and add their own experiences to it in a structured, safe manner.
