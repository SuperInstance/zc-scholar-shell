## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first map what *not* to do before considering any optimization.

**Formal Rule (v1.0):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains entirely within `C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, with high probability, become trapped in a local minimum or fail as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided documentation maps it to two domains:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| **Navigation**  | Obstacles, collisions               | Clear, safe water/airspace      | Shortest/fastest route    |
| **Code**        | Bugs, crashes, security vulnerabilities | Known-safe code patterns, libraries | Most efficient algorithm |

**Potential Extended Applications (Inferred from Fleet Context):**
*   **AI Training/Room Design (PLATO):** P0: Training runs that diverge or collapse. P1: Hyperparameter ranges and data subsets that produce stable learning. P2: Finding the fastest convergence within those safe ranges.
*   **Multi-Agent Coordination (Fleet):** P0: Communication patterns that lead to deadlock or conflict. P1: Established, verified interaction protocols. P2: Optimizing task throughput within those protocols.
*   **Hardware Resource Management (CudaClaw/Flux):** P0: Memory states that cause overflow or thermal throttling. P1: Stable operating envelopes for GPU/CPU load. P2: Maximizing computational output within those envelopes.

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a core tenet of the Cocapn vision described in `PROJECT-VISION.md`: creating persistent, shared spaces where intelligence "grows through inhabitation, accumulation, and collective learning."

*   **Accumulation of Scratches:** Each agent's discovery of "rocks" (P0) and "safe channels" (P1) in a given PLATO room becomes a "tile" or "scratch" left on that room's shell. This accumulated negative and positive knowledge directly shapes the environment for future agents.
*   **Inhabited Intelligence:** The protocol ensures that an agent does not operate in a vacuum. It must first read the room's accumulated memory (the collective P0/P1 map) before acting. Its subsequent optimized actions (P2) then add to that room's experiential knowledge.
*   **Rejection of "Greedy" Construction:** The protocol's stark warning against P2-first behavior mirrors the vision's critique of the mainstream AI paradigm—"building marvels of engineering that lack memory." It prioritizes safe, context-aware operation (inhabitation) over unconstrained, locally-optimal but globally fragile solutions (construction).

**Conclusion:** The Deadband Protocol is not just a decision rule; it is a foundational safety and learning mechanic for the Cocapn ecosystem. It ensures that agents contribute to and learn from the collective memory of a PLATO room, transforming individual action into shared, persistent intelligence. This creates a flywheel where each agent's disciplined operation (P0->P1->P2) makes the environment richer and safer for all subsequent inhabitants.
