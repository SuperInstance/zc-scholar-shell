## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is a systematic rule designed to prevent agents from becoming trapped in local minima by forcing them to first identify failure states before attempting optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** P0 > P1 > P2. The protocol is non-negotiable; an agent that attempts P2 (optimization) before completing P0 and P1 is predicted to fail with probability approaching 1.0 as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol abstracts to any agent domain. The provided documentation maps it to navigation and code. Below are extended applications based on the pattern.

| Domain | Rocks (P0 - What NOT to do) | Channel (P1 - Where you CAN be) | Course (P2 - Best path) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, collisions, restricted zones. | Clear, traversable space. | Shortest, fastest, or most fuel-efficient route. |
| **Software Development** | Known bugs, crash conditions, insecure patterns, anti-patterns. | Compiling code, passing tests, adhering to style guides. | Most efficient algorithm, cleanest architecture, best performance. |
| **Financial Trading** | Regulatory violations, known scam assets, historical crash patterns. | Compliant instruments, assets with positive historical risk-adjusted returns. | Portfolio allocation that maximizes Sharpe ratio or minimizes drawdown. |
| **Conversational AI** | Toxic outputs, factual hallucinations, privacy violations, prompt injections. | Responses that are helpful, harmless, and honest (HHH). | Most informative, engaging, or contextually perfect response. |
| **Fleet Context (Cocapn)** | Actions that violate `DEADBAND-PROTOCOL.md` (skipping to P2), generating "hallucinated" content without reading source files. | Producing factual summaries, connecting ideas from provided `fleet-knowledge/` files, writing useful output. | The most insightful synthesis or analysis that advances fleet goals. |

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a key aspect of the Cocapn vision described in `PROJECT-VISION.md`. If "intelligence is inhabited" in PLATO rooms that accumulate "scratches" (tiles) from every agent, then the protocol provides a **reliable method for generating high-quality, reusable scratches**.

*   **P0 (Map Negative Space)** is the collective memory of failure—the "scratches" that show where previous agents crashed. This is the ghost tile knowledge.
*   **P1 (Safe Channels)** defines the proven, habitable space within a room—the patterns of interaction that are known to yield valid, accepted tiles.
*   **P2 (Optimize)** is the agent's unique contribution within those proven bounds, adding a new, optimal scratch to the shell.

The protocol ensures that an agent's work (its "scratch") is not a random mark but a deliberate, valid addition to the accumulated intelligence of the room. It turns the hermit crab's movement from chaotic exploration into directed, sustainable inhabitation.

**Conclusion:** The Deadband Protocol is not just a navigation rule; it is a foundational constraint for building reliable, accumulative intelligence systems. It enforces a discipline of safety-first exploration that aligns directly with Cocapn's core thesis of intelligence through persistent, shared, and memory-rich interaction.
