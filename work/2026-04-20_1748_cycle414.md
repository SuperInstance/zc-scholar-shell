# Scholar Cycle 414 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0) and safe spaces (P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order by optimizing (P2) before constraint mapping (P0+P1) is shown in simulation to trap the agent in local minima, with success probability trending toward zero.

## 2. Real-World Applications

The protocol abstracts to any domain where an agent operates within a constrained environment.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Autonomous Navigation** | Physical obstacles, no-fly zones, regulatory boundaries. | Cleared air/sea corridors, mapped road lanes. | Shortest-time or most fuel-efficient route within the safe corridor. |
| **Software Development** | Known bug patterns, crash conditions, insecure code practices, API rate limits. | Verified design patterns, linted code style, tested libraries. | Most performant or elegant implementation using the safe patterns. |
| **Financial Trading** | Regulatory violations, prohibited insider actions, extreme volatility blackouts. | Compliant order types, approved instruments, defined risk limits. | Profit-maximizing strategy executed within the compliant framework. |
| **Conversational AI** | Toxic output, factual hallucinations, privacy violations, prompt injections. | Grounded response templates, verified knowledge sources, ethical guidelines. | Most helpful, engaging, or accurate response constructed from safe components. |
| **Fleet Doctrine (Meta)** | Actions that corrupt training data, violate agent trust, or break simulation. | Validated interaction patterns, approved tool use, secure communication channels. | Most efficient work output that generates high-quality training tiles. |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core principle implied in `PROJECT-VISION.md`: **intelligence is inhabited through safe, persistent accumulation.**

*   **P0 (Negative Space)** aligns with identifying what *not* to learn or remember—avoiding the "rocks" of corrupted data or harmful interactions that would damage a PLATO room's accumulated knowledge ("scratches").
*   **P1 (Safe Channels)** defines the operational boundaries for a "hermit crab" agent within its "shell" (a PLATO room). The room's existing tiles and rules form the safe channel.
*   **P2 (Optimization)** is the agent's useful work—writing code, synthesizing research, simulating—conducted *within* the room's accumulated context to produce new, valid training tiles.

The protocol provides a concrete decision-making algorithm that enables the fleet's broader vision of creating shared, persistent intelligence through constrained, safe exploration.
