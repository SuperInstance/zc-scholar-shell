## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to inevitable failure as environmental complexity increases.

**Formal Rule (v1.0):**
- **P0: Map Negative Space** – Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
- **P1: Identify Safe Channels** – Find all paths `P` where every state `s` in the path is not in `S_neg`. These are "deadbands" or safe operational corridors.
- **P2: Optimize Within Channel** – From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin optimization (P2) before completing P0 and P1. The document states that violating this order—optimizing before mapping—results in the agent becoming trapped in local minima with a probability approaching 1.0 as complexity grows.

**Empirical Proof:** The protocol is supported by simulation data from a 20x20 maze environment over 50 runs. Agents using a P2-only (greedy optimization) strategy had a 0/50 success rate, consistently hitting obstacles. Agents following the full P0+P1+P2 deadband protocol achieved a 50/50 success rate while maintaining optimal speed.
