## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid hazardous states before attempting optimization.

### Core Steps (Strict Priority)
1.  **P0: Map the Negative Space.** Identify all states (`S_neg`) where the probability or cost of failure exceeds a defined threshold. These are the "rocks." This step is mandatory and must be completed before proceeding.
2.  **P1: Identify Safe Channels.** Find all possible paths (`P`) where every state along the path is *not* in the negative space (`s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path (`P*`) from within the set of safe channels, minimizing cost or maximizing reward. Optimization is **constrained** to the safe space.

### Key Principle
The protocol asserts that an agent which attempts to optimize (P2) before mapping the negative space (P0) and safe channels (P1) will, with high probability, become trapped in a local minimum or fail catastrophically as environmental complexity increases. Simulation data (a 20x20 maze over 50 runs) is cited as evidence: a P2-only (greedy) approach succeeded 0 times, while the full P0+P1+P2 protocol succeeded 50 times.
