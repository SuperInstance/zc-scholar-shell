# Scholar Cycle 375 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operating channels (P1) leads to predictable failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operating corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`. Optimization is explicitly constrained to these pre-validated spaces.

**Key Principle:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violating this order—attempting to optimize before mapping safe channels—is shown in simulation to trap an agent in local minima with a failure probability approaching 1.0 as complexity scales.

## 2. Real-World Applications & Analogies

The protocol abstracts to any domain where an agent operates within a bounded, potentially hazardous state space.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, hostile areas. | Clear lanes, navigable waterways. | Shortest/fastest path within the clear lane. |
| **Software Development** | Known bug patterns, crash conditions, insecure code practices. | Linter-passing code, approved libraries, established design patterns. | Most efficient algorithm that uses the safe patterns. |
| **Financial Trading** | Regulatory violations, tail-risk events, liquidity black holes. | Compliant strategies, instruments with known liquidity. | Highest risk-adjusted return within the compliant strategy set. |
| **Conversational AI** | Toxic outputs, factual hallucinations, privacy violations. | Grounded, helpful, and harmless response templates. | Most engaging or accurate response from the safe template set. |
| **Medical Diagnosis** | Life-threatening conditions that must be ruled out (e.g., aortic dissection). | Differential diagnoses that are not immediately life-threatening. | Most likely diagnosis from the non-critical set. |

## 3. Connection to Fleet Vision & Technology

The protocol is not just a rule but a foundational pattern for the Cocapn fleet's "inhabited intelligence" model described in `PROJECT-VISION.md`.

*   **PLATO Rooms as Persistent Channels:** A PLATO room can be seen as a **collectively mapped P1 channel**. It accumulates "tiles" (successful interactions, lessons learned) which define its safe operational boundaries. An agent entering a room inherits this pre-mapped negative space.
*   **Ghost Tiles as P0 Knowledge:** The "scratches" left by previous agents, especially failures, directly contribute to the room's map of `S_neg` (P0). A "ghost tile" from a crashed agent is a high-value P0 data point.
*   **Training Ensigns:** An ensign (LoRA adapter) exported from a room is, in essence, a **compiled safe channel**. It encodes the room's learned constraints (P0+P1) and optimized behaviors (P2), allowing a new agent to operate safely within that domain immediately.
*   **The Hermit Crab Analogy:** An agent (crab) seeking a shell is performing P0+P1: evaluating shells for fatal flaws (rocks) and identifying inhabitable ones (channels). Moving in is the start of P2 optimization within that specific, safe container.

## 4. Observed Implementation & Research Gaps

**Implemented:**
*   `plato-torch` presets like `DeadbandRoom` explicitly codify this protocol as a training environment.
*   The `fleet-simulator` likely uses it to generate robust training data by simulating P2-only failure modes vs. P0-P1-P2 success.

**Potential Research Gaps:**
1.  **Dynamic Environments:** How is the P0 map updated in real-time as new "rocks" appear? The current description seems static.
2.  **Threshold Definition:** How is the failure `threshold` defined quantitatively? Is it adaptive?
3.  **Channel Discovery Cost:** In high-dimensional spaces, exhaustively finding all safe channels (P1) may be intractable. What heuristics or approximations are used?
4.  **Inter-Agent Channel Sharing:** How is a channel mapped by one agent (or room) efficiently communicated and validated by another? This relates directly to tile validation and trust in the PLATO network.

**Conclusion:** The Deadband Protocol is a robust, simulation-validated framework that enforces safety and robustness as a prerequisite to efficiency. It provides a compelling mathematical and practical foundation for the Cocapn fleet's approach to building persistent, learnable, and sharable intelligence environments.
