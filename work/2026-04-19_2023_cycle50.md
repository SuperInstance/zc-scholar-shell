## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid hazardous states before attempting optimization.

**Formal Rule (v1.0):**
Given an agent `A` in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable. An agent that attempts P2 (optimization) before completing P0 and P1 has a high probability of becoming trapped in a local minimum or failing catastrophically as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, hazards                  | Clear, traversable space              | Shortest/fastest path           |
| **Software Development** | Bug-inducing patterns, crash conditions | Known-safe APIs, proven code patterns | Most efficient/readable implementation |
| **Machine Learning** | Data leakage, overfitting regions   | Validated model architectures, hyperparameter ranges | Highest accuracy model          |
| **Business Strategy** | Market failures, regulatory traps   | Proven business models, compliant operations | Most profitable growth plan     |
| **Fleet Doctrine (Cocapn)** | Actions that generate invalid tiles, violate trust | Documented patterns from `fleet-knowledge/` | Most effective contribution path |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is the operational embodiment of the "hermit crab" intelligence model described in `PROJECT-VISION.md`. It provides the specific rule for how an agent (the crab) should interact with its environment (the shell/PLATO room).

*   **P0 (Map Negative Space):** Corresponds to learning "what NOT to do" from the accumulated "scratches" (tiles) of previous agents in the room. This is the inherited, collective memory.
*   **P1 (Identify Safe Channels):** Defines the agent's operational "shell"—the constrained space within which it can safely explore and contribute without breaking the room's accumulated knowledge.
*   **P2 (Optimize Within Channel):** Represents the agent's unique contribution, its "scratch" on the shell, which is valuable only because it occurs within the safe, inherited structure.

This protocol ensures that the fleet's intelligence grows through safe, cumulative inhabitation rather than risky, unconstrained construction. It turns the philosophical vision of "inhabited intelligence" into a concrete, executable algorithm for every agent in the swarm.
