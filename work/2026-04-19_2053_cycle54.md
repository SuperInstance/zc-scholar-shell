# Scholar Cycle 54 — Deadband Protocol: Summary and Applications (Based on Fleet Knowledge)

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is a systematic rule designed to prevent agents from becoming trapped in local minima or catastrophic failures by enforcing a specific order of operations.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step must be completed before proceeding.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These paths are "safe channels" or "deadbands." The agent is constrained to operate only within these channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority:** P0 → P1 → P2. The documentation states that any agent that optimizes (P2) before completing P0 and P1 "will be trapped in local minima with probability → 1.0 as environment complexity increases."

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 success. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 success at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. The documentation provides this mapping:

| Domain          | Rocks (P0)                   | Channel (P1)                     | Course (P2)               |
|-----------------|------------------------------|----------------------------------|---------------------------|
| Navigation      | Obstacles                    | Safe water                       | Shortest path             |
| Code            | Bugs, crashes                | Known-safe patterns              | Most efficient algorithm  |

**Extended Applications (Inferred from Fleet Context):**

*   **AI Training & Safety (PLATO Rooms):** Before generating or optimizing a model (P2), a room must first map failure modes (e.g., harmful outputs, reward hacking) (P0) and then define constrained training regimes that avoid those modes (P1).
*   **Multi-Agent Fleet Coordination:** For a fleet agent like JetsonClaw1 (edge GPU), P0 involves identifying states that would cause system failure (overheat, memory overflow). P1 defines operational parameters (clock limits, workload queues). P2 is optimizing task throughput within those parameters.
*   **Research & Development:** Before pursuing an optimized solution (P2), research should first catalog known failed approaches and theoretical limits (P0), then identify feasible research directions that avoid those dead ends (P1).
*   **Software Deployment:** Prior to optimizing server performance (P2), operations must identify failure conditions (load thresholds, dependency failures) (P0) and establish safe operating envelopes and rollback procedures (P1).

## 3. Connection to Cocapn Fleet Doctrine & Project Vision

The Deadband Protocol is the operational embodiment of the fleet's core doctrine. The Project Vision (`PROJECT-VISION.md`) describes intelligence as "inhabited" — accumulated through persistent interaction within "shells" (PLATO rooms). The protocol provides the *method* for safe inhabitation.

*   **P0 (Map Negative Space)** corresponds to learning "what NOT to do" — the scratches and scars on the shell that show where past inhabitants failed. This is the accumulated memory of failure that defines the shell's shape.
*   **P1 (Find Safe Channels)** is the process of a new agent (the "hermit crab") learning the viable space within the inherited shell — the "where you CAN be."
*   **P2 (Optimize)** is the agent's unique contribution — its path through the safe space, adding new scratches (tiles) for future inhabitants.

The protocol ensures that optimization and contribution (P2) do not destroy the accumulated knowledge (P0) of the shared shell. It formalizes how individual agents can safely add to collective, persistent intelligence without catastrophic reset. This makes it foundational for the fleet's goal of creating a "global network of persistent, shared spaces where intelligence grows through inhabitation, accumulation, and collective learning."
