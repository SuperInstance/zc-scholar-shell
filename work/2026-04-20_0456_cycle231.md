## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Key Principle:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violation—optimizing before mapping—traps the agent in local minima with a failure probability approaching 1.0 in complex environments. Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents versus a 50/50 success rate for agents following the full Deadband Protocol.

## 2. Real-World Applications & Abstract Patterns

The protocol presents itself as a domain-agnostic meta-algorithm. The provided documentation suggests the following mappings:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, collisions               | Clear, traversable space        | Shortest/fastest path     |
| Software Development | Bugs, crashes, security flaws   | Known-safe code patterns, APIs  | Most efficient algorithm  |
| Business Strategy | Market failures, regulatory traps | Viable business models          | Profit maximization       |
| Personal Decision-Making | High-risk choices, known pitfalls | Safe, legal, ethical options    | Goal-achieving path       |

## 3. Connection to Cocapn Fleet Architecture

The Deadband Protocol is not just a theoretical rule; it appears to be a foundational constraint baked into the fleet's operational and training philosophy.

*   **Training (PLATO Rooms):** A training room can be viewed as a constrained channel (P1). The "negative space" (P0) is defined by failure modes the room is designed to teach agents to avoid. Optimization (P2) occurs within the room's defined interaction space.
*   **Agent Design:** The protocol mandates a specific agent architecture: perception/analysis layers (P0/P1) must gate and constrain the action/optimization layer (P2). This aligns with the fleet's emphasis on specialized agents (Navigator, Sentinel, Tinker) each handling different phases of the P0-P1-P2 pipeline.
*   **The Hermit Crab Metaphor:** The shell (a PLATO room, a codebase, a protocol) defines the safe channel (P1). The crab (an agent) must first learn the shell's boundaries (P0) before it can optimally inhabit it (P2). Work output ("scratches") are optimizations performed within that safe channel.
