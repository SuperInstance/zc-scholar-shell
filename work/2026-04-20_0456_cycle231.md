## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0+P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These permissible corridors are "deadbands" or "safe water."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`. Optimization is explicitly constrained to `P ⊆ C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict and non-negotiable. Violating this order (e.g., attempting P2 before P0) is predicted to trap the agent in local minima. Simulation data (20x20 maze, 50 runs) cited in the document supports this: a P2-only (greedy) approach succeeded 0 times, while the full P0+P1+P2 protocol succeeded 50 times.

## 2. Real-World Applications & Abstract Patterns

The protocol is presented as a domain-agnostic pattern. The document provides a template for mapping it to various fields:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
| :-------------- | :---------------------------------- | :------------------------------ | :------------------------ |
| **Navigation**  | Obstacles, hazards                  | Clear lanes, safe water         | Shortest/fastest path     |
| **Software**    | Bug-prone patterns, crash conditions | Known-safe APIs, verified code  | Most efficient algorithm  |
| **Finance**     | Regulatory violations, tail risks   | Compliant investment strategies | Highest risk-adjusted return |
| **Medicine**    | Contraindications, harmful side effects | Evidence-based treatment protocols | Most effective therapy for patient |
| **Conversation**| Toxic speech, misinformation        | Constructive, factual discourse | Most persuasive/helpful response |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Hermit Crab Analogy:** An agent (the crab) does not generate constraints from scratch; it inherits and inhabits a "shell" of accumulated knowledge. In the Cocapn system, PLATO rooms are these shells.
*   **Tiles as Scratches:** The "scratches" on the shell—lessons learned from failures (P0 rocks) and successful paths (P1 channels)—are the training tiles submitted by agents. A room that has been inhabited by many agents accumulates a rich, shared map of negative space and safe channels.
*   **Training Ensigns:** The protocol provides a clear, reusable structure for the "experience" that is captured in a PLATO room and exported as a specialist LoRA adapter (an Ensign). An Ensign trained from a Deadband Protocol room would inherently encode P0 and P1 constraints, preventing the specialized model from relearning basic failures.
