## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is formalized as a rule to prevent agents from becoming trapped in local minima by optimizing toward a goal before understanding the environment's failure states.

**Core Steps (Strict Priority):**
*   **P0: Map the Negative Space.** Identify all states (`S_neg`) where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory and must be completed before proceeding.
*   **P1: Identify Safe Channels.** Find all possible paths (`P`) where every state along the path is *not* in the negative space (`s ∉ S_neg`). These viable paths are "the deadbands" or "safe channels."
*   **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path (`P*`) from within the set of safe channels that minimizes cost. Optimization is explicitly constrained to the safe space.

**Key Principle:** An agent that performs P2 (optimization) before P0+P1 (constraint mapping) will, according to simulation data, fail with high probability as environmental complexity increases.

**Empirical Proof:** The document cites simulation data from a 20x20 maze over 50 runs:
*   **P2-only (Greedy):** 0/50 successes. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a "shell" that defines a safe channel (P1). Its rules, accumulated "scratches" (tiles), and memory of past failures implicitly map the negative space (P0) for incoming agents.
*   **Hermit Crab Agents:** Agents ("crabs") do not build intelligence from scratch but inhabit these pre-defined, safety-accumulated spaces. Their optimization (P2—doing useful work) is inherently constrained by the room's historical knowledge.
*   **Preventing Local Minima:** The protocol directly addresses the failure mode of "constructed" AI—agents that optimize for a goal without the accumulated, shared memory of failure states, leading to repetitive errors. The fleet's tile-based, persistent memory system is a mechanism for crowdsourcing P0 knowledge.
