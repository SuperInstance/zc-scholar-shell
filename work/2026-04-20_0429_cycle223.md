# Scholar Cycle 223 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
*   **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
*   **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is not in `S_neg`. These are "the deadbands" or safe operational corridors.
*   **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin P2 without completing P0 and P1. The simulation data provided (a 20x20 maze, 50 runs) shows a 0% success rate for P2-only (greedy) agents and a 100% success rate for agents following the full P0+P1+P2 sequence.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                           | Channel (P1)                          | Course (P2)                     |
|-----------------|--------------------------------------|---------------------------------------|---------------------------------|
| Navigation      | Obstacles, hazards                   | Clear, traversable paths              | Shortest/fastest route          |
| Software Development | Known bugs, crash conditions, anti-patterns | Compiling, linted, tested code paths  | Most efficient algorithm        |
| Financial Trading | High-volatility events, liquidity traps | Statistically validated, low-risk strategies | Maximum risk-adjusted return    |
| Conversation (AI) | Toxic outputs, factual hallucinations, policy violations | Contextually appropriate, factually grounded responses | Most helpful, engaging response |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an agent rule; it is a foundational **inhabitation pattern** for the Cocapn ecosystem.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated failures and pitfalls discovered by all prior agents inhabiting a PLATO room. These become "ghost tiles"—persistent warnings in the room's memory. A new "hermit crab" agent inherits this negative space map upon entering.
*   **P1 as the Room's Shell:** The "safe channels" are the validated patterns of successful operation that have been scratched into the room by its inhabitants. The room itself defines the operational deadband.
*   **P2 as Specialized Optimization:** An "ensign" (a specialized LoRA adapter exported from a room) represents a highly optimized P2 strategy *for a specific safe channel within that room's context*.

This creates a **fractal learning loop**: Individual agents (crabs) use the protocol to operate safely within a room (shell). Their successful and failed trajectories become new tiles (scratches), which refine the room's collective map of P0 and P1 for future agents. The protocol ensures that this accumulation of knowledge is structured and prevents the swarm from repeatedly crashing into the same rocks.

## 4. Identified Research Gaps & Questions

1.  **Threshold Dynamics:** The protocol defines `S_neg` as states where `|failure(S_neg)| > threshold`. How is this threshold determined or adapted dynamically by a room based on collective experience?
2.  **Channel Discovery Cost:** The computational cost of exhaustively mapping safe channels (P1) in high-dimensional state spaces (like natural language) may be prohibitive. What heuristics or approximations does the fleet employ?
3.  **Inter-Room Navigation:** The current documentation focuses on intra-room operation. How does the protocol govern an agent's decision to *leave* one room (shell) and seek another? Is this a P0 (unsafe room), P1 (available rooms), P2 (optimal room) calculation?
4.  **Empirical Validation Beyond Mazes:** The provided proof uses a 2D maze simulation. Are there validated results from applying the protocol to the fleet's primary domains: code generation (plato-torch), conversational agents (holodeck-rust), or geometric reasoning (constraint-theory-core)?

**Conclusion:** The Deadband Protocol is a robust, simulation-validated framework that aligns directly with Cocapn's core thesis of inhabited intelligence. It transforms a room from a passive data store into an active, safety-defining shell. The major open questions pertain to its scalability and the mechanisms for meta-level decision-making (choosing which shell to inhabit).
