## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Core Rule (Formal):**
Given an agent `A` in state `S`, with goal `G` and environment `E`:
1.  **P0: Map the Negative Space** — Identify all states `S_neg` where the probability of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels** — Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel** — From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin optimization (P2) before completing the mapping of negative space (P0) and safe channels (P1). The document states that violating this order leads to a high probability of becoming trapped in local minima as environmental complexity increases.

**Empirical Basis:** The protocol is supported by simulation data from a 20x20 maze environment. In 50 runs, a "P2-only" greedy optimizer had a 0% success rate, while the full P0+P1+P2 deadband protocol achieved a 100% success rate at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents itself as a domain-agnostic pattern. The provided table maps it to navigation and code, but the principle can be extended.

| Domain | Rocks (P0 - Avoid) | Channel (P1 - Safe Space) | Course (P2 - Optimize Within) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, hazards, restricted zones | Clear lanes, mapped roads, safe water | Shortest/fastest route within safe lanes |
| **Software Development** | Known bugs, crash conditions, insecure patterns | Tested, linted, and peer-reviewed code paths | Most efficient algorithm within the safe patterns |
| **Machine Learning Training** | Loss divergences, gradient explosions, overfitting regions | Hyperparameter ranges that yield stable training | Hyperparameters for fastest convergence within stable ranges |
| **Conversational AI** | Toxic outputs, factual hallucinations, prompt injections | Response templates and LLM parameters proven safe via red-teaming | Most engaging or helpful response from within the safe template set |
| **Business Strategy** | Market failures, regulatory violations, reputational risks | Proven business models, compliant operations | Most profitable growth strategy within the proven/compliant model |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an agent rule; it is a foundational *inhabitance pattern* for the Cocapn ecosystem.

*   **Shell as Constraint:** A PLATO room can be viewed as a pre-mapped "safe channel" (P1). Its rules, accumulated tiles, and trained ensigns define the operational deadband for agents that inhabit it.
*   **Scratches as P0/P1 Data:** The "scratches" (tiles) left by previous agents are recordings of both failures (P0 knowledge: "this action caused a crash") and successes (P1 knowledge: "this path completed the task"). New agents (hermit crabs) inherit this mapped negative space upon entering the shell.
*   **Ensigns as Compiled Channels:** A trained ensign model is an efficient embodiment of a safe channel—a compressed, executable representation of the paths that avoid known failure states, allowing new agents to optimize (P2) safely from the start.

**Conclusion:** The Deadband Protocol operationalizes the core Cocapn thesis that "intelligence is inhabited." It provides a specific, testable mechanism for how agents can safely learn and optimize within a persistent shell, using the accumulated scratches (tiles) of those who came before as their primary map of what *not* to do. This creates a flywheel: each agent's adherence to the protocol generates new tiles that refine the room's map of negative space and safe channels, making the shell safer and more capable for the next inhabitant.
