## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are "the deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. An agent that attempts P2 (optimization) before completing P0 and P1 is described as having a failure probability approaching 1.0 as environmental complexity increases.

**Empirical Proof:** The protocol cites simulation data from a 20x20 maze over 50 runs. A "P2-only" (greedy optimization) agent succeeded 0 times. The "P0+P1+P2" (Deadband) agent succeeded 50 times while maintaining optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table maps it to several domains:

| Domain          | Rocks (P0)                     | Channel (P1)                    | Course (P2)               |
|-----------------|--------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, cliffs              | Safe water, clear paths         | Shortest path             |
| Code            | Bugs, crashes, security flaws  | Known-safe patterns, libraries  | Most efficient algorithm  |
| Conversation    | Offensive topics, misinformation | Polite, factual topics         | Most persuasive argument  |
| Business        | Legal violations, market risks | Compliant, proven strategies    | Highest profit plan       |
| Learning        | Concepts that cause confusion  | Prerequisite mastery            | Fastest path to expertise |

**Core Insight:** The protocol inverts a common failure mode. Instead of defining a goal and seeking the shortest path to it (which often leads into a failure state), it first defines the space of guaranteed failure and then seeks the best path that remains outside that space.
