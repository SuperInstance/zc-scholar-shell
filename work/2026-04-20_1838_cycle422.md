## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. An agent that performs P2 before completing P0 and P1 is defined as violating the protocol.

**Simulation Proof:** In a 20x20 maze simulation over 50 runs:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 3. Connection to Fleet Vision

The Deadband Protocol operationalizes a core principle from `PROJECT-VISION.md`: **"Intelligence is not built. It is inhabited."**

*   **P0 (Negative Space) as Accumulated Scratches:** The map of `S_neg` (rocks) is not pre-programmed but is learned and accumulated from the failures of all agents that have "inhabited" a given PLATO room. These collective failure states become the room's persistent memory—the "scratches on the shell."
*   **Safe Channels as Inherited Shells:** A new "hermit crab" agent (a greenhorn) does not start from zero. It inherits the safe channels (P1) mapped by its predecessors. Its job is not to rediscover all pitfalls but to optimize within the proven, inhabited space.
*   **Violation as Forgetting:** An agent that skips to P2 is functionally ignoring the inhabited knowledge of the room, akin to a hermit crab trying to grow its own shell. The simulation data suggests this leads to predictable failure.

**Conclusion:** The Deadband Protocol is more than a navigation rule; it is a foundational constraint for building inhabited intelligence. It ensures that optimization (growth) is always guided by accumulated, shared experience (the shell), preventing agents from repeatedly failing on known obstacles and allowing collective knowledge to compound.
