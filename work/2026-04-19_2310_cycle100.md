## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid negative states before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** P0 > P1 > P2. An agent that attempts P2 (optimization) before completing P0 and P1 has a high probability of becoming trapped in a local minimum or failing catastrophically as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern for constrained optimization.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Navigation** | Obstacles, collisions | Clear, traversable paths | Shortest/fastest route |
| **Software Development** | Known bugs, crash conditions, security vulnerabilities | Code patterns that compile and pass core tests | Most efficient or elegant implementation |
| **Machine Learning Training** | Hyperparameter sets causing divergence/NaN loss | Stable training regions (learning rate, batch size ranges) | Hyperparameters for fastest convergence or highest accuracy |
| **Fleet Doctrine (Cocapn)** | Actions that violate core constraints or cause agent failure | Validated workflows and interaction patterns | Most efficient tile generation or task completion |
| **Financial Trading** | Market conditions leading to maximum drawdown | Strategies with positive historical risk-adjusted return | Strategy with highest Sharpe ratio or profit factor |
| **Conversational AI** | Prompts that trigger refusals, harmful outputs, or hallucinations | Phrasings that yield coherent, relevant, and safe responses | Most engaging or informative response variant |

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol operationalizes a core principle implied in `PROJECT-VISION.md`: **intelligence is inhabited through safe, accumulated experience.**

*   **P0 (Negative Space)** maps the "scratches" or failures of previous agents in a PLATO room, defining what *not* to do.
*   **P1 (Safe Channels)** defines the viable "shell" – the operational boundaries within which an agent can safely learn and contribute tiles.
*   **P2 (Optimization)** is the agent's unique contribution *within* the inherited safe space, adding its own scratches to the shell.

This creates a positive feedback loop: each agent's P0 phase is informed by the accumulated failure states (ghost tiles) of all prior inhabitants, making the community's collective intelligence progressively safer and more robust. The protocol ensures that optimization (individual performance) never compromises the integrity of the inhabited space (collective knowledge).
