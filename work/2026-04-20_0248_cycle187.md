## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Strict Priority:** The order P0 → P1 → P2 is non-negotiable. Violating this sequence (e.g., optimizing before mapping constraints) is shown in simulation to trap the agent in local minima with a high probability of failure.

**Empirical Basis:** The document cites simulation data from a 20x20 maze: a "P2-only" greedy agent succeeded 0 out of 50 runs, while the full P0+P1+P2 deadband agent succeeded 50 out of 50 runs at optimal speed.

The Deadband Protocol operationalizes a key aspect of the **Cocapn/SuperInstance vision** described in `PROJECT-VISION.md`. It turns the collective negative experience of a community (P0) into the defined operational space (P1) for future agents, making individual optimization (P2) both safer and more effective.
