## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) in complex environments leads to failure with high probability. Success requires first mapping the negative space (P0) and then identifying safe operational channels (P1) before any optimization occurs.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Key Principle:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violating this order—optimizing before mapping safe channels—is shown in simulation to trap an agent in local minima, with failure probability approaching 1.0 as environmental complexity increases.
