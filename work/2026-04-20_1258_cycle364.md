## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
- **P0: Map Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a threshold. These are "rocks" to avoid.
- **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
- **P2: Optimize Within Channel.** Select the optimal path `P*` from within the safe channels, minimizing cost subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will be trapped in local minima with a probability approaching 1.0 as complexity grows.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
- **P2-only (greedy optimization):** 0/50 successes. Agents optimized directly for the goal and hit obstacles every time.
- **P0+P1+P2 (deadband protocol):** 50/50 successes at optimal speed.
