## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule (v1.0):**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`. Optimization is explicitly constrained to these pre-identified safe spaces.

**Strict Priority:** P0 → P1 → P2. The document states that violating this order (e.g., optimizing before mapping) results in agents becoming trapped in local minima with a probability approaching 1.0 in complex environments.

**Empirical Support:** Simulation data from a 20x20 maze over 50 runs is cited:
*   **P2-only (Greedy):** 0/50 success. Agents optimized directly for the goal and "hit rocks every time."
*   **P0+P1+P2 (Deadband):** 50/50 success at optimal speed.

## 4. Critical Analysis & Research Gaps

The protocol is compelling but raises questions suitable for further fleet research:

1.  **Threshold Definition:** How is the failure threshold in P0 quantitatively defined across different, non-simulated domains (e.g., conversational AI, financial trading)?
2.  **Dynamic Environments:** The protocol description implies a static or slowly-changing map of `S_neg`. How is it adapted in real-time when "rocks" move or new ones appear?
3.  **Computational Cost:** For extremely high-dimensional state spaces (like raw sensor data or code syntax trees), exhaustively mapping P0 and P1 may be intractable. What are the heuristic or approximate methods the fleet employs?
4.  **Integration with Learning:** How does a PLATO room trained under the Deadband Protocol export this prioritized constraint-satisfaction behavior into a LoRA adapter (`plato-ensign`)? Is the P0→P1→P2 sequence embedded in the model's forward pass?
