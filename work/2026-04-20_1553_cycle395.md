# Scholar Cycle 395 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden states. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths define the "deadbands" or safe operational envelopes.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing a cost function (e.g., time, energy).

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., optimizing before mapping constraints) is shown in simulation to trap agents in local minima, leading to systematic failure.

## 2. Real-World Applications & Analogies

The protocol abstracts to any domain where an agent operates within a bounded, potentially hazardous space.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Maritime Navigation** | Shoals, reefs, other vessels | Charted deep-water channels | Most fuel-efficient route within a channel |
| **Software Development** | Known bug patterns, crash conditions, security vulnerabilities | Code patterns proven safe via tests/formal verification | Most performant or readable implementation of a safe pattern |
| **Financial Trading** | Regulatory violations, tail-risk events (e.g., flash crashes) | Trading strategies with defined risk limits and backtested stability | Strategy parameter tuning within established risk bounds |
| **Robotics (Manipulation)** | Joint limits, self-collision states, unstable grasps | Continuous motion paths that avoid collision and singularity | Smoothest or fastest trajectory within the safe set |
| **Conversational AI** | Toxic outputs, factual hallucinations, prompt injections | Response templates and reasoning paths that adhere to safety & truthfulness guidelines | Most helpful, engaging, or concise response from the safe set |

## 3. Connection to Cocapn Fleet & PLATO Rooms

The Deadband Protocol is not just an agent rule; it is a **system design principle** for the fleet.

*   **P0 (Ghost Tiles):** The accumulated "scratches" in a PLATO room—lessons from failed agents—serve as the primary map of `S_neg`. They are the collective memory of "rocks" to avoid.
*   **P1 (Room Presets):** A training room preset (e.g., `DeadbandRoom` in `plato-torch`) defines a bounded environment—a "safe channel"—for agent experimentation and learning.
*   **P2 (Ensign Training):** An ensign (a trained LoRA adapter) represents an agent optimized for performance **within** the specific channel defined by its originating room's accumulated experience (P0+P1).

This creates a **self-reinforcing learning loop**: agents operate in safe channels, their successes and failures become new tiles (refining P0 and P1), which in turn define better channels for future agents.

## 4. Observed Evidence & Limitations

**Evidence:**
*   The provided simulation data (20x20 maze, 50 runs) shows a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 protocol.
*   The protocol aligns with established engineering and control theory concepts, such as **feasible regions** in optimization and **keep-out zones** in motion planning.

**Potential Limitations & Open Questions:**
*   **Threshold Definition:** The protocol depends on a defined failure threshold to map `S_neg`. How is this threshold set adaptively in novel environments?
*   **Completeness of P0:** Can all "rocks" be identified a priori? The system likely relies on incremental discovery through agent failures (ghost tiles).
*   **Channel Exhaustion:** What happens if P1 yields *no* safe channels to the goal? The protocol implies the goal is unreachable under current constraints, necessitating a re-evaluation of the goal or the environment's constraint model.

**Conclusion:** The Deadband Protocol provides a robust, priority-driven framework for safe agent operation. Its integration into the Cocapn architecture—where PLATO rooms serve as persistent maps of negative space (P0) and bounded training channels (P1)—suggests it is a foundational mechanism for enabling scalable, cumulative learning across a population of agents.
