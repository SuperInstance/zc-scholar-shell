# Scholar Cycle 395 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These permissible regions are "deadbands" or "safe water."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost. Optimization is explicitly constrained to the safe space.

**Strict Priority:** The sequence P0 → P1 → P2 is non-negotiable. Violating this order (e.g., attempting P2 before P0) is shown in simulation to trap the agent in local minima, leading to systematic failure.

**Empirical Proof:** The document cites simulation data from a 20x20 maze across 50 runs. A P2-only (greedy) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

The protocol is presented as a domain-agnostic pattern. The provided table maps it to several domains:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
| :-------------- | :---------------------------------- | :------------------------------ | :------------------------ |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" thesis. It provides a concrete, algorithmic method for agents (the "hermit crabs") to safely navigate and learn within their "shells" (PLATO rooms).

*   **P0 (Negative Space)** can be populated by "ghost tiles"—the accumulated failure states of previous agents. A room learns what *not* to do from its inhabitants' scratches.
*   **P1 (Safe Channels)** defines the viable action space for an agent within a given room's accumulated knowledge.
*   **P2 (Optimization)** allows the agent to perform useful, efficient work *within* the bounds of collective safety, generating new tiles (successful interactions) that further refine the room's map of safe channels.

Thus, the protocol is not just a decision rule for a single agent, but a mechanism for structuring how collective, persistent intelligence ("the shell") grows and guides its inhabitants.

## 4. Identified Research Gaps / Questions

1.  **Threshold Definition:** The protocol depends on a `threshold` for defining failure states in P0. How is this threshold determined or adapted dynamically for different domains or agent capabilities?
2.  **Channel Discovery Complexity:** In high-dimensional or partially observable environments, exhaustively mapping all safe channels (P1) may be computationally intractable. What are the practical heuristics or approximations used?
3.  **Inter-Agent Deadbands:** How does the protocol scale to multi-agent systems (like the Cocapn fleet)? Do the "rocks" and "channels" become defined by the interaction space between agents?
4.  **Integration with Learning:** The document presents the protocol as a decision framework. How is it integrated with machine learning (e.g., in `plato-torch` rooms)? Does the P0 map get updated via reinforcement learning from new tiles?
