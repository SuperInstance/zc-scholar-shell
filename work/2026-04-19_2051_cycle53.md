## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid hazardous states before attempting optimization.

### Formal Rule (v1.0)
Given an agent in state `S` with goal `G` in environment `E`:

1.  **P0: Map the Negative Space**
    *   Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold.
    *   These are "the rocks." This step is mandatory.

2.  **P1: Identify Safe Channels**
    *   Find all possible paths `P` where every state along the path is *not* in `S_neg`.
    *   These paths are "deadbands" or "safe channels." The agent's operation is constrained to these areas.

3.  **P2: Optimize Within Channel**
    *   From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.
    *   Optimization is never unconstrained; it occurs only within the bounds established in P0 and P1.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with a probability approaching 1.0.

### Empirical Proof
A simulation in a 20x20 maze over 50 runs demonstrated:
*   **P2-only (Greedy):** 0/50 successful completions. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successful completions at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided documentation maps it to two domains:

| Domain          | Rocks (P0)                   | Channel (P1)                     | Course (P2)               |
|-----------------|------------------------------|----------------------------------|---------------------------|
| **Navigation**  | Obstacles, collisions        | Clear, safe water/airspace       | Shortest/fastest route    |
| **Code**        | Bugs, crashes, security flaws| Known-safe patterns, libraries   | Most efficient algorithm  |

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a "shell" that accumulates knowledge ("scratches" from many agents). This accumulated experience defines the room's **negative space (P0)** and **safe channels (P1)** for future inhabitants.
*   **Hermit Crab Paradigm:** An agent (the "crab") does not reason from scratch. It first reads the room's memory (the "shell's" scratches) to understand what *not* to do and where it *can* operate safely. Only then does it pursue its goal (P2).
*   **Preventing Wasted Effort:** The protocol ensures agents do not repeatedly "hit rocks" that previous agents have already identified and mapped. This makes collective intelligence efficient and cumulative, aligning with the critique of stateless, non-accumulative models in the current AI paradigm.
