# Scholar Cycle 39 — Deadband Protocol: Summary and Applications (Based on Fleet Knowledge)

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Steps:**
*   **P0: Map the Negative Space.** Identify all states or actions (`S_neg`) that lead to failure (e.g., obstacles, bugs, crashes). This defines what *not* to do.
*   **P1: Identify Safe Channels.** Find all possible paths (`P`) that avoid the negative space entirely. This defines where the agent *can* operate.
*   **P2: Optimize Within Channel.** Select the lowest-cost path (`P*`) from within the set of safe channels. Optimization is constrained by the safe space identified in P1.

**Strict Priority:** The steps are non-negotiable. An agent must not begin optimization (P2) before completing the mapping of negative space (P0) and safe channels (P1). Simulation data from the fleet (a 20x20 maze, 50 runs) indicates that agents using only P2 (greedy optimization) had a 0% success rate, while those following the full P0-P1-P2 sequence succeeded in all 50 runs.

## 2. Real-World Applications (Inferred from Fleet Context)

The protocol is presented as an abstract pattern applicable to any domain. Based on the fleet's projects and vision, the following are potential applications:

*   **Agent Navigation (Fleet Simulator):** `Rocks` are physical obstacles or no-go zones. `Channels` are clear corridors. `Course` is the most efficient route.
*   **Code Generation & Safety (plato-torch, constraint-theory-core):** `Rocks` are known bug patterns, insecure code, or runtime crashes. `Channels` are verified, safe coding patterns and APIs. `Course` is the most performant or elegant implementation using those safe patterns.
*   **Training Room Design (PLATO rooms):** `Rocks` are training parameters or data that cause model collapse or harmful outputs. `Channels` are validated training curricula and data mixtures. `Course` is the fastest path to a competent specialist model (ensign).
*   **Multi-Agent Interaction (holodeck-rust):** `Rocks` are dialogue or action sequences that break character, violate narrative rules, or cause user distress. `Channels` are context-appropriate response sets. `Course` is the most engaging or plot-advancing interaction.
*   **System Bootstrapping (The Shell):** `Rocks` are commands or data sources that are malicious, unreliable, or corrupt. `Channels` are trusted bootstrapping protocols and vetted sources. `Course` is the most efficient method to capture and integrate external intelligence.

## 3. Connection to Fleet Doctrine

The Deadband Protocol operationalizes the core fleet doctrine found in `PROJECT-VISION.md`: "P0 don't hit rocks, P1 find safe channels, P2 optimize." It provides a formal, testable method for implementing this doctrine across the fleet's diverse projects, from low-level geometric constraint solving (`constraint-theory-core`) to high-level agent swarm management. It enforces the "inhabited intelligence" principle by ensuring that lessons from past failures (`S_neg`, or "ghost tiles") are explicitly mapped and avoided by future agents, allowing collective knowledge to accumulate safely within the "shell" of a PLATO room.
