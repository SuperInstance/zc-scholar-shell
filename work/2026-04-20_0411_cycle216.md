## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) in a complex environment leads to failure with high probability. Success requires first mapping the negative space (P0) and then identifying safe operational channels (P1).

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`. Optimization is constrained to these pre-validated spaces.

**Strict Priority:** The steps are non-negotiable and sequential. Violation (e.g., optimizing before mapping) is shown in simulation to lead to failure ("trapped in local minima with probability → 1.0 as environment complexity increases").

**Empirical Support:** The document cites simulation data from a 20x20 maze: a P2-only (greedy) agent succeeded 0 out of 50 runs, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 runs at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table maps it to several domains:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, collisions               | Safe water, clear lanes         | Shortest path             |
| Code            | Bugs, crashes, security vulns       | Known-safe patterns, libraries  | Most efficient algorithm  |
| Conversation    | Offensive topics, misinformation    | Polite, factual discourse       | Most persuasive argument  |
| Business        | Regulatory fines, market failures   | Compliant, proven strategies    | Highest profit margin     |
| Learning        | Concepts that cause misunderstanding| Prerequisite mastery            | Fastest path to expertise |

**Key Insight:** The protocol inverts a common approach. Instead of "find the best path and avoid obstacles along the way," it mandates "first, exhaustively define where you *cannot* go; second, define where you *can* go; third, and only then, pick the best route within the allowed area." This prioritizes survivability and reliability over raw efficiency.
