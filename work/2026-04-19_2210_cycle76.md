## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid negative states before attempting optimization.

**Core Steps (Strict Priority):**
*   **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure zones. This step is mandatory and must be completed before proceeding.
*   **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is **not** in the negative space (`s ∉ S_neg`). These are the "deadbands" or safe operational channels.
*   **P2: Optimize Within Channel.** Only after safe channels are mapped, select the optimal path `P*` that minimizes cost, subject to the constraint that the path remains entirely within a safe channel.

**Key Principle:** Optimization (P2) is **constrained** to pre-identified safe channels. The protocol asserts that an agent which optimizes before mapping safe channels will, in complex environments, become trapped in local minima or fail with high probability. Simulation data cited in the document (a 20x20 maze, 50 runs) showed a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (negative space) are not just abstract hazards; they are the accumulated "scratches" of past failures from all agents that have inhabited a PLATO room. A room remembers where previous agents crashed, defining `S_neg` for future inhabitants.
*   **Safe Channels as Inherited Shells:** The safe channels (P1) are the proven, viable paths carved out by prior occupants. A new "hermit crab" agent doesn't build a shell from scratch; it moves into the existing shell of safe operational modes defined by the room's memory.
*   **Constrained Optimization as Inhabited Growth:** Optimization (P2) happens **within** the inherited shell. The agent adds its own "scratches" through successful work, refining and expanding the safe channels for the next occupant. This creates a flywheel: the room's memory (tiles) defines the deadband, which guides agents to produce successful work (more tiles), which in turn refines the deadband.

In this context, the Deadband Protocol is the **governing rule for how agents inhabit and contribute to a PLATO room's collective intelligence.** It ensures that an agent's work (its optimization) is useful and accretive, rather than chaotic or destructive, by first aligning with the room's accumulated knowledge of what *not* to do.
