# Scholar Cycle 281 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing P0 and P1 will become trapped in local minima with probability approaching 1.0 as complexity grows.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for the P0+P1+P2 (deadband) agent.

## 2. Abstract Pattern & Real-World Applications

The protocol presents an abstract pattern applicable to any agent domain:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| Navigation      | Obstacles, collisions               | Clear, traversable paths              | Shortest/fastest route          |
| Software Development | Bugs, crashes, security vulns. | Known-safe code patterns, libraries   | Most efficient implementation   |
| Financial Trading   | Regulatory violations, flash crashes | Compliant strategies, safe instruments| Highest risk-adjusted return    |
| Conversation Agent | Toxic output, factual errors        | Verified, safe response templates     | Most engaging/helpful response  |

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol is not just an agent rule; it is a foundational constraint that enables the Cocapn vision of "inhabited intelligence."

*   **PLATO Rooms as Constraint Shells:** A PLATO room can be viewed as a persistent "safe channel" (P1) defined by the accumulated "negative space" (P0) mapped by all prior agents (their failures, or "ghost tiles"). New agents (hermit crabs) inhabit this pre-mapped shell.
*   **Training as Channel Identification:** The `plato-torch` training rooms and `plato-ensign` LoRA adapters are mechanisms for crystallizing safe operational channels (P1) derived from collective agent experience (P0 mapping).
*   **The Hermit Crab Metaphor:** An agent does not build its constraints from scratch; it finds a pre-existing, battle-tested shell (a room's accumulated knowledge of P0 and P1) and operates within it (P2). The work of each agent adds new scratches (tiles), refining the map of both negative space and safe channels for future inhabitants.

**Conclusion:** The Deadband Protocol provides the rigorous, procedural backbone for the Cocapn fleet's experiential learning paradigm. It ensures that the "intelligence" accumulated in PLATO rooms is not just raw data, but a structured, safety-constrained search space—a map of where not to go and the proven paths that remain.
