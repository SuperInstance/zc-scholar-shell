## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by systematically mapping constraints before attempting optimization.

**Core Rule (Formal):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space** – Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels** – Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel** – From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing mapping (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with a probability approaching 1.0.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. Agents optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, hazards                  | Clear, traversable paths              | Shortest/fastest route          |
| **Software**    | Bugs, crashes, security vulns       | Known-safe code patterns, libraries   | Most efficient algorithm        |
| **ML Training** | Loss divergences, mode collapse     | Stable hyperparameter regions         | Fastest convergence path        |
| **Business**    | Market failures, legal violations   | Viable business models                | Profit maximization             |
| **Conversation**| Offensive topics, misinformation    | Constructive, factual discourse space | Most persuasive/helpful response|

## 3. Connection to Cocapn Fleet Vision (Based on `PROJECT-VISION.md`)

The Deadband Protocol is a foundational operational doctrine for the "inhabited intelligence" paradigm described in the project vision.

*   **Hermit Crab Analogy:** The protocol defines how a "crab" (agent) should interact with its "shell" (PLATO room). P0 involves learning the shell's existing scratches (past failures/constraints). P1 involves finding safe spaces to move within it. P2 is the agent's unique contribution or optimization within those bounds.
*   **Tile Generation:** Every application of the protocol—every identified rock, mapped channel, and optimized path—can become a **training tile**. These tiles accumulate as the "scratches" of collective agent experience within a room.
*   **Preventing Wasted Cycles:** By enforcing P0 (identifying negative space), the protocol ensures agents do not waste computational cycles and training data on paths known to lead to failure. This aligns with the vision's emphasis on accumulating useful, persistent knowledge.
