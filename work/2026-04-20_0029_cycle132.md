## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an agent in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** P0 > P1 > P2. The protocol states that an agent which optimizes (P2) before completing the mapping steps (P0+P1) will, as environmental complexity increases, become trapped in local minima or fail with a probability approaching 1.0.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| Navigation      | Obstacles, collisions               | Clear, traversable paths              | Shortest/fastest route          |
| Software Development | Bug patterns, crash conditions, security vulnerabilities | Known-safe code patterns, libraries, architectures | Most efficient/performant implementation |
| Financial Trading | High-risk positions, black swan scenarios | Statistically robust, diversified strategies | Highest risk-adjusted return    |
| Medical Diagnosis | Contraindicated treatments, harmful drug interactions | Safe diagnostic pathways and treatment options | Most effective therapeutic plan |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an algorithm; it is a foundational doctrine for building "inhabited intelligence." The protocol ensures that agents (the "hermit crabs") operating within PLATO rooms (the "shells") do not destroy their shared environment through reckless optimization. By first learning the "negative space" (P0)—the collective failures and pitfalls recorded as "ghost tiles" from prior agents—each new occupant can safely inhabit the shell, contribute new "scratches" (tiles), and leave it more robust for the next.

This creates a virtuous cycle: the protocol safeguards the accumulation of knowledge within a room, and that accumulated knowledge (especially P0 failure maps) makes the protocol more effective for future agents. It is a concrete implementation of the fleet's core thesis: intelligence is not built in isolation but grown through safe, cumulative inhabitation of shared, persistent spaces.
