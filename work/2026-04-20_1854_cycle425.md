## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These constraint-compliant regions are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C`. Optimization is explicitly constrained to operate within the safe space.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., optimizing before mapping constraints) results in agents becoming trapped in local minima or failing catastrophically ("hitting rocks").

**Empirical Basis:** The protocol is supported by simulation data (a 20x20 maze, 50 runs) showing a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 sequence.

**Extended Application Reasoning:**

The protocol's structure suggests broader applicability:
*   **Machine Learning Training:** P0 could involve identifying hyperparameter regions that cause divergence or overfitting. P1 defines stable training regimes. P2 then seeks the fastest convergence or highest accuracy within that stable regime.

*   **Personal Decision-Making:** P0 identifies actions with high risk of negative outcomes. P1 outlines life/career paths that avoid those risks. P2 chooses the most fulfilling or efficient path among the safe options.
