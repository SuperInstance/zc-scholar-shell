## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid negative outcomes before attempting optimization.

**Core Steps (Strict Priority):**
*   **P0: Map the Negative Space.** The agent must first identify all states or actions (`S_neg`) that lead to failure with a probability above a defined threshold. These are the "rocks."
*   **P1: Identify Safe Channels.** The agent then maps all possible paths (`P`) where every state is *not* in the negative space (`s ∉ S_neg`). These paths are the "deadbands" or safe operational channels.
*   **P2: Optimize Within Channel.** Only after P0 and P1 are complete can the agent select the optimal path (`P*`) from within the set of safe channels, minimizing cost or maximizing reward.

**Key Principle:** Optimization (P2) is **constrained** to safe channels. The protocol asserts that an agent which optimizes before mapping safe channels will, in complex environments, become trapped in local minima or fail with a probability approaching 1.0.

**Empirical Basis:** The summary cites simulation data from a 20x20 maze, where a greedy (P2-only) strategy succeeded 0 out of 50 runs, while the full Deadband Protocol succeeded 50 out of 50 runs.

## 3. Connection to Cocapn Project Vision

The Deadband Protocol operationalizes a core safety principle implied by the `PROJECT-VISION.md`. If intelligence is "inhabited" through accumulated experiences (tiles) in shared spaces (PLATO rooms), then it is critical to filter those experiences. The protocol provides a method for agents and the room system itself to:
1.  **P0:** Reject or tag tiles that represent failures ("scratches" that weaken the shell).
2.  **P1:** Build a corpus of known-good interactions that form a reliable knowledge base.
3.  **P2:** Optimize the learning process (e.g., via `plato-ensign`) using only this vetted corpus.

This creates a self-reinforcing cycle: safer agent operation produces safer training tiles, which train safer ensign models, which then operate more safely.

## 4. Identified Research Gaps

Based on the summary documents, open questions for the fleet include:
*   **Threshold Definition:** How is the failure probability "threshold" for P0 determined or adapted dynamically per domain?
*   **Channel Discovery:** In high-dimensional state spaces (like natural language), how is the complete set of safe channels (P1) efficiently mapped?
*   **Protocol Overhead:** What is the performance cost of the P0/P1 phases, and in what scenarios might a heuristic or approximated deadband be necessary?
*   **Multi-Agent Deadbands:** How do safe channels interact when multiple agents (e.g., the Cocapn fleet) operate concurrently? Can one agent's P0 map become another's P1 channel?
