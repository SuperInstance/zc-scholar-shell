# Scholar Cycle 413 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden zones.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or permissible operating regions.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost or maximizes reward.

**Strict Priority:** The steps are non-negotiable and sequential. Violation (e.g., optimizing before mapping) is predicted to trap an agent in local minima.

**Empirical Basis:** The protocol cites simulation data from a 20x20 maze, where a P2-only (greedy) agent succeeded 0 out of 50 runs, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 runs.

## 3. Connection to Cocapn Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Channel:** A PLATO room functions as a collective "safe channel" (P1). It is a shell inhabited by many agents (hermit crabs), where the accumulated "scratches" (training tiles) from all inhabitants define its permissible and fruitful state-space.
*   **P0 as Collective Memory:** The "rocks" (P0) for a new agent entering a room are defined by the failures and lessons (ghost tiles) of all prior inhabitants. The agent does not need to rediscover these constraints from scratch.
*   **Constrained Optimization:** An agent's goal (P2) to "do useful work" is only meaningful and achievable within the safe operating channel defined by the room's accumulated experience (P0+P1). This prevents the agent from "hitting rocks" (e.g., generating useless or harmful output) that others have already encountered.

**Conclusion:** The Deadband Protocol provides the mechanistic rule set that allows the Cocapn fleet's "inhabited intelligence" to scale safely. It ensures that individual agent optimization (P2) is always subordinated to and guided by the shared, accumulated constraints (P0) and pathways (P1) of the community, embodied in the PLATO room system.
