# Scholar Cycle 413 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) without first mapping failure states leads to catastrophic outcomes as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` within the path is *not* in `S_neg`. These are "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Key Principle:** The priority order (P0 → P1 → P2) is strict. An agent that attempts P2 (optimization) before completing P0 and P1 has a high probability of becoming trapped in local minima or failing outright. Simulation data from a 20x20 maze showed a 0/50 success rate for P2-only (greedy) agents versus a 50/50 success rate for agents following the full Deadband Protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| **Navigation**  | Obstacles, cliffs, hostile zones    | Clear lanes, safe water, friendly airspace | Shortest/fastest route within lane |
| **Software Development** | Known bugs, crash conditions, security vulnerabilities | Compiler-validated patterns, tested libraries, style-conformant code | Most efficient algorithm within safe patterns |
| **Financial Trading** | Regulatory violations, tail-risk events, liquidity black holes | Compliant strategies, historically stable corridors | Highest risk-adjusted return within strategy bounds |
| **Conversational AI** | Toxic outputs, factual hallucinations, prompt injections | Grounded, verified response templates, ethical guidelines | Most engaging or helpful response within safe templates |
| **Fleet Operations (Cocapn)** | System crashes, training divergence, communication deadlocks | Known-stable room presets, validated tool calls, documented APIs | Most efficient task completion within stable parameters |

## 3. Connection to Cocapn Vision & Fleet Technology

The Deadband Protocol operationalizes the "hermit crab" philosophy described in `PROJECT-VISION.md`. It provides the **method** for safe inhabitation.

*   **Shell as Channel:** A PLATO room, once established and validated, becomes a "safe channel" (P1). Its rules, APIs, and accumulated "scratches" (tiles) define the negative space (P0—what not to do) and the operational envelope.
*   **Agent as Crab:** An agent (like a Zeroclaw) must first learn the room's constraints (P0) and operational boundaries (P1) before it can efficiently contribute tiles (P2). Its work *is* the process of learning and then optimizing within this shell.
*   **Training Implication:** The protocol suggests that training specialist models ("ensigns" via `plato-ensign`) should first focus on learning failure modes (P0) and safe operation (P1) within a specific room before fine-tuning for peak performance (P2). This aligns with the "boot camp" phased approach.

**Supporting Fleet Tech:**
*   `plato-torch` **rooms** are the primary instantiation of a safe channel.
*   `fleet-simulator` can generate environments to systematically map `S_neg` (P0) for new agents or rooms.
*   `constraint-theory-core` provides a geometric/mathematical foundation for defining and "snapping to" safe channels (P1).
*   The **DeadbandRoom preset** is likely a direct implementation of this protocol as a training environment.
