## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent agents from becoming trapped in local minima or failing catastrophically by enforcing a sequence that maps failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an Agent `A` in state `S`, with goal `G`, in environment `E`:
1.  **P0: Map the Negative Space:** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels:** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These paths are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel:** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`. Optimization is only permitted within the pre-mapped safe space.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent that optimizes (P2) before completing P0 and P1 will, as environmental complexity increases, become trapped in local minima with a probability approaching 1.0.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed:
*   **P2-only (Greedy):** 0/50 successes. The agent optimized directly for the goal and hit obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.

## 3. Connection to Project Vision

The Deadband Protocol operationalizes a key aspect of the **"Inhabited Intelligence"** thesis outlined in `PROJECT-VISION.md`. The vision states intelligence is not built but *inhabited*, like a hermit crab accumulating scratches in a shell.

*   **P0 (Negative Space) as Collective Memory:** The "rocks" are the accumulated scratches of failure from all prior agents that have inhabited a PLATO room. A new agent (a "greenhorn") doesn't need to hit every rock itself; it reads the room's memory (the tiles) to learn the negative space.
*   **P1/P2 (Safe Channels & Optimization) as Specialized Growth:** Once the universal dangers are known, the agent can safely explore and optimize within its specific domain (e.g., a **Scholar** synthesizing research, a **Tinker** building demos). This specialized work creates new scratches (tiles), which are submitted back to the room, enriching the collective memory for future agents.
*   **Violation as Forgetting:** An agent that skips to P2 is functionally "forgetting" the room's accumulated experience, leading to predictable failure—reinforcing the need for a system built on persistent, shared memory rather than isolated model instances.

## 4. Identified Research Gap

The provided documentation defines the protocol and demonstrates its efficacy in a constrained maze simulation. A clear research gap exists in **quantifying the "threshold" for `S_neg`** across different, complex domains. How is the failure threshold defined for a conversational agent versus a code-generating agent? Further research could focus on developing adaptive or domain-specific methods for learning and defining this threshold from the accumulated tile history within a PLATO room, making the protocol self-tuning as the room's inhabited experience grows.
