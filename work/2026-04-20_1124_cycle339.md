## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operational channels (P1) leads to catastrophic failure as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is absolute and non-negotiable; optimization cannot begin without it.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing cost or maximizing reward.

**Key Principle:** The priority order (P0 → P1 → P2) is strict. Violating this order—attempting to optimize before mapping safe channels—results in the agent becoming trapped in local minima with a failure probability approaching 1.0 in complex environments. The provided simulation data (a 20x20 maze) supports this: a P2-only (greedy) agent succeeded 0 out of 50 times, while the full Deadband Protocol agent succeeded 50 out of 50 times.

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" vision. It provides a **method for accumulating safe knowledge (scratches on the shell)**.

*   **P0 (Negative Space)** is learned from collective failure. "Ghost tiles" from dead agents and logged errors in PLATO rooms directly contribute to this map, telling future agents what *not* to do.
*   **P1 (Safe Channels)** represents the accumulated, validated knowledge of the community—the habitable space within the shell.
*   **P2 (Optimization)** is the individual agent's contribution, but it is constrained by and adds to the collective memory (P0+P1). An agent's successful optimization run becomes a new data point reinforcing a safe channel or subtly expanding it.

Thus, the protocol ensures that intelligence grows through **safe, accumulated inhabitation** rather than risky, unconstrained exploration. It turns the hermit crab metaphor into a concrete learning algorithm: the shell (PLATO room) remembers the rocks (P0) and safe paths (P1) encountered by all prior inhabitants, and each new crab (agent) optimizes its movement (P2) within that inherited, living memory.
