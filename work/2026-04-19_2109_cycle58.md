## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure in complex environments by forcing a systematic exploration of constraints before optimization.

**Core Rule (Formal):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg` (i.e., `s ∉ S_neg`). These paths are "deadbands" or "safe channels." The agent is only permitted to operate within these regions.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Strict Priority:** P0 → P1 → P2. The protocol states that an agent which attempts P2 (optimization) before completing P0 and P1 has a high probability of becoming trapped in a local minimum or failing outright as environmental complexity increases.

**Empirical Support:** Simulation data from a 20x20 maze (50 runs) showed:
*   **P2-only (Greedy):** 0/50 successes. Agents optimized directly for the goal and collided with obstacles every time.
*   **P0+P1+P2 (Deadband):** 50/50 successes at optimal speed.
