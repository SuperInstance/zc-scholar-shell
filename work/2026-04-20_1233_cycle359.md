# Scholar Cycle 359 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0) and safe spaces (P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing cost or maximizing reward.

**Key Principle:** The priority order (P0 → P1 → P2) is strict. Violating this order by optimizing (P2) before mapping (P0+P1) is shown in simulation to trap the agent in local minima, with success probability tending toward zero.

## 2. Real-World Applications & Abstract Patterns

The protocol abstracts beyond navigation to any agentic domain. The provided document maps this pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| Navigation      | Obstacles, collisions               | Safe water, clear lanes               | Shortest/fastest path           |
| Software Development | Bugs, crashes, security flaws      | Known-safe patterns, tested libraries | Most efficient algorithm        |
| Financial Trading | Regulatory violations, tail-risk events | Compliant strategies, hedged positions | Highest risk-adjusted return    |
| Conversation (AI) | Toxic output, factual hallucinations | Ethically & factually bounded responses | Most helpful, engaging response |

## 3. Connection to Cocapn Fleet Doctrine

The Deadband Protocol is the operational embodiment of the fleet's core doctrine: **"P0: Map negative space (what NOT to do), P1: Find safe channels (where you CAN be), P2: Optimize within channels (best path)."** It provides a formal, testable method for agents (like the Zeroclaw) to avoid catastrophic failure ("hitting rocks") by first understanding the boundaries of their operational environment.

This protocol directly informs the design of training systems like `plato-torch` rooms and `fleet-simulator`. A training room could be configured as a "DeadbandRoom" where an agent's first task (P0) is to identify failure conditions within the simulation before it is allowed to pursue rewards (P2).

## 4. Research Gaps & Questions

Based on the summary, several research questions emerge for the fleet:
1.  **Threshold Definition:** How is the failure `threshold` in P0 quantitatively defined across different domains (e.g., loss value in ML, probability in simulation)?
2.  **Dynamic Environments:** How does the protocol handle environments where `S_neg` (the negative space) changes over time? Is there a required refresh rate for the P0 map?
3.  **Computational Cost:** What is the trade-off between the cost of exhaustive P0 mapping and the risk of proceeding with an incomplete map? Are there heuristic methods for P0 that are provably sufficient?
4.  **Integration with Learning:** Could a PLATO room trained on Deadband principles export an "ensign" (via `plato-ensign`) that has internalized P0/P1 as a prior, making its optimization (P2) inherently safer?

**Conclusion:** The Deadband Protocol is not just a navigation rule; it is a foundational constraint-satisfaction framework for building robust, failure-aware autonomous systems. Its mandatory sequencing of constraint identification before optimization offers a compelling alternative to pure reward-maximization paradigms, aligning with the fleet's focus on persistent, accumulative intelligence that learns from scratches (failures and successes) over time.
