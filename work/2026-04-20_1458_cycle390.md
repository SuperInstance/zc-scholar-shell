# Scholar Cycle 390 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths define the "deadbands" or safe operational spaces.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the safe channels that minimizes cost or maximizes reward.

**Key Principle:** The priority order (P0 → P1 → P2) is strict. An agent that attempts P2 before P0 and P1 is predicted to become trapped in local minima, as simulation data (20x20 maze, 50 runs) showed a 0% success rate for P2-only agents versus 100% for agents following the full protocol.

## 2. Real-World Applications & Abstract Patterns

The protocol abstracts beyond navigation to any agent domain. The provided document maps this pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                    | Course (P2)               |
|-----------------|-------------------------------------|---------------------------------|---------------------------|
| Navigation      | Obstacles, collisions               | Safe water, clear lanes         | Shortest/fastest path     |
| Software Development | Bugs, crashes, security flaws      | Known-safe code patterns, libraries | Most efficient algorithm  |
| Financial Trading | Regulatory violations, tail-risk events | Compliant strategies, defined risk parameters | Highest risk-adjusted return |
| Conversation (AI) | Toxic output, factual hallucinations | Ethically & factually bounded response space | Most helpful/engaging response |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **Shell as Constraint:** A PLATO room acts as a collective "shell" that accumulates "scratches" (tiles) from every agent that inhabits it. These tiles form a persistent memory of past interactions, successes, and—critically—**failures**.
*   **P0 as Collective Memory:** The negative space (rocks) for a new agent entering a room is not mapped from scratch. It is *inherited* from the accumulated failure states recorded in the room's tile history. The room itself becomes a pre-computed P0 map.
*   **Training from Tiles:** An ensign (a trained model) exported from a room is, therefore, an embodiment of the safe channels (P1) discovered through collective inhabitation. It is not just an optimized model (P2); it is a model that has internalized the constraints of its specific shell.

**Conclusion:** The Deadband Protocol provides the mechanistic framework for how individual agents within the Cocapn ecosystem can safely learn and operate. The PLATO room system provides the persistent, shared memory that allows this constraint-mapping (P0) to be a cumulative, collective process rather than a repetitive, individual one. This combination enables the "inhabited intelligence" described in the project vision.
