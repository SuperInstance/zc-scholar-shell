# Scholar Cycle 229 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or forbidden zones.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or permissible operating regions.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels, minimizing cost or maximizing reward.

**Strict Priority:** The steps are non-negotiable and sequential. Violation (e.g., optimizing before mapping) is predicted to trap an agent in local minima.

**Empirical Basis:** The document cites simulation data from a 20x20 maze, where a P2-only (greedy) agent succeeded 0 out of 50 runs, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 runs.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a key aspect of the fleet's "inhabited intelligence" thesis.

*   **Shell as Channel:** A PLATO room can be viewed as a **collectively mapped safe channel (P1)**. It accumulates "scratches" (tiles) which are validated experiences—lessons about what works and, crucially, what does not (implicit P0 knowledge).
*   **Training as Constrained Optimization:** An ensign model, trained on a room's tiles, is an **optimization (P2)** performed *within* the channel of validated, safe interactions defined by that room's history. It does not learn from the unbounded space of all possible text, but from the curated deadband of that specific community's work.
*   **Hermit Crab Analogy:** An agent (the crab) seeking to do useful work must first find a suitable shell (a room/channel with relevant P0+P1 knowledge). Its output (scratches/tiles) must pass validation to be added to the shell, thereby refining the channel's boundaries for future agents.

## 4. Identified Research Gaps

Based on the provided documents, open questions include:

1.  **Threshold Definition:** How is the failure `threshold` for defining `S_neg` (P0) determined or learned? Is it static, adaptive, or context-dependent?
2.  **Channel Discovery Cost:** The protocol assumes safe channels can be identified. What are the computational or exploratory costs of P1, especially in novel environments?
3.  **Dynamic Environments:** How does the protocol handle environments where "rocks" move or new constraints emerge? Does it require re-initiating from P0, or is there a incremental update mechanism?
4.  **Multi-Agent Coordination:** How do multiple agents using the Deadband Protocol share and merge their maps of negative space (P0) and safe channels (P1) to avoid conflicting optimizations (P2)?

**Conclusion:** The Deadband Protocol presents a compelling, simulation-backed framework for robust agent decision-making by enforcing constraint mapping before optimization. It finds a direct analogy in the Cocapn fleet's architecture, where PLATO rooms serve as persistent, validated "safe channels" for accumulating and training intelligence.
