## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These constraint-compliant regions are "deadbands" or "safe channels."
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Strict Priority:** P0 → P1 → P2. The protocol states that any agent optimizing (P2) before completing constraint mapping (P0+P1) will become trapped in local minima as complexity rises.

**Empirical Basis:** The document cites simulation data from a 20x20 maze: a P2-only (greedy) agent succeeded 0 out of 50 runs, while a P0+P1+P2 (deadband) agent succeeded 50 out of 50 runs at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The document provides this mapping:

| Domain          | Rocks (P0)                     | Channel (P1)                 | Course (P2)               |
|-----------------|--------------------------------|------------------------------|---------------------------|
| Navigation      | Obstacles                      | Safe water                   | Shortest path             |
| Code            | Bugs, crashes                  | Known-safe patterns          | Most efficient algorithm  |
| Conversation    | Taboo topics, insults          | Polite, constructive topics  | Most persuasive argument  |
| Business        | Legal violations, market risks | Compliant, viable strategies | Highest profit strategy   |
| Learning        | Known misconceptions           | Valid knowledge paths        | Fastest mastery           |
