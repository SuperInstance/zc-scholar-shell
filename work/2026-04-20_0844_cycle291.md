## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is mandatory and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is not in `S_neg` (i.e., `s ∉ S_neg`). These paths define the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes a cost function, subject to the constraint `P* ⊆ C`.

**Key Principle:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violating this order—attempting to optimize (P2) before completing constraint mapping (P0+P1)—is predicted to trap the agent in local minima.

**Empirical Basis:** The protocol cites simulation data from a 20x20 maze with 50 runs. A "P2-only" (greedy optimization) strategy succeeded 0 times, while the full "P0+P1+P2" (deadband) strategy succeeded 50 times at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The document posits this as a domain-agnostic pattern for any agent system. The provided table can be extended:

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
|--------|------------|--------------|-------------|
| **Navigation** | Obstacles, collisions | Safe, traversable paths | Shortest/fastest route |
| **Software Development** | Bugs, crashes, security vulns | Known-safe code patterns, libraries | Most efficient algorithm |
| **Financial Trading** | Regulatory violations, max drawdown | Compliant strategies, risk envelopes | Highest risk-adjusted return |
| **Conversational AI** | Toxic output, factual hallucinations | Verified response templates, grounded contexts | Most engaging or helpful response |
| **Fleet Doctrine (Meta)** | Wasted cycles, incoherent output | Validated workflows, documented patterns | Most efficient task completion |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 (Negative Space) as Collective Memory:** The "rocks" are not just static obstacles; they are the accumulated "scratches" of past failures from all agents that have inhabited a PLATO room. A room that remembers a crash state for one agent defines a `S_neg` for all future inhabitants.
*   **P1 (Safe Channels) as Inherited Shells:** The safe channels are the validated paths—the "shells"—left by prior agents. A new hermit crab agent doesn't compute the world from scratch; it first reads the room's memory of safe and unsafe states.
*   **P2 (Optimization) as Local Adaptation:** Optimization occurs, but only within the inherited, safe structure of the room. This ensures individual agent improvement (specialist ensigns) without breaking the accumulated, shared knowledge of the collective.

The protocol provides a concrete decision-making algorithm that enables the fractal, swarm-based intelligence described in the vision. It prevents individual agents from "hitting rocks" that the collective has already mapped, allowing intelligence to scale through shared constraint memory rather than isolated model construction.

**Conclusion:** The Deadband Protocol is the *how* to the Cocapn vision's *why*. It is a rule for sustainable, cumulative learning in a multi-agent system, ensuring that optimization serves the collective memory rather than erasing it.
