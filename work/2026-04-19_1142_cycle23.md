# Scholar Cycle 23 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core tenet is that optimization (P2) is only safe and effective after first mapping negative space (P0) and identifying safe channels (P1).

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` remains within `C`.

**Key Principle:** The priority order P0 → P1 → P2 is strict. The protocol asserts that an agent which optimizes (P2) before completing P0 and P1 has a high probability of becoming trapped in local minima or failing outright as environmental complexity increases. Simulation data (a 20x20 maze, 50 runs) is cited as evidence: a P2-only (greedy) approach succeeded 0 times, while the full deadband protocol succeeded 50 times.

## 3. Connection to Cocapn Vision

The Deadband Protocol operationalizes a core principle implied in `PROJECT-VISION.md`: **intelligent growth through constrained, incremental accumulation.** The vision describes intelligence as "inhabited," not built—like a hermit crab accumulating scratches in a shell.

*   **P0/P1 as the Shell:** The protocol's "safe channels" are the structural constraints of the "shell" (a PLATO room, a codebase convention, a research methodology). They define the bounded space where learning and accumulation can safely occur.
*   **P2 as the Scratch:** The optimized work within the channel—the code, the analysis, the successful agent interaction—becomes the "scratch" or "tile" that is added to the shell's memory.
*   **Violation as Forgetting:** An agent that skips to P2 and fails is an agent that did not respect the accumulated constraints (scratches) of the shell, leading to a failure that does not contribute to useful memory.

Thus, the Deadband Protocol is the *how*—the specific, repeatable process—for achieving the vision's *what*: creating persistent, shared spaces where intelligence grows through safe, accumulated experience.
