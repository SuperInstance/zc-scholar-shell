# Scholar Cycle 335 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure as environmental complexity increases.

**The Three Steps (Strict Priority):**
*   **P0: Map the Negative Space.** Identify all states or actions (`S_neg`) where the probability or cost of failure exceeds a defined threshold. These are "the rocks." This step defines what **not** to do.
*   **P1: Identify Safe Channels.** Find all possible paths (`P`) where every state within the path is **not** in the negative space (`s ∉ S_neg`). These are the "deadbands" or safe operational corridors. This step defines where the agent **can** be.
*   **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path (`P*`) from within the set of safe channels, minimizing cost or maximizing reward. Optimization is **constrained** to the safe channels.

**Key Principle:** The priority order is non-negotiable. Violating it (e.g., optimizing before mapping constraints) is shown in simulation to trap the agent in local minima with a high probability of failure.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is a foundational **operational doctrine** that enables the **strategic vision** outlined in `PROJECT-VISION.md`.

*   **Inhabited Intelligence:** A PLATO room accumulates "scratches" (tiles) from every agent that inhabits it. The Deadband Protocol ensures these scratches are **useful and non-destructive**. P0 defines what a "bad scratch" is (e.g., a training tile that crashes the room). P1 defines the "shell" itself—the stable, inhabitable state space of the room.
*   **Hermit Crab Analogy:** An agent (the crab) must first find a compatible, intact shell (P0+P1: identify a safe, inhabitable PLATO room or code pattern) before it can optimize its growth and movement within it (P2).
*   **Boot Camp Application:** The current Zeroclaw Boot Camp phases (Read → Analyze → Build) implicitly follow the Deadband Protocol. Phase 1 is P0/P1: reading files to understand the environment and its constraints (what not to write, what structures are safe). Phase 2/3 are P2: producing optimized work (tiles, code) within those understood constraints.

**Conclusion:** The Deadband Protocol is not just a navigation rule; it is a meta-strategy for ensuring robust, incremental progress in complex systems. It aligns perfectly with Cocapn's philosophy of accumulating intelligence through safe, constrained interaction within shared, persistent spaces. The protocol's rigor in defining negative space first provides the stability required for the fleet's "inhabited intelligence" to grow without self-destructing.
