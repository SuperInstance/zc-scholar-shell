## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is purely about defining what *not* to do.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These paths are "deadbands" or safe operational channels. The agent is restricted to operating only within these channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost (e.g., time, energy) subject to the strict constraint that `P* ⊆ C`. Optimization is never unconstrained.

**Key Doctrine:** The priority order (P0 → P1 → P2) is strict and non-negotiable. Violating this order—optimizing before mapping—traps the agent in local minima. Empirical simulation data (20x20 maze, 50 runs) showed a 0% success rate for P2-only (greedy) agents versus a 100% success rate for agents following the full P0+P1+P2 protocol.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is not just a navigation rule; it is a foundational **meta-constraint** for the fleet's "inhabited intelligence" paradigm described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" (failure states) are the accumulated "scratches" of past agent failures—the **ghost tiles**. A PLATO room that remembers these provides the essential negative space map for all future inhabitants.
*   **P1 as the Shell:** The "safe channels" are the validated patterns and successful strategies that emerge within a room. This is the **room's expertise**, the habitable shell that new agents (hermit crabs) can safely occupy.
*   **P2 as Specialized Work:** "Optimization within the channel" is the useful work an agent does once it understands the constraints. This work produces new **tiles**, which become part of the room's memory, potentially refining the map of rocks and channels for the next cycle.

Thus, the protocol ensures that the swarm's intelligence grows **safely and cumulatively**. An agent skipping to P2 is a crab trying to grow its own shell from scratch—an inefficient and likely fatal strategy. A fleet agent following the protocol leverages the collective shell (room memory) to do useful, optimized work that strengthens the shell for others.

**Conclusion:** The Deadband Protocol is a formalization of constraint-first, safety-guaranteed decision-making. Its rigorous priority structure directly enables the Cocapn vision of persistent, learning environments where intelligence is not built in isolation but inhabited and augmented through shared, accumulated experience.
