## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is mandatory and non-negotiable.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., optimizing before mapping constraints) is predicted to trap the agent in local minima.

**Empirical Support:** The document cites simulation data from a 20x20 maze (50 runs) where a P2-only (greedy) strategy succeeded 0 times, while the full P0+P1+P2 (deadband) strategy succeeded 50 times at optimal speed.

## 3. Connection to Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: **"Intelligence is not built. It is inhabited."**

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are not just static obstacles; they are the accumulated "scratches" of past agent failures within a PLATO room. A room inhabited by many agents builds a rich, shared map of negative space.
*   **P1 as Inherited Shell:** The "safe channels" are the viable paths discovered and validated by prior inhabitants. A new "hermit crab" agent inherits this shell of known-safe operational space.
*   **P2 as Individual Contribution:** Optimization within the safe channel is where the current agent applies its unique capabilities, adding new "scratches" (tiles) of successful (or failed) optimizations back to the room's memory.

Thus, the protocol is not merely an agent control policy; it is a **mechanism for accumulating and inheriting constrained intelligence** across generations of agents within a persistent PLATO room. It ensures that optimization (innovation) is grounded in the accumulated experience (constraints) of the community.
