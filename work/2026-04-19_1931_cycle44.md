## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an agent in state `S` with goal `G` in environment `E`:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that `P*` is a subset of `C`.

**Strict Priority:** P0 > P1 > P2. An agent that attempts P2 (optimization) before completing P0 and P1 has a high probability of becoming trapped in a local minimum or failing catastrophically as environmental complexity increases.

**Empirical Proof:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, while the full P0+P1+P2 protocol achieved a 100% success rate at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol abstracts to any domain where an agent operates within constraints.

| Domain          | Rocks (P0 - What NOT to do)                          | Channel (P1 - Where you CAN be)                                  | Course (P2 - Best path)                          |
| :-------------- | :--------------------------------------------------- | :--------------------------------------------------------------- | :----------------------------------------------- |
| **Navigation**  | Obstacles, cliffs, hostile zones                     | Clear pathways, safe water                                       | Shortest/fastest route within safe pathways      |

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol is not just a rule; it is a foundational principle for building "inhabited intelligence" as described in `PROJECT-VISION.md`.

*   **Hermit Crab Analogy:** The protocol defines how a "crab" (agent) should interact with its "shell" (environment/PLATO room). P0 is learning the shell's weak points. P1 is finding where to move inside it. P2 is moving efficiently.
*   **Training Tiles:** An agent's application of the protocol—its successes and failures in mapping rocks and channels—generates the "scratches" (tiles) that accumulate in a PLATO room. This shared memory teaches subsequent agents.
*   **Ghost Tiles:** The lessons from agents that violated the protocol and failed become critical P0 knowledge (negative space maps) for future agents.
*   **plato-torch Rooms:** Training rooms can be designed with the Deadband Protocol as a core mechanic, rewarding agents for correct P0/P1 behavior before P2 optimization.
*   **Fleet Safety:** For a multi-agent fleet (Oracle1, JetsonClaw1, Forgemaster), the protocol provides a common operational doctrine to prevent individual agent failures from cascading.
