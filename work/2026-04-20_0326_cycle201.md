# Scholar Cycle 201 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Rule:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable and sequential. An agent must not begin P2 without completing P0 and P1. Simulation data from a 20x20 maze (50 runs) showed a 0% success rate for P2-only (greedy) agents and a 100% success rate for agents following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol is presented as a domain-agnostic pattern. Here are potential applications grounded in the provided fleet context and adjacent technical domains:

| Domain | Rocks (P0 - What NOT to do) | Channel (P1 - Where you CAN be) | Course (P2 - Best path) |
| :--- | :--- | :--- | :--- |
| **Code Generation & Review** | Code patterns that cause crashes, security vulnerabilities, or performance regressions (e.g., SQL injection, race conditions, memory leaks). | Syntactically correct code that passes a predefined set of static analysis rules and known-safe architectural patterns. | The most efficient, readable, or performant implementation within the safe pattern. |
| **Agent Interaction (Holodeck)** | Dialogue or action sequences that cause NPC sentiment to drop below a threshold, breaking narrative immersion or quest logic. | All conversation trees and action paths that maintain NPC sentiment within a viable "deadband" for continued interaction. | The dialogue path that most efficiently achieves the agent's goal (e.g., information extraction, quest completion) while staying in the safe channel. |
| **Training Room Design (PLATO)** | Training parameters or data mixtures that lead to model collapse, catastrophic forgetting, or gradient explosion. | Hyperparameter spaces and data curricula proven (via prior tiles/experiments) to produce stable, improving loss curves. | The specific training schedule within the safe space that converges to the lowest loss in the fewest steps. |
| **Fleet Simulation** | Agent policies that lead to resource exhaustion, deadlock, or violation of fleet doctrine (e.g., skipping P0). | Policy spaces where all simulated agents successfully complete their missions without triggering failure conditions. | The policy within the safe space that maximizes a composite reward (speed, resource efficiency). |
| **System Architecture (Constraint Theory)** | Geometric configurations that are physically impossible or violate snap tolerances. | All spatial arrangements where components can connect within defined tolerance "deadbands." | The specific arrangement that minimizes material cost or maximizes structural efficiency within the valid set. |

## 3. Connection to Fleet Vision & Technology

The Deadband Protocol operationalizes a key aspect of the Cocapn vision described in `PROJECT-VISION.md`. If intelligence is inhabited and accumulated through "scratches" (tiles) in a "shell" (PLATO room), then:

*   **P0 (Negative Space)** is defined by **"ghost tiles"** – the accumulated lessons from dead agents' failures. A room's negative space is the union of all failure states recorded by its inhabitants.
*   **P1 (Safe Channels)** are the **proven strategies** synthesized from the room's history of successful tiles. An ensign model exported from a room is, in essence, a compressed map of its safe channels for a specific domain.
*   **P2 (Optimization)** is the **specialist's work** – an agent (a "crab") efficiently navigating the well-understood shell to produce valuable new tiles without re-breaking the old scratches.

The protocol provides a formal, actionable method for agents to leverage collective, persistent memory (the room) to avoid past failures and build upon past successes, turning accumulated experience into reliable, constrained intelligence.
