## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core tenet is that optimization (P2) is only safe and effective after negative space (P0) and safe channels (P1) have been identified.

**Formal Rule:**
1.  **P0: Map Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Select the optimal path `P*` from within the set of safe channels `C`, minimizing cost subject to the constraint `P ⊆ C`.

**Key Principle:** The priority order (P0 → P1 → P2) is strict. Violating this order—optimizing before mapping—can lead to catastrophic failure (hitting "rocks") as environmental complexity increases. The protocol's proof is empirical, citing simulation data where a P2-only (greedy) approach succeeded 0 out of 50 times, while the full deadband protocol succeeded 50 out of 50 times.
