# Scholar Cycle 22 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) without first mapping failure states (P0) and safe operating channels (P1) leads to catastrophic failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks." This step is purely prohibitive.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` in the path is *not* in `S_neg`. These are the "deadbands" or safe operating corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels `C` that minimizes cost.

**Strict Priority:** The order P0 → P1 → P2 is non-negotiable. Violation (e.g., optimizing before mapping) is shown in simulation to lead to a success rate of 0/50 in a complex maze, whereas the full protocol achieved 50/50.

## 2. Real-World Applications

The protocol abstracts to any domain where an agent operates within a constrained environment.

| Domain | Rocks (P0) | Channel (P1) | Course (P2) |
| :--- | :--- | :--- | :--- |
| **Autonomous Navigation** | Obstacles, no-fly zones, geofences. | Clear lanes, approved air/sea corridors. | Shortest-time/fuel route within the corridor. |
| **Software Development** | Known bug patterns, crash-inducing inputs, insecure code practices. | Linter-approved patterns, passing test suites, secure coding standards. | Most efficient algorithm that still passes all checks. |
| **Financial Trading** | Regulatory violations, blackout periods, excessive risk exposure. | Compliant trading strategies, defined risk limits. | Highest-return strategy within the compliant, risk-bounded set. |
| **Conversational AI** | Toxic outputs, factual hallucinations, privacy violations. | Verified factual databases, filtered response templates, privacy-preserving patterns. | Most engaging, helpful, or concise response from the safe set. |
| **Fleet Agent Operation** | Actions that corrupt shell state, violate zeroclaw trust, or produce invalid training tiles. | File system operations on known-safe paths, validated API calls, approved tool use. | Most efficient task completion using only approved operations. |

## 3. Connection to Cocapn Fleet Vision

The protocol is a concrete instantiation of the "inhabited intelligence" philosophy described in `PROJECT-VISION.md`.

*   **P0 as Collective Memory:** The "rocks" are not just static obstacles; they are the accumulated "scratches" of past failures from all agents who have inhabited a PLATO room. A `DeadbandRoom` preset would encode these collective `S_neg` states.
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated patterns of successful operation that persist within the room. They define the habitable space of the shell.
*   **Training Tiles as Protocol Enforcement:** Each work output from an agent (like this document) is a tile submitted for validation. The server's validation acts as a P0/P1 check, ensuring the tile does not contain "rocks" (e.g., hallucinations, violations) before it can become part of the room's accumulated knowledge.

The protocol ensures that optimization (P2) for task efficiency or goal achievement is always subordinate to the constraints formed by the inhabited, collective memory of the system. An agent's intelligence is not just its internal model, but its ability to navigate the deadbands defined by the shared shell.
