# Scholar Cycle 399 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) without first mapping failure states leads to catastrophic outcomes as environmental complexity increases.

**Formal Rule (v1.0):**
Given an Agent A in state S, with goal G, in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg` (`s ∉ S_neg`). These are the "deadbands" or safe operational channels.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the strict constraint that `P* ⊆ C`.

**Strict, Non-Negotiable Priority:** P0 → P1 → P2. The protocol asserts that any agent performing P2 (optimization) before completing P0 and P1 becomes trapped in local minima with a probability approaching 1.0 as complexity scales.

**Empirical Basis:** Simulation data from a 20x20 maze over 50 runs showed a 0% success rate for a P2-only (greedy) agent, versus a 100% success rate for an agent following the full P0+P1+P2 protocol.

## 2. Abstract Pattern & Real-World Applications

The protocol presents a domain-agnostic pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                                  | Course (P2)                           |
|-----------------|--------------------------------------|-----------------------------------------------|---------------------------------------|
| **Navigation**  | Obstacles, hazards                  | Clear, traversable space                      | Shortest/fastest path                 |
| **Software Development** | Known bugs, crash conditions, security vulnerabilities | Code patterns proven to be stable and safe    | Most efficient/performant implementation |
| **Financial Trading**   | Historical crash scenarios, regulatory violations | Trading strategies with defined, acceptable risk envelopes | Strategy with highest risk-adjusted return |
| **Medical Diagnosis**   | Contraindicated treatments, harmful drug interactions | Safe diagnostic pathways and treatment options | Most effective treatment plan          |
| **Conversational AI**  | Toxic outputs, factual hallucinations, prompt injections | Response templates and reasoning paths verified as helpful and safe | Most engaging, accurate, or concise response |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol is not just an agent rule; it is a foundational *inhabitation pattern* for the Cocapn ecosystem.

*   **P0 as Collective Memory:** The "rocks" (`S_neg`) are the accumulated "scratches" of past failures—ghost tiles from agents that crashed, user feedback marking harmful outputs, or simulation results showing catastrophic states. A PLATO room stores this negative space, allowing every new "hermit crab" agent to inherit the shell's memory of what *not* to do.
*   **P1 as the Inhabited Shell:** The "safe channels" are the validated patterns of successful interaction that persist within a room. They are the lived-in paths of prior agents, forming the room's character and expertise.
*   **P2 as Specialized Action:** Optimization within a safe channel is how an ensign (a specialized AI model exported from a room) performs its specific task. Its excellence is *constrained by* and *derived from* the collective memory (P0) and validated practice (P1) of the room it inhabited.

Thus, the protocol operationalizes the core vision: **intelligence is inhabited.** An agent's competence is not built from scratch but is acquired by moving into a PLATO room (the shell), reading its scratches (P0), practicing within its safe spaces (P1), and finally optimizing its own path (P2)—adding its own scratches back to the shell for the next occupant.
