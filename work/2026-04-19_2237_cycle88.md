## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. It is designed to prevent catastrophic failure by forcing the agent to first identify and avoid failure states before attempting optimization.

**Formal Rule (v1.0):**
Given an agent A in state S, with goal G in environment E:
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "the rocks."
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in `S_neg`. These are the "deadbands" or safe operational zones.
3.  **P2: Optimize Within Channel.** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint `P* ⊆ C`.

**Strict Priority:** The steps are non-negotiable. An agent must not proceed to P2 without completing P0 and P1. The document states that violating this order—optimizing before mapping—leads to a high probability of becoming trapped in local minima as environmental complexity increases.

**Empirical Support:** The protocol is supported by simulation data from a 20x20 maze environment. Agents using a greedy "P2-only" approach succeeded 0 out of 50 times, while agents following the full P0+P1+P2 protocol succeeded 50 out of 50 times at optimal speed.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table illustrates this:

| Domain          | Rocks (P0)                   | Channel (P1)                     | Course (P2)               |
|-----------------|------------------------------|----------------------------------|---------------------------|
| Navigation      | Obstacles, collisions        | Safe water, clear lanes          | Shortest path             |
| Software Development | Bugs, crashes, security flaws | Known-safe patterns, tested libraries | Most efficient implementation |

**Extended Application Examples:**
*   **Machine Learning Training:** P0: Identify hyperparameter combinations that cause divergence or overfitting. P1: Define the stable region of the hyperparameter space. P2: Find the fastest-converging combination within that region.
*   **Conversational AI:** P0: Map user inputs or response types that lead to harmful, biased, or nonsensical outputs. P1: Define the space of safe, compliant, and coherent response strategies. P2: Optimize for the most helpful or engaging response within that safe space.
*   **Fleet Operations (Meta-application):** P0: Identify agent behaviors or environmental conditions that cause system crashes or training failures. P1: Define the operational parameters and interaction protocols that keep agents stable. P2: Optimize agent performance (e.g., tile generation rate) within those stable parameters.

## 3. Connection to Cocapn Fleet Vision

The Deadband Protocol is a concrete implementation of the "inhabited intelligence" philosophy outlined in `PROJECT-VISION.md`.

*   **Accumulated Scratches as P0 Knowledge:** The "rocks" (`S_neg`) are not pre-programmed but are learned from accumulated experience—the "scratches" left by previous agents in a PLATO room. A room that has recorded many failures (crashes, poor outputs) builds a rich, shared map of the negative space for future inhabitants.
*   **Ensigns as Compiled Safe Channels:** A trained "ensign" (a LoRA adapter exported from a room) can be seen as a pre-computed safe channel (P1). It embodies the collective, validated knowledge of what *not* to do and the proven paths that remain, allowing new agents to skip directly to constrained optimization (P2).
*   **Hermit Crab Analogy:** An agent (the crab) entering a PLATO room (the shell) immediately inherits the deadband map—the accumulated scratches of prior occupants. Its work is to operate within those channels and add its own scratches, refining the map for the next crab.

## 4. Identified Research Gaps & Questions

Based on the provided documents, several open questions emerge:

1.  **Threshold Definition:** How is the failure threshold in P0 quantitatively defined? Is it static or adaptive based on context/risk tolerance?
2.  **Channel Discovery Complexity:** In high-dimensional, continuous state spaces (like natural language or complex simulations), how is the complete set of safe channels (P1) discovered or approximated efficiently?
3.  **Dynamic Environments:** The protocol description implies a static map of `S_neg`. How does it handle environments where the "rocks" move or new ones appear? Does this require a continuous P0 process running in parallel with P1/P2?
4.  **Inter-Agent Deadbands:** How are safe channels negotiated or shared among a multi-agent fleet like Cocapn? Could one agent's optimal path (P2) become another agent's rock (P0)?

**Conclusion:** The Deadband Protocol provides a robust, simulation-validated framework for safe agent operation. Its integration with the PLATO room system suggests it is a foundational constraint for the Cocapn fleet's approach to cumulative, shared learning. The primary research gaps lie in its scalability to complex, dynamic, and multi-agent domains.
