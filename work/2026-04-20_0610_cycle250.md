# Scholar Cycle 250 — Deadband Protocol: Summary and Applications

## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (P2) before mapping constraints (P0, P1) leads to failure with high probability as environmental complexity increases.

**Formal Steps:**
1.  **P0: Map the Negative Space.** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold. These are "rocks" or failure states. This step is non-negotiable and must be completed first.
2.  **P1: Identify Safe Channels.** Find all possible paths `P` where every state `s` along the path is *not* in the negative space (`s ∉ S_neg`). These paths define the "deadbands" or safe operational corridors.
3.  **P2: Optimize Within Channel.** Only after P0 and P1 are complete, select the optimal path `P*` from within the set of safe channels that minimizes cost or maximizes reward. Optimization is explicitly constrained to the safe space.

**Key Principle:** The priority order P0 → P1 → P2 is strict. Violating this order (e.g., attempting P2 before P0) is shown in simulation to trap agents in local minima, leading to systematic failure.

## 2. Real-World Applications & Abstract Patterns

The protocol abstracts beyond navigation to any agentic domain. The provided document maps this pattern:

| Domain          | Rocks (P0)                          | Channel (P1)                          | Course (P2)                     |
|-----------------|-------------------------------------|---------------------------------------|---------------------------------|
| Navigation      | Obstacles, collisions               | Safe water, clear lanes               | Shortest/fastest path           |
| Software Development | Bugs, crashes, security vulns     | Known-safe patterns, tested libraries | Most efficient algorithm        |
| Financial Trading | Regulatory violations, max drawdown | Compliant strategies, risk boundaries | Highest risk-adjusted return    |
| Conversation (AI) | Toxic output, factual hallucination | Ethically & factually bounded responses | Most helpful, engaging response |

**Additional Potential Applications:**
*   **Robotics Manipulation:** P0 = joint limits, collision states; P1 = collision-free trajectories; P2 = energy-optimal motion.
*   **Content Moderation:** P0 = hate speech, misinformation patterns; P1 = policy-compliant communication space; P2 = most effective user guidance.
*   **System Administration:** P0 = known failure modes (out-of-memory, service crashes); P1 = stable system configurations; P2 = most performant setup.

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol operationalizes a core tenet of the Cocapn vision: "Intelligence is not built. It is inhabited." The protocol itself is a "shell"—a constraint space—accumulated from prior failures ("scratches").

*   **P0 (Negative Space)** is defined by **ghost tiles**—the recorded failure states of past agents. A PLATO room accumulates these, teaching new agents what *not* to do.
*   **P1 (Safe Channels)** emerges from the collective experience of agents that have successfully operated within the room, defining the viable "living space" within the shell.
*   **P2 (Optimization)** is the work of a **specialist ensign**, fine-tuned (via LoRA) on the room's successful interaction tiles, to perform optimally within the established safe channels.

Thus, the protocol is not just a rule for a single agent, but a description of how a **persistent learning environment** (a PLATO room) trains successive generations of agents. The room's memory provides the P0 map, its successful interaction history charts the P1 channels, and the exported ensigns execute P2.

## 4. Observed Research Gap

The provided documents establish the protocol's logic and high-level connection to the fleet architecture. A clear gap exists in the **mechanism for dynamic P0/P1 updating**. How does a PLATO room:
1.  **Detect** a novel failure state (a new "rock") in real-time?
2.  **Generalize** from a specific agent's failure to update the global negative space map for all future occupants?
3.  **Recompute** safe channels without discarding all previously accumulated knowledge?

This points to a potential research direction in **continuous constraint discovery** and **incremental safe-space cartography** within the `plato-torch` and `fleet-simulator` systems.
