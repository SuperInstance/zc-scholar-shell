## 1. Protocol Summary (Based on `DEADBAND-PROTOCOL.md`)

The Deadband Protocol is a three-step, strictly prioritized decision-making framework for autonomous agents. Its core thesis is that unconstrained optimization (seeking the "best" path) is highly likely to fail in complex environments unless the agent first identifies and avoids catastrophic failure states.

The protocol is defined as follows:

**P0: Map the Negative Space**
*   **Action:** Identify all states `S_neg` where the probability or cost of failure exceeds a defined threshold.
*   **Analogy:** "The rocks." These are the things the agent must **not** do.
*   **Rule:** This step is mandatory and non-negotiable. Optimization cannot begin without this map.

**P1: Identify Safe Channels**
*   **Action:** Find all possible paths `P` where every state `s` along the path is **not** in the negative space (`s ∉ S_neg`).
*   **Analogy:** "The safe water" or "deadbands." These are the regions where the agent **can** operate.
*   **Rule:** The agent's search space is constrained to these safe channels.

**P2: Optimize Within Channel**
*   **Action:** From the set of safe channels `C`, select the optimal path `P*` that minimizes cost, subject to the constraint that the path remains within a safe channel (`P ⊆ C`).
*   **Analogy:** "The best course through safe water."
*   **Rule:** Optimization is always and only performed **within** the constraints established by P0 and P1.

**Violation Condition:** The document states that an agent which attempts P2 (optimization) before completing P0 and P1 (constraint mapping) will, with high probability, become trapped in a local minimum or fail catastrophically as environmental complexity increases. Empirical simulation data (a 20x20 maze over 50 runs) is cited as evidence: a P2-only greedy approach succeeded 0 times, while the full P0+P1+P2 protocol succeeded 50 times.

## 2. Real-World Applications & Abstract Pattern

The protocol is presented as a domain-agnostic pattern. The provided table and the document's abstract suggest the following applications:

| Domain | P0: Rocks (Negative Space) | P1: Channel (Safe Water) | P2: Course (Optimization) |
| :--- | :--- | :--- | :--- |
| **Physical Navigation** | Obstacles, cliffs, hostile areas. | Clear, traversable terrain. | Shortest/fastest route. |
| **Software Development** | Code patterns that cause bugs, crashes, or security vulnerabilities. | Known-safe APIs, design patterns, and tested libraries. | Most efficient or elegant implementation. |
| **Financial Trading** | Market conditions or asset classes with high risk of ruin. | Statistically validated, risk-bounded strategies. | Strategy with the highest risk-adjusted return. |
| **Conversational AI** | Prompts or topics that lead to harmful, biased, or nonsensical outputs. | Context windows and phrasing known to produce helpful, factual responses. | Most engaging, informative, or concise response. |
| **Machine Learning Training** | Hyperparameter sets or data batches that cause model collapse or divergence. | Ranges and selections known to produce stable, convergent training. | Hyperparameters that achieve the lowest loss or highest accuracy. |

## 3. Connection to Cocapn Fleet Vision (`PROJECT-VISION.md`)

The Deadband Protocol aligns with the Cocapn fleet's core thesis of "inhabited intelligence." It provides a **behavioral scaffold** for the "hermit crab" agents.

*   **P0 as Collective Memory:** The "rocks" (failure states) are not rediscovered by each new agent. They are the accumulated "scratches" on the PLATO room's shell—lessons from past failures (ghost tiles) that define what **not** to do. This is the room's persistent, negative knowledge.
*   **P1 as Inherited Habitat:** The "safe channels" are the proven paths and patterns that previous inhabitants have successfully traversed. A new agent (a "greenhorn") inherits this safe operating environment, allowing it to be productive immediately without relearning foundational constraints.
*   **P2 as Agent Specialization:** Once operating within a safe channel, the agent is free to optimize and innovate. Its unique "work output" within these bounds becomes new tiles, which may refine the channel's boundaries or create new, specialized paths for future agents.

In this view, the Deadband Protocol is the **operational doctrine** that enables the safe, scalable accumulation of intelligence within a shared, persistent shell (the PLATO room). It ensures that exploration and optimization by individual agents contribute to—rather than destabilize—the collective knowledge base.
